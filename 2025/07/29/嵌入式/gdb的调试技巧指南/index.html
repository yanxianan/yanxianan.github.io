GCC/G++ 调试技巧指南

在程序开发过程中，调试是定位和解决问题的关键环节。GCC/G++ 配合 GDB（GNU 调试器）提供了强大的调试能力，能够帮助开发者追踪程序执行流程、查看变量状态、分析内存问题等。本文将详细介绍基于 GCC/G++ 和 GDB 的调试技巧，涵盖从编译准备到高级调试的全流程。

一、调试前的编译准备

要使用 GDB 调试程序，需在编译阶段生成包含调试信息的可执行文件。GCC/G++ 通过 -g 选项添加调试信息（如变量名、行号、函数信息等），常用的调试级别如下：

  • -g：生成基本调试信息（推荐默认使用）。

  • -g3：包含更多调试信息（如宏定义、注释等），适合复杂项目。

  • -ggdb：生成 GDB 专用的调试信息，优化调试体验。

3. 监视断点：跟踪变量 / 表达式变化

监视断点（watch）用于在变量或表达式的值发生变化时自动暂停程序，适合追踪变量被意外修改的场景。

  • watch <变量/表达式>：当值被修改时暂停（写监视）。

  • rwatch <变量/表达式>：当值被读取时暂停（读监视）。

  • awatch <变量/表达式>：当值被读取或修改时暂停（读写监视）。

  1. 宏调试：

  1. 保存调试会话：

通过 save commands <文件> 保存当前调试命令，下次通过 source <文件> 加载，避免重复输入：

  1. 避免常见陷阱：
    • 优化导致的变量不可见：若变量被 -O2 等优化掉，需改用 -O0 或 -Og 编译。
    • 多线程竞争：使用 scheduler-locking 锁定线程，逐步调试竞争条件。
    • 动态库调试：确保动态库也包含调试信息（编译时加 -g），否则可能无法查看库内函数。

八、总结

GCC/G++ 与 GDB 结合提供了强大的调试能力，核心流程可概括为：

  1. 用 -g 编译含调试信息的程序；

  2. 通过 GDB 启动程序，设置断点控制执行；

  3. 利用 print、backtrace 等命令查看变量和调用栈；

  4. 结合多线程调试、核心转储分析等高级功能解决复杂问题。

掌握这些技巧能显著提升调试效率，尤其在定位逻辑错误、内存泄漏、多线程竞争等问题时不可或缺。实际开发中，可根据项目复杂度灵活选择调试命令，逐步积累调试经验。
