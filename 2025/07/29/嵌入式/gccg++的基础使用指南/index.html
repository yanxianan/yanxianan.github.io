GCC/G++ 的基础使用

一、GCC 简介

GCC（GNU Compiler Collection，GNU 编译器套件）是由 GNU 开发的开源编程语言编译器，最初仅支持 C 语言，后逐步扩展至 C++、Objective-C、Fortran、Java、Ada 和 Go 等多种语言，同时包含这些语言的库（如 C++ 的 libstdc++、Java 的 libgcj 等）。它是跨平台开发的核心工具之一，广泛应用于 Linux、Unix 及 Windows（通过 MinGW 等工具链）环境。

程序编译的核心目标是将人类可读的源代码（如 C/C++ 代码）转化为计算机可直接执行的机器代码，这一过程主要分为预处理、编译、汇编、链接四个阶段，每个阶段都有明确的任务和输出结果。

二、程序编译的四个阶段

（一）预处理阶段：宏替换与文本转换

预处理是编译流程的第一步，主要对源代码中的特殊指令和符号进行处理，生成无宏定义、无条件编译指令的中间文件。具体处理内容包括：

  1. 宏定义替换：将#define定义的宏在代码中展开（如#define PI 3.14会替换所有PI为 3.14）。

  2. 文件包含处理：解析#include指令，将指定的头文件内容插入到当前文件中（如#include <stdio.h>会插入标准输入输出库的声明）。

  3. 条件编译处理：根据#ifdef、#ifndef、#if等指令选择性保留代码（如调试时保留日志打印，发布时移除）。

  4. 特殊符号替换：将__LINE__（当前行号）、FILE（当前文件名）等特殊符号替换为实际值。

（二）编译阶段：生成汇编代码

编译阶段会对预处理后的文件（.i）进行语法检查，确认无误后将其转换为汇编语言代码（.s）。此阶段会检测代码规范性（如变量未定义、函数参数不匹配等），并优化代码逻辑。

汇编阶段将汇编代码（.s）转换为机器可识别的二进制目标文件（.o或.obj）。目标文件包含函数和变量的二进制指令，但尚未解决外部引用（如调用其他文件的函数）。

（四）链接阶段：生成可执行文件或库

链接阶段将多个目标文件（.o）及所需的库文件组合，解决外部引用问题，最终生成可执行文件或库文件。链接分为静态链接和动态链接两种方式。

1. 静态链接

  • 原理：将库函数的代码直接拷贝到可执行文件中。

  • 优点：可执行文件独立运行，不依赖外部库。

  • 缺点：文件体积大，库更新后需重新编译程序。

2. 动态链接

  • 原理：仅在可执行文件中记录库的引用，运行时动态加载库文件。

  • 优点：文件体积小，库更新后无需重新编译程序。

  • 缺点：依赖系统中存在对应的动态库，否则无法运行。

三、常用 GCC 选项总结

选项 作用
-E 仅执行预处理，生成.i文件
-S 执行预处理和编译，生成.s文件
-c 执行预处理、编译、汇编，生成.o文件
-o 指定输出文件名称
-Wall 显示所有警告信息
-O 开启优化（level为 1-3，0 表示关闭）
-I 指定头文件搜索路径（如-I./include）
-L 指定库文件搜索路径（如-L./lib）
-l 链接指定的库（如-lm链接数学库libm.so）
-static 强制使用静态链接

六、总结

GCC 是开发 C/C++ 程序的核心工具，其编译过程分为预处理、编译、汇编、链接四个阶段。通过掌握各阶段的命令和选项，可灵活控制编译流程，优化程序性能。实际开发中，需根据需求选择静态链接或动态链接，并合理使用库文件管理代码复用。
