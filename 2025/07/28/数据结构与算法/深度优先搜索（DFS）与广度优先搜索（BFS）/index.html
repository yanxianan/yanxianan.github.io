深度优先搜索（DFS）与广度优先搜索（BFS）

发表于 2025-07-28 更新于 2025-09-01 分类于 数据结构与算法
本文详细介绍深度优先搜索（DFS）和广度优先搜索（BFS）的原理、实现方法、适用场景及示例代码，帮助读者掌握这两种经典搜索算法。

深度优先搜索（DFS）和广度优先搜索（BFS）是两种常用的图遍历算法，在数据结构、算法设计及实际应用中有着广泛的用途。本文将深入解析这两种算法的核心思想、实现方式、优缺点及典型应用场景。

一、算法基本概念

1. 深度优先搜索（DFS）

深度优先搜索是一种沿着图的深度优先遍历图的节点的算法。它的核心思想是：从起始节点出发，尽可能深地探索分支，当无法继续前进时，回溯到上一节点，选择其他未探索的分支继续探索，直到所有节点都被访问过。

可以形象地理解为 “不撞南墙不回头”，优先深入探索一条路径，而非横向扩展。

2. 广度优先搜索（BFS）

广度优先搜索则是一种按层次逐层遍历图的节点的算法。其核心思想是：从起始节点出发，先访问该节点的所有邻接节点（第一层），再依次访问每个邻接节点的邻接节点（第二层），以此类推，直到所有节点都被访问过。

类似于 “水波扩散”，从起点开始，由近及远地遍历所有节点。

二、算法实现方式

1. DFS 的实现

DFS 通常可以通过递归或栈（Stack） 来实现：

  • 递归实现：利用函数调用栈的特性，递归访问每个未被访问的邻接节点。

  • 栈实现：手动维护一个栈，将待访问节点入栈，弹出节点并访问其未被访问的邻接节点，入栈顺序可控制遍历方向。

三、算法优缺点及适用场景

1. DFS 的优缺点与适用场景

  • 优点：

    • 内存占用相对较少（递归深度较小时）。
    • 适合解决 “路径存在性”“连通性”“拓扑排序” 等问题。

  • 缺点：

    • 可能陷入深层路径，无法找到最短路径（在无权图中）。
    • 递归实现可能因深度过大导致栈溢出。

  • 适用场景：

    • 迷宫求解（寻找一条可行路径）。
    • 拓扑排序（有向无环图）。
    • 连通分量查找。

2. BFS 的优缺点与适用场景

  • 优点：

    • 在无权图中，能找到从起始节点到目标节点的最短路径（边数最少）。
    • 按层次遍历，适合 “层次相关” 问题。

  • 缺点：

    • 内存占用较大（需要存储大量待访问节点）。
    • 不适合深度较大的图（可能导致队列过大）。

  • 适用场景：

    • 最短路径问题（无权图）。
    • 层次遍历（如二叉树的层序遍历）。
    • 社交网络中的 “一度好友”“二度好友” 查找。

四、算法对比总结

特性 DFS BFS
遍历方式 深度优先，回溯探索 广度优先，层次扩展
实现工具 递归 / 栈 队列
最短路径（无权图） 不保证 保证
内存占用 通常较少（取决于深度） 通常较多（取决于广度）
典型应用 拓扑排序、连通分量 最短路径、层次遍历

五、总结

DFS 和 BFS 是两种基础且重要的搜索算法，各有其适用场景。理解它们的核心思想、实现方式及优缺点，能帮助我们在解决实际问题时选择合适的算法。

在实际开发中，两种算法也常结合使用（如 “双向 BFS” 优化搜索效率）。掌握这两种算法，是深入学习更复杂算法（如 A * 搜索、回溯法）的基础。
