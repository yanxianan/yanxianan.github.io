🎯 目标

在 x86_64 主机 + 虚拟机（或直接在主机）环境下，为ARM架构（或其他架构，如 RISC-V）构建并运行一个完整的 Linux 系统（U-Boot + Kernel + RootFS），使用 QEMU 模拟目标硬件。

✅ 推荐方案：使用 QEMU + ARM 架构模拟

我们以 ARMv7 / ARM64（aarch64） 为例，这是嵌入式最常用的架构之一。

🐧 三、编译 Linux 内核

1. 下载内核源码

推荐启用：

  • CONFIG_BLK_DEV_INITRD=y （支持 initramfs）
  • CONFIG_DEVTMPFS=y 和 CONFIG_DEVTMPFS_MOUNT=y
  • CONFIG_SERIAL_AMBA_PL011=y 和 CONFIG_SERIAL_AMBA_PL011_CONSOLE=y（串口支持）

✅ Linux 内核裁剪与编译
✅ 设备树理解
✅ 根文件系统构建（BusyBox / init 脚本）

🚀 进阶建议

  1. 替换根文件系统为 Buildroot/Yocto 生成的完整系统
  2. 添加 NFS 根文件系统支持，实现网络挂载调试
  3. 使用 GDB + QEMU 调试内核或 U-Boot
  4. 编写自己的驱动并编译进内核
  5. 尝试 RISC-V 架构模拟（QEMU 也支持）

🧩 附：常见问题

Q：为什么不用 x86？

A：x86 不是嵌入式主流，且 U-Boot 在 x86 上使用较少（多用 GRUB/UEFI）。ARM 是嵌入式标准。

Q：能否模拟真实开发板，比如 Raspberry Pi？

A：QEMU 对树莓派支持有限。推荐使用官方 SDK 或购买开发板。但 virt 板足够学习系统构建。

Q：initramfs 启动后没有网络/USB？

A：需要在内核配置中启用对应驱动，或使用更完整的根文件系统（如 Buildroot）。
