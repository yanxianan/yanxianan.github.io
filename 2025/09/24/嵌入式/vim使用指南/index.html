Vim操作命令笔记：从入门到高效的必备指南

发表于 2025-09-24 分类于 嵌入式
本文记录一些vim编辑器的常用操作命令。

Vim 是 Linux/UNIX 环境下最经典的文本编辑器之一，核心优势是“模式化编辑”和“键盘流高效操作”——无需依赖鼠标，通过快捷键即可完成文本编辑、搜索替换、窗口管理等所有操作。对于嵌入式开发、后端开发等频繁编辑代码的场景，熟练使用 Vim 能大幅提升效率。本文按“基础→进阶→实用技巧”的逻辑，整理 Vim 核心操作命令，适合新手入门和老手查漏补缺。

一、Vim 核心：模式切换（新手必懂！）

Vim 与其他编辑器（如 VS Code、记事本）的最大区别是“多模式编辑”，不同模式下快捷键功能不同。核心模式有 4 种，切换逻辑是入门关键：

模式名称 核心作用 进入方式 退出方式（回正常模式）
正常模式（Normal） 光标移动、快捷键操作（核心模式） 打开 Vim 默认进入；其他模式按 ESC -
插入模式（Insert） 输入文本（类似普通编辑器） 正常模式下按 i/a/o/I/A/O 按 ESC
命令行模式（Command） 执行命令（如保存、退出、搜索） 正常模式下按 :（冒号） 执行命令后自动回正常模式；或按 ESC
可视模式（Visual） 选中文本（批量操作） 正常模式下按 v/V/<Ctrl+v> 按 ESC

🔔 关键提醒：所有快捷键都需在“正常模式”下触发，如果操作没反应，先按 ESC 确认回到正常模式！

二、基础操作：光标移动与文本编辑

1. 正常模式：光标移动（高效核心）

正常模式下，光标移动不依赖方向键，用字母键即可完成，速度更快。按“移动范围”分类如下：

快捷键 功能描述 示例场景
h/j/k/l 左/下/上/右移动（替代 ←↓↑→ 键） 按 j 向下移动一行
w 光标跳至下一个单词开头 从“hello world”的 h 跳至 w
b 光标跳至上一个单词开头 从“world”的 w 跳回 h
e 光标跳至当前单词结尾 从“hello”的 h 跳至 o
0 光标跳至当前行开头（数字 0） 行首有缩进时，跳至最左侧
^ 光标跳至当前行第一个非空白字符 跳过行首缩进，定位到代码开头
$ 光标跳至当前行结尾 快速编辑行尾注释
gg 光标跳至文件开头 打开大文件后定位到首行
G 光标跳至文件结尾 查看日志文件最后几行
nG 光标跳至第 n 行（n 为数字） 按 10G 跳至第 10 行
H 光标跳至当前屏幕顶部 快速回到屏幕可视范围顶部
M 光标跳至当前屏幕中间 屏幕内容多时定位到中间
L 光标跳至当前屏幕底部 快速查看屏幕底部内容
Ctrl+f 屏幕向下翻一页（forward） 类似 PageDown 键
Ctrl+b 屏幕向上翻一页（backward） 类似 PageUp 键
Ctrl+d 屏幕向下翻半页（down） 缓慢滚动查看内容
Ctrl+u 屏幕向上翻半页（up） 同上

2. 插入模式：文本输入（6种进入方式）

正常模式下按以下键进入插入模式，区别在于“光标位置”，根据场景选择更高效：

快捷键 进入插入模式的位置 适用场景
i 光标当前位置前插入 在单词中间补字符（如“heo”→补“l”）
a 光标当前位置后插入 在单词末尾加字符（如“hello”→加“!”）
o 在光标当前行下方新建一行插入 向下新增代码行
I 光标跳至当前行开头插入 在行首加注释（如“//”）
A 光标跳至当前行结尾插入 在行尾加分号（如 C/C++ 语句结尾）
O 在光标当前行上方新建一行插入 向上新增代码行

3. 正常模式：文本编辑（删除/复制/粘贴/撤销）

这部分是 Vim 最常用的编辑操作，所有删除的内容会自动存入“寄存器”，支持后续粘贴，无需担心误删。

（1）删除操作（d = delete）

快捷键 功能描述 示例
x 删除光标当前位置的字符 光标在“h”上，按 x 删除“h”
dw 删除光标从当前位置到下一个单词开头的内容 “hello world”中，光标在“h”→按 dw 删除“hello”
db 删除光标从当前位置到上一个单词开头的内容 “hello world”中，光标在“w”→按 db 删除“ world”
dd 删除光标当前行 快速删除一行代码
ndd 删除光标当前行开始的 n 行（n 为数字） 按 3dd 删除当前行及下方 2 行
d$ 删除光标从当前位置到行尾的内容 清除行尾多余字符
d^ 删除光标从当前位置到行首的内容 清除行首多余空格
dG 删除光标从当前行到文件结尾的内容 清空文件后半部分
dgg 删除光标从当前行到文件开头的内容 清空文件前半部分

（2）复制操作（y = yank，意为“ Yank 复制”）

快捷键 功能描述 示例
yw 复制光标从当前位置到下一个单词开头的内容 复制一个单词
yy 复制光标当前行 复制一行代码
nyy 复制光标当前行开始的 n 行 按 5yy 复制 5 行代码
y$ 复制光标从当前位置到行尾的内容 复制行尾部分内容
y^ 复制光标从当前位置到行首的内容 复制行首部分内容
yG 复制光标从当前行到文件结尾的内容 复制文件后半部分
ygg 复制光标从当前行到文件开头的内容 复制文件前半部分

（3）粘贴操作（p = put，意为“ Put 粘贴”）

快捷键 功能描述 示例
p 在光标当前位置后粘贴内容 复制一行后，按 p 粘贴到下一行
P 在光标当前位置前粘贴内容 复制一行后，按 P 粘贴到上一行
np 粘贴 n 次复制的内容（n 为数字） 按 3p 粘贴 3 次

（4）撤销与恢复（u = undo，Ctrl+r = redo）

快捷键 功能描述 示例
u 撤销上一步操作 误删一行后，按 u 恢复
Ctrl+r 恢复（反撤销）上一步被撤销的操作 撤销多了，按 Ctrl+r 找回
U 撤销当前行的所有修改 一行改乱了，按 U 恢复原样

三、进阶操作：搜索替换与窗口管理

1. 搜索操作（正常模式 + 命令行模式）

Vim 支持快速搜索文本，且能高亮匹配结果，适合大文件中找关键词。

（1）正常模式：快速搜索

快捷键 功能描述 示例
/关键词 从光标位置向下搜索“关键词” 按 /printf 搜索所有 printf 函数
?关键词 从光标位置向上搜索“关键词” 按 ?error 向上找 error 日志
n 跳至下一个匹配结果 搜索后按 n 继续找下一个
N 跳至上一个匹配结果 按 N 回到上一个匹配位置
* 快速搜索当前光标所在单词（向下） 光标在“hello”上，按 * 搜索所有 hello
# 快速搜索当前光标所在单词（向上） 同上，向上搜索

（2）命令行模式：搜索配置

命令 功能描述 示例
:set hlsearch 开启搜索结果高亮显示（默认开启） 按 :set hlsearch 启用
:set nohlsearch 关闭搜索结果高亮显示 按 :nohlsearch 临时关闭高亮
:set ignorecase 搜索时忽略大小写（如 Hello 和 hello 视为相同） 按 :set ic 简写启用
:set noignorecase 搜索时区分大小写 按 :set noic 简写关闭

2. 替换操作（命令行模式，:s 命令）

Vim 的替换功能支持“局部替换”和“全局替换”，语法灵活，核心格式为：
:范围 s/旧内容/新内容/选项

命令示例 功能描述 适用场景
:s/old/new 替换当前行第一个“old”为“new” 一行内改单个关键词
:s/old/new/g 替换当前行所有“old”为“new” 一行内改多个相同关键词
:#,#s/old/new/g 替换第 # 行到第 # 行所有“old”为“new” 按 :10,20s/printf/print/g 改 10-20 行的 printf 为 print
:%s/old/new/g 替换整个文件所有“old”为“new” 批量修改变量名（如把 foo 改为 bar）
:%s/old/new/gc 替换整个文件所有“old”为“new”，且每次替换前确认 避免误改，按 y 确认替换，n 跳过
:%s/^/\/\/ /g 在所有行首加“// ”（注释行） 批量注释 C/C++ 代码
:%s/^\/\/ //g 移除所有行首的“// ”（取消注释） 批量取消注释

🔔 说明：^ 代表行首，$ 代表行尾；选项 g 代表“全局（global）”，c 代表“确认（confirm）”。

3. 窗口管理（多文件编辑必备）

Vim 支持分屏显示多个文件，适合对比代码、同时编辑头文件和源文件，核心命令在命令行模式下执行：

命令 功能描述 操作示例
:sp 文件名 水平分屏（上下分屏），新窗口打开“文件名” 按 :sp main.c 水平分屏打开 main.c
:vsp 文件名 垂直分屏（左右分屏），新窗口打开“文件名” 按 :vsp test.h 垂直分屏打开 test.h
Ctrl+w+h/j/k/l 切换到左/下/上/右侧的窗口 按 Ctrl+w+j 切换到下方窗口
Ctrl+w+w 循环切换所有窗口 快速在多个分屏间切换
Ctrl+w+= 所有分屏窗口平均分配大小 分屏大小乱了，快速恢复平衡
Ctrl+w+- 当前窗口高度减小（水平分屏时） 缩小当前窗口高度
Ctrl+w++ 当前窗口高度增大（水平分屏时） 增大当前窗口高度
:q 关闭当前窗口（若只有一个窗口则退出Vim） 按 :q 关闭不需要的分屏
:qall 关闭所有窗口并退出Vim 结束编辑时用
:wall 保存所有窗口的文件 批量保存多个编辑的文件

四、实用技巧：提升效率的关键操作

1. 文件操作（命令行模式）

命令 功能描述 示例
:e 文件名 在当前窗口打开新文件（关闭当前文件） 按 :e config.h 打开 config.h
:new 新建一个空白文件并水平分屏显示 快速新建临时文件
:w 保存当前文件 编辑中随时保存，避免丢失
:w 新文件名 另存为“新文件名”（不覆盖原文件） 按 :w main_bak.c 备份 main.c
:wq 保存当前文件并退出Vim 编辑完成后常用
:q! 强制退出Vim，不保存修改 编辑错了，放弃修改退出
:ls 查看当前打开的所有文件（缓冲区列表） 按 :ls 查看已打开的文件编号
:b 编号/文件名 切换到“编号”或“文件名”对应的文件 按 :b 2 切换到第2个缓冲区的文件

2. 可视模式：批量操作（v/V/Ctrl+v）

可视模式用于“选中一块文本”后批量操作，比鼠标选中更精准，支持三种选中方式：

快捷键 可视模式类型 选中方式 适用场景
v 字符可视模式 按 v 后移动光标，选中单个字符 选中单词、短语
V 行可视模式 按 V 后移动光标，选中整行 批量删除/复制多行代码
Ctrl+v 块可视模式（列模式） 按 Ctrl+v 后移动光标，选中矩形块 批量添加/删除列注释

块可视模式示例：批量添加行注释（C/C++ 代码）

  1. 光标移到要注释的第一行开头；
  2. 按 Ctrl+v 进入块可视模式；
  3. 按 j 向下移动光标，选中所有要注释的行的“行首”；
  4. 按 I（大写 I）进入插入模式，输入 // （双斜杠+空格）；
  5. 按 ESC，所有选中的行首会自动添加 // ，完成批量注释。

3. 寄存器：灵活粘贴（进阶技巧）

Vim 的“寄存器”类似剪贴板，默认删除/复制的内容会存入“无名寄存器”（""），但也支持自定义寄存器（a-z），适合同时复制多段内容：

操作示例 功能描述 场景
"ayy 复制当前行到“a 寄存器” 第一个要复制的内容存 a 寄存器
"byw 复制当前单词到“b 寄存器” 第二个要复制的内容存 b 寄存器
"ap 粘贴“a 寄存器”的内容 粘贴第一个内容
"bp 粘贴“b 寄存器”的内容 粘贴第二个内容
"0p 粘贴最近一次复制的内容（不受删除操作影响） 避免删除操作覆盖复制的内容

4. 宏录制：重复执行操作（自动化）

Vim 宏（Macro）可录制一系列操作，后续通过快捷键重复执行，适合“重复相同编辑动作”（如格式化多行代码）：

宏录制步骤：

  1. 正常模式下按 q+字母（如 qa），开始录制宏（存入 a 寄存器），状态栏会显示 recording；
  2. 执行要重复的操作（如删除行尾空格、添加分号）；
  3. 按 q 结束录制；
  4. 按 @+字母（如 @a）执行宏；按 n@a（如 5@a）重复执行 5 次宏。

示例：给 10 行代码末尾加分号

  1. 光标移到第一行末尾；
  2. 按 qa 开始录制宏；
  3. 按 A;ESC（A 到行尾插入，加 ;，回正常模式）；
  4. 按 j 移动到下一行；
  5. 按 q 结束录制；
  6. 按 9@a 执行 9 次宏，完成剩余 9 行的操作。

5. Vim 配置：个性化设置（.vimrc 文件）

Vim 启动时会加载用户目录下的 .vimrc 文件（Linux 路径 ~/.vimrc，Windows 路径 C:\Users\用户名\_vimrc），在该文件中添加配置可自定义 Vim 行为，常用配置如下：

配置后保存 .vimrc，重启 Vim 即可生效；也可在 Vim 中按 :source ~/.vimrc 临时生效。

五、总结：Vim 学习建议

  1. 先练基础模式：新手初期重点掌握“正常模式→插入模式”切换、光标移动（h/j/k/l/w/b）、删除/复制/粘贴（dd/yy/p），这些是所有操作的基础；
  2. 拒绝鼠标依赖：刻意不用鼠标，强迫自己用快捷键，1-2 周后会形成肌肉记忆；
  3. 按需查命令：遇到复杂操作（如批量替换、分屏）时，对照本文或 :help 命令（如 :help :s 查看替换命令帮助）查找，逐步积累；
  4. 个性化配置：根据自己的开发语言（如 C/C++、Python）调整 .vimrc，提升长期使用体验。

Vim 的核心魅力是“用键盘流实现高效编辑”，熟练后能大幅减少手部在键盘和鼠标间的切换，尤其适合长时间代码开发。坚持练习，你会发现它比图形化编辑器更“顺手”！
