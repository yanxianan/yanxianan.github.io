Linux系统常用命令大全

发表于 2025-09-11 分类于 嵌入式
本文整理了Linux系统从基础到进阶的常用命令，涵盖文件操作、系统管理、网络配置等场景，每个命令均附带具体实例，适合新手快速上手。

Linux系统的强大之处在于其丰富的命令行工具，无论是日常操作、系统管理还是开发调试，命令行都能提供高效的解决方案。本文精选了Linux系统中最常用的命令，从基础的文件操作到复杂的系统监控，每个命令都配有实际场景的使用例子，帮助你快速掌握Linux命令行技能。

一、文件与目录操作命令

1. ls：列出目录内容

功能：查看当前目录或指定目录下的文件和子目录。

常用参数：

  • -l：以长格式显示（权限、所有者、大小、修改时间等）
  • -a：显示所有文件（包括隐藏文件，以.开头的文件）
  • -h：以人类可读的单位显示文件大小（如K、M、G）
  • -t：按修改时间排序（最新的在前）

实例：

2. cd：切换目录

功能：切换当前工作目录。

特殊路径：

  • ~：当前用户的家目录
  • .：当前目录
  • ..：上一级目录
  • -：上一次所在的目录

实例：

3. pwd：显示当前目录路径

功能：打印当前工作目录的绝对路径。

实例：

4. mkdir：创建目录

功能：创建新的目录。

常用参数：

  • -p：递归创建目录（若父目录不存在则自动创建）

实例：

5. rm：删除文件或目录

功能：删除指定的文件或目录（危险操作，谨慎使用）。

常用参数：

  • -f：强制删除（不提示确认）
  • -r：递归删除目录及其内容
  • -i：删除前提示确认

实例：

6. cp：复制文件或目录

功能：复制文件或目录到指定位置。

常用参数：

  • -i：若目标文件存在，提示是否覆盖
  • -r：递归复制目录及其内容
  • -v：显示复制过程

实例：

7. mv：移动或重命名文件/目录

功能：移动文件/目录到新位置，或重命名文件/目录。

实例：

8. find：查找文件或目录

功能：在指定目录下按名称、类型、大小等条件查找文件。

常用参数：

  • -name：按文件名查找（支持通配符*和?）
  • -type：按类型查找（f为文件，d为目录）
  • -size：按大小查找（+10M表示大于10M，-1k表示小于1k）
  • -mtime：按修改时间查找（-7表示7天内修改的）

实例：

9. grep：文本搜索工具

功能：在文件中搜索匹配指定模式的文本行。

常用参数：

  • -i：忽略大小写
  • -n：显示匹配行的行号
  • -r：递归搜索目录下的所有文件
  • -v：显示不匹配的行

实例：

二、文件内容操作命令

1. cat：查看文件内容

功能：连接并打印文件内容（适合查看小型文件）。

常用参数：

  • -n：显示行号
  • -b：只对非空行显示行号

实例：

2. less/more：分页查看文件内容

功能：分页显示大型文件内容（less功能更强大，支持上下滚动和搜索）。

常用操作（less中）：

  • 空格键：翻页
  • b：向上翻页
  • /关键词：向下搜索关键词
  • q：退出

实例：

3. head/tail：查看文件开头/结尾内容

功能：

  • head：显示文件开头的几行（默认10行）
  • tail：显示文件结尾的几行（默认10行）

常用参数：

  • -n：指定显示的行数
  • -f：实时跟踪文件新增内容（常用于监控日志）

实例：

4. wc：统计文件内容

功能：统计文件的行数、单词数、字节数。

常用参数：

  • -l：只统计行数
  • -w：只统计单词数
  • -c：只统计字节数

实例：

三、系统管理命令

1. top/htop：查看系统进程和资源占用

功能：实时显示系统进程的CPU、内存占用情况（htop是top的增强版，界面更友好）。

常用操作（top中）：

  • P：按CPU使用率排序
  • M：按内存使用率排序
  • k：终止指定进程（输入进程ID）
  • q：退出

实例：

2. ps：查看进程状态

功能：显示当前系统的进程快照（静态查看）。

常用参数：

  • aux：显示所有用户的所有进程（最常用组合）
  • -ef：显示进程的详细信息（包括父进程ID）

实例：

3. kill/pkill：终止进程

功能：向进程发送信号（默认是终止信号）。

常用信号：

  • 15（SIGTERM）：正常终止（默认）
  • 9（SIGKILL）：强制终止（无法被进程捕获）

实例：

4. df/du：查看磁盘空间

功能：

  • df：显示磁盘分区的使用情况
  • du：显示文件或目录的磁盘占用情况

常用参数：

  • -h：以人类可读的单位显示（K、M、G）
  • -s：du专用，显示总大小（不显示子目录详情）

实例：

5. free：查看内存使用情况

功能：显示系统内存（物理内存和交换分区）的使用情况。

常用参数：

  • -h：以人类可读的单位显示

实例：

四、用户与权限管理命令

1. sudo：以超级用户权限执行命令

功能：允许普通用户以root权限执行指定命令（需在sudoers文件中配置权限）。

实例：

2. chmod：修改文件权限

功能：改变文件或目录的访问权限（读r、写w、执行x）。

权限表示方式：

  • 符号表示：u（所有者）、g（所属组）、o（其他用户）；+（添加）、-（移除）、=（设置）
  • 数字表示：r=4、w=2、x=1，三者之和表示权限（如755表示rwxr-xr-x）

实例：

3. chown：修改文件所有者

功能：改变文件或目录的所有者和所属组。

实例：

五、网络相关命令

1. ping：测试网络连接

功能：向目标主机发送ICMP请求，检查网络连通性。

常用参数：

  • -c：指定发送包的数量
  • -i：指定发送间隔（秒）

实例：

2. ifconfig/ip：查看网络接口信息

功能：显示或配置网络接口（ip是新一代工具，逐步替代ifconfig）。

实例：

3. netstat/ss：查看网络连接

功能：显示网络连接、路由表、端口监听等信息（ss是netstat的替代工具，性能更好）。

实例：

4. curl/wget：下载文件或测试HTTP请求

功能：

  • curl：发送HTTP请求，支持多种协议，适合测试API
  • wget：下载文件，支持断点续传

实例：

六、压缩与解压命令

1. tar：打包与压缩文件

功能：打包多个文件为一个归档文件，可结合gzip或bzip2进行压缩。

常用参数：

  • -c：创建归档
  • -x：解压归档
  • -v：显示过程
  • -f：指定归档文件名
  • -z：用gzip压缩/解压（.tar.gz）
  • -j：用bzip2压缩/解压（.tar.bz2）

实例：

2. zip/unzip：ZIP格式压缩/解压

功能：处理ZIP格式的压缩文件（Windows系统常用格式，跨平台兼容性强，适合与Windows设备交互或压缩单个文件）。

常用参数：

  • -r：递归压缩目录及其子目录内容（压缩目录时必须添加）
  • -q：静默压缩/解压（不显示过程信息）
  • -l：查看压缩包内文件列表（不解压）
  • -d：指定解压目标目录

实例：

七、其他实用命令（嵌入式开发高频场景补充）

1. history：查看与复用命令历史

功能：显示当前用户执行过的命令历史，支持快速复用历史命令（嵌入式开发中频繁执行编译、烧录命令时尤其高效）。

常用参数：

  • 直接输入history：显示所有历史命令（默认保存1000条，可通过HISTSIZE环境变量修改）
  • history N：显示最近N条命令

实例：

2. man/--help：获取命令帮助

功能：man是Linux系统自带的“离线手册”，提供命令的详细参数说明和使用场景；--help则是快速查看命令核心参数（适合临时记忆模糊时快速查阅）。

实例：

3. sudo进阶：权限与执行控制

功能：除了基础的“提权执行”，sudo还支持限制用户执行特定命令、保留环境变量等（嵌入式开发中多用户协作或避免误操作时常用）。

实例：

4. scp：跨设备文件传输（嵌入式开发必备）

功能：基于SSH协议在本地主机与远程设备（如开发板、服务器）之间传输文件（嵌入式开发中常用于传输内核镜像、根文件系统）。

常用参数：

  • -r：递归传输目录
  • -P：指定远程设备的SSH端口（默认22，若开发板修改过端口需指定）
  • -q：静默传输（不显示进度）

实例：

5. watch：实时监控命令输出

功能：周期性执行指定命令并显示输出，适合实时监控文件变化、系统资源或编译进度（嵌入式开发中监控日志、磁盘占用时常用）。

总结

本文整理的命令覆盖了Linux系统基础操作与嵌入式开发高频场景，从文件管理、系统监控到跨设备交互，每个命令均结合嵌入式开发实际需求补充了实例。对于嵌入式工程师而言，命令行的核心价值在于“高效自动化”——通过熟练组合命令、编写简单脚本（如编译烧录一键脚本），可大幅减少重复操作，提升开发效率。

建议在实际开发中逐步积累“命令组合模板”（如日志筛选、文件传输），遇到陌生命令时优先通过man或--help查阅官方说明，避免因参数错误导致数据丢失或设备异常（尤其是rm -rf、dd等高危命令）。
