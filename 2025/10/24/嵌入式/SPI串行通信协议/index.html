SPI串行通信协议

发表于 2025-10-24 分类于 嵌入式
本文介绍SPI通信相关知识。

当然可以！SPI（Serial Peripheral Interface，串行外设接口）是一种高速、全双工、同步、主从式的串行通信协议，由 Motorola 在 1980 年代提出，广泛用于微控制器与各类外设（如 Flash、ADC、DAC、LCD、传感器、RF 模块等）之间的短距离通信。

一、SPI 的核心特点

特性 说明
通信方式 同步（由主设备提供时钟）
数据流向 全双工（主从可同时收发）
速度 高速（可达几十 Mbps，甚至上百 Mbps）
线数 通常 4 线（可简化为 3 线或 2 线）
拓扑结构 主从架构（1 主 + N 从）
驱动方式 推挽输出（信号边沿陡峭，抗干扰强）
协议复杂度 无地址、无应答、无校验（简单高效）

二、SPI 的信号线（4 线制标准）

信号线 全称 方向 作用
SCLK Serial Clock 主 → 从 时钟信号，由主设备产生，同步数据传输
MOSI Master Out Slave In 主 → 从 主设备发送，从设备接收的数据线
MISO Master In Slave Out 从 → 主 从设备发送，主设备接收的数据线
CS/SS Chip Select / Slave Select 主 → 从 片选信号，低电平有效，用于选择特定从设备

📌 注意：每个从设备需要独立的 CS 信号线，因此从设备越多，主设备占用的 GPIO 越多。

三、SPI 的工作原理

1. 主从架构

  • 主设备（Master）：通常是 MCU，负责产生 SCLK 和 CS 信号，发起通信。
  • 从设备（Slave）：如传感器、Flash 等，被动响应主设备。

2. 通信流程

  1. 主设备将目标从设备的 CS 拉低（激活从设备）。
  2. 主设备开始发送 SCLK 时钟。
  3. 在每个时钟边沿：
    • 主设备通过 MOSI 发送 1 位数据；
    • 从设备通过 MISO 同时发送 1 位数据（全双工）。
  4. 通常传输 8 位（1 字节）为单位，可连续发送多字节。
  5. 通信结束后，主设备将 CS 拉高，结束本次传输。

✅ 优势：无需应答、无地址开销，效率极高。

四、SPI 的时序模式：CPOL 与 CPHA

SPI 有 4 种工作模式，由两个参数决定：

参数 含义
CPOL（Clock Polarity） 时钟空闲时的电平： • CPOL=0：空闲时 SCLK=0（低电平） • CPOL=1：空闲时 SCLK=1（高电平）
CPHA（Clock Phase） 数据采样时机： • CPHA=0：在第一个时钟边沿采样 • CPHA=1：在第二个时钟边沿采样

四种模式组合：

模式 CPOL CPHA 时钟空闲电平 采样边沿
Mode 0 0 0 低 上升沿（0→1）
Mode 1 0 1 低 下降沿（1→0）
Mode 2 1 0 高 下降沿（1→0）
Mode 3 1 1 高 上升沿（0→1）

⚠️ 主从设备必须使用相同的模式，否则通信失败！

五、SPI 的接线方式

1. 标准 1 主 N 从（独立片选）

  • 优点：简单可靠，从设备互不影响。
  • 缺点：每增加一个从设备，需多一个 CS 引脚。

2. 菊花链（Daisy Chain）

  • 所有从设备共用一个 CS，MISO 串接：
    Master → MOSI → Slave1 → Slave2 → … → MISO → Master
  • 适用于级联移位寄存器等场景（如 LED 驱动）。
  • 缺点：不能单独访问某个从设备，延迟大。

六、SPI 的优缺点

✅ 优点：

  • 高速：无协议开销，可达系统时钟的 1/2（如 STM32 可达 50+ Mbps）。
  • 全双工：收发同时进行，效率高。
  • 硬件实现简单：多数 MCU 内置 SPI 控制器。
  • 信号质量好：推挽输出，边沿陡峭，抗干扰强。
  • 灵活的数据长度：支持 4~16 位（甚至更多）每帧。

❌ 缺点：

  • 引脚占用多：每个从设备需独立 CS。
  • 无流控/应答机制：无法确认从设备是否收到（需软件实现）。
  • 无标准协议层：不同厂商设备命令格式各异，需查手册。
  • 不支持多主：只能有一个主设备（除非用额外逻辑仲裁）。
  • 距离短：一般用于板内通信（< 30cm），高速时更短。

七、SPI vs I²C vs UART（简要对比）

特性 SPI I²C UART
线数 3~4（+CS） 2 2（TX/RX）
速度 高（Mbps~100+ Mbps） 低（100kbps~3.4Mbps） 中（9600~10Mbps）
全/半双工 全双工 半双工 全双工（异步）
主从结构 1 主多从 多主多从 点对点
地址机制 片选（硬件） 软件地址 无（点对点）
抗干扰 强（推挽） 弱（开漏） 中（依赖电平标准）
距离 短（板内） 短 可较长（加驱动）

八、典型应用场景

  • Flash 存储器：W25Q128（SPI Flash）
  • 显示屏：OLED（SSD1306）、TFT LCD
  • ADC/DAC：MCP3008（ADC）、MCP4921（DAC）
  • 传感器：MPU6050（陀螺仪）、BMP280（气压）
  • RF 模块：nRF24L01、LoRa（SX1278）
  • 音频编解码器：PCM5102

九、使用注意事项

  1. 模式匹配：务必确认主从设备的 CPOL/CPHA 一致。
  2. CS 时序：有些设备要求 CS 在字节间不能释放（如 Flash 写命令）。
  3. 电源与地：高速 SPI 对电源噪声敏感，需良好去耦。
  4. 走线等长：MOSI/MISO/SCLK 尽量等长，减少 skew。
  5. 上拉/下拉：CS 通常需上拉电阻，防止上电时误触发。

十、扩展：QSPI / Dual/Quad SPI

为提升速度，许多 Flash 支持 Dual SPI（2 位并行） 或 Quad SPI（4 位并行），使用额外的 IO 引脚同时传输多位数据，速率提升 2~4 倍。STM32 等 MCU 也支持 QSPI 接口，用于高速访问外部 Flash。
