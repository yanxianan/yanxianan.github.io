C++ 中的智能指针

发表于 2025-10-27 分类于 C_C++
本文详细介绍智能指针（Smart Pointer）。

智能指针（Smart Pointer）是 C++ 中用于自动管理动态分配内存的工具，它们封装了原始指针，并在其生命周期结束时自动释放所管理的资源，从而避免内存泄漏、悬空指针等问题。C++11 引入了标准库中的三种主要智能指针：std::unique_ptr、std::shared_ptr 和 std::weak_ptr。下面将逐一详细讲解其原理、特点及用法示例。

一、std::unique_ptr：独占式智能指针

特点：

  • 独占所有权：一个对象只能被一个 unique_ptr 拥有。
  • 不可复制（copy），但可移动（move）。
  • 轻量级，无额外开销（除指针本身外）。
  • 析构时自动调用 delete（或自定义删除器）。

二、std::shared_ptr：共享式智能指针

特点：

  • 共享所有权：多个 shared_ptr 可指向同一对象。
  • 使用引用计数机制：当最后一个 shared_ptr 被销毁时，对象才被释放。
  • 可复制、可赋值。
  • 有轻微性能开销（引用计数的原子操作）。

三、std::weak_ptr：弱引用智能指针（解决循环引用）

特点：

  • 不增加引用计数，仅观察 shared_ptr 管理的对象。
  • 不能直接访问对象，必须通过 lock() 转换为 shared_ptr。
  • 主要用于打破 shared_ptr 之间的循环引用。

四、何时使用哪种智能指针？

场景 推荐智能指针
独占资源，不需要共享 std::unique_ptr
多个所有者共享资源 std::shared_ptr
需要观察资源但不拥有（如缓存、观察者） std::weak_ptr
避免循环引用 std::weak_ptr + std::shared_ptr

五、最佳实践

  1. 优先使用 std::make_unique 和 std::make_shared
    它们更安全（异常安全）、更高效（make_shared 可能减少一次内存分配）。

  2. 避免混合使用原始指针和智能指针
    不要将同一个原始指针交给多个智能指针管理。

  3. 不要对同一对象创建多个独立的 shared_ptr

  4. 慎用 get()
    get() 返回原始指针，但不转移所有权，容易误用。

六、总结

智能指针是现代 C++ 内存管理的核心工具：

  • unique_ptr：高效、独占，适合大多数场景。
  • shared_ptr：灵活共享，但注意性能和循环引用。
  • weak_ptr：辅助 shared_ptr，解决循环引用问题。

合理使用智能指针，可以显著提升代码的安全性、可维护性和健壮性。
