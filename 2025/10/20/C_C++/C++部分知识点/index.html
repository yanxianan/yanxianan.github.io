C++中的命名以及引用

发表于 2025-10-20 分类于 C_C++
本文详细介绍 C ++中的命名以及引用。

本文讲解 驼峰命名法、左值引用 和 右值引用，这些都是 C++（以及许多其他编程语言）中的重要概念。

一、驼峰命名法（Camel Case）

✅ 定义：

驼峰命名法是一种变量、函数或类名的命名风格，特点是单词首字母大写（除第一个单词外），整体连在一起，形似驼峰，故得名。

📌 两种常见形式：

类型 名称 示例 说明
小驼峰（lower camelCase） 第一个单词小写，后续单词首字母大写 findContentChildren、userName 常用于函数名、变量名（C++/Java/JavaScript 等）
大驼峰（Upper CamelCase / PascalCase） 所有单词首字母都大写 MyClass、HttpRequest 常用于类名、结构体名

💡 为什么叫“驼峰”？

因为大写字母像“驼峰”一样凸起：
findContentChildren → find（低） + Content（高） + Children（高）

✅ C++ 中的惯例：

  • 函数名、变量名：小驼峰（如 maxValue, getUserInput）
  • 类名、结构体名：大驼峰（如 StudentRecord, BinaryTree）

📝 注意：C++ 标准库（如 STL）多用下划线命名法（如 push_back, vector_size），但用户代码常用驼峰。

二、左值引用（Lvalue Reference）

✅ 语法：

🔍 什么是“左值”（Lvalue）？

  • 左值：有内存地址、可以出现在赋值表达式左边的值。
  • 通常是具名变量（有名字、可取地址）。

✅ 左值引用的特点：

  • 是已有对象的别名。
  • 必须在定义时初始化。
  • 修改引用 = 修改原对象。

三、右值引用（Rvalue Reference）—— C++11 新特性

✅ 语法：

🔍 什么是“右值”（Rvalue）？

  • 右值：临时对象或字面量，没有持久内存地址，通常出现在表达式右边。
  • 例如：42、func() 返回的临时对象、a + b 的结果。

✅ 右值引用的作用：

  1. 实现移动语义（Move Semantics）：避免不必要的深拷贝，提升性能。
  2. 实现完美转发（Perfect Forwarding）。

四、对比总结

特性 左值引用 (T&) 右值引用 (T&&)
绑定对象 左值（具名变量） 右值（临时对象）
是否可修改原对象 是 是（但对象即将销毁）
典型用途 避免拷贝传参、返回引用 移动语义、完美转发
示例 int &r = x; int &&r = 42; 或 std::move(x)
C++ 版本 C++98 C++11 起

五、实际应用建议

  • 函数参数传递大对象 → 用 const T&（避免拷贝，只读）
  • 需要转移资源（如实现移动构造函数）→ 用 T&&
  • 通用引用（模板中） → T&& 可能是左值或右值（依赖类型推导）
