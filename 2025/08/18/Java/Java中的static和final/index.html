Java中的static和final

发表于 2025-08-18 更新于 2025-09-01 分类于 Java
在 Java 中，`static` 和 `final` 是两个非常重要的关键字，它们可以用来修饰变量，但它们的含义和作用完全不同。当它们单独使用或组合使用时，会产生不同的效果。

Java中的static和final关键字

1. static 关键字

含义：static 表示“静态的”或“属于类的”，而不是属于某个具体的对象实例。

修饰变量时的特点：

  • 类变量：被 static 修饰的变量称为类变量或静态变量。它属于类本身，而不是类的任何特定实例。
  • 共享性：无论创建多少个该类的对象，static 变量在内存中只有一份拷贝。所有实例共享这个变量。
  • 生命周期：static 变量在类加载时被初始化，并且在程序的整个生命周期内存在，直到类被卸载。
  • 访问方式：可以通过 类名.变量名 直接访问，也可以通过 对象实例.变量名 访问（但不推荐，容易引起误解）。

2. final 关键字

含义：final 表示“最终的”或“不可改变的”。

修饰变量时的特点：

  • 不可变性：一旦被 final 修饰的变量被赋值（初始化），它的值就不能再被修改。
  • 赋值时机：
    • final 成员变量：必须在声明时初始化，或者在构造器中初始化。确保在对象构造完成前，final 变量已被赋值。
    • final 局部变量：可以在声明时初始化，也可以在后续的代码中赋值，但只能赋值一次。
  • 引用类型：如果 final 修饰的是一个对象引用，那么这个引用本身不能改变（即不能指向另一个对象），但对象内部的状态（其成员变量）是可以改变的。

3. static final 组合使用

当 static 和 final 同时修饰一个变量时，它表示一个全局常量。

  • static：表示这个变量属于类，所有实例共享。
  • final：表示这个变量的值不可改变。

因此，static final 变量通常用来定义常量，并且命名习惯上使用全大写字母和下划线。

特点：

  • 在类加载时初始化（对于基本类型和字符串字面量，通常在编译期就能确定，称为“编译期常量”）。
  • 值不可变。
  • 通过 类名.常量名 访问。

对比总结

特性 static final static final
所属 属于类，不依赖实例 属于变量本身，可以是实例变量或类变量 属于类，是类级别的常量
可变性 值可以改变（除非也用 final 修饰） 值不可改变 值不可改变
实例共享 是，所有实例共享一份 否，每个实例有自己的 final 实例变量（但值固定） 是，所有实例共享同一个常量值
生命周期 类加载时创建，程序结束时销毁 实例变量：对象创建时初始化，对象销毁时结束
局部变量：方法执行期间		 类加载时初始化，程序结束时销毁
典型用途 计数器、共享资源 防止变量被意外修改、定义实例常量 定义全局常量（如配置、数学常数）

简单记忆：

  • static = 共享 (One copy for all instances)
  • final = 不变 (Value cannot be changed)
  • static final = 共享且不变 (A constant shared by all instances)
