查看本机硬件 CPU 架构/指令集：

uname -m 是 “查看本机硬件 CPU 架构/指令集” 的最简命令。
它把内核检测到的 机器硬件名称 原样打印出来，不做任何转换。
常见返回值一览（按出现频率排序）

返回值示例 含义 常见场景
x86_64 / amd64 64 位 Intel/AMD PC、服务器、笔记本、虚拟机 绝大多数桌面/服务器 Linux
aarch64 64 位 ARMv8-A（AArch64） 树莓派 3/4/5、RK3399、苹果 M 系列、安卓手机刷 Linux
armv7l 32 位 ARMv7（Little-endian） 树莓派 2 早期版、某些嵌入式板、老手机
armv6l 32 位 ARMv6 树莓派 1/Zero、旧款路由器
i686 / i386 32 位 x86 老旧 PC、极小容器镜像
mips / mips64 MIPS32 / MIPS64 路由器、龙芯
ppc64le 64 位 PowerPC Little-endian IBM POWER 服务器、OpenPOWER
s390x IBM z/Architecture 大型机、LinuxONE
riscv64 64 位 RISC-V 新兴开发板、SiFive

────────────────
与 uname -a 的区别

  • -m 只看“硬件架构”；
  • -a 会把内核名称、节点名、内核版本、架构等全部打出来，内容更长。
  • uname -m 返回的就是 “CPU 身份证”；看到 x86_64 就是 PC，aarch64 就是 ARM64 板。

只想运行单个可执行文件可以使用QEMU 的user-mode，如果交叉编译的程序依赖很多系统库，就需要启动一个虚拟机镜像运行。

常用的 QEMU user-mode 运行器：

目标架构 QEMU 命令
x86_64 qemu-x86_64
arm qemu-arm
aarch64 qemu-aarch64
riscv64 qemu-riscv64
mips qemu-mips

这种方式只模拟用户空间，不需要虚拟机系统，速度比系统仿真快。
