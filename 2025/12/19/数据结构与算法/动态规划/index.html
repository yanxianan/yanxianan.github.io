动态规划（Dynamic Programming, 简称 DP）

🧠 一、什么是动态规划？

动态规划 是一种用于求解具有重叠子问题和最优子结构性质的问题的算法设计方法。它通过将原问题分解为若干子问题，先求解子问题，再利用子问题的解构造原问题的解，从而避免重复计算，提高效率。

💡 动态规划 ≠ 递归，但常借助递归（带记忆化）或迭代实现。
它的本质是 “用空间换时间” —— 存储中间结果，避免重复计算。

🔑 二、动态规划的两个核心性质

1. 最优子结构（Optimal Substructure）

原问题的最优解包含子问题的最优解。

例如：

  • 最短路径问题中，A 到 C 的最短路径如果经过 B，那么 A→B 和 B→C 也一定是最短的。
  • 股票买卖最大利润：第 i 天的最大利润依赖于前 i-1 天的状态。

2. 重叠子问题（Overlapping Subproblems）

在递归求解过程中，某些子问题会被反复计算多次。

例如：
斐波那契数列 f(n) = f(n-1) + f(n-2) 中，f(3) 会被 f(5)、f(4) 等多次调用。

✅ 动态规划通过记忆化（缓存子问题结果）来消除重复计算。

🛠️ 三、动态规划的解题步骤（四步法）

  1. 定义状态（State）
    明确 dp[i] 或 dp[i][j] 表示什么含义（这是最关键的一步！）。
  2. 写出状态转移方程（Recurrence Relation）
    找出当前状态如何由之前的状态推导而来。
  3. 确定初始条件（Base Case）
    设置边界值，如 dp[0] = 0、dp[1] = 1 等。
  4. 确定遍历顺序 & 返回结果
    是从前往后？还是从后往前？最终答案是 dp[n] 还是 max(dp)？

📚 四、常见 DP 类型与经典例题

类型 特点 经典题目
线性 DP 状态只与前几个状态有关 斐波那契、爬楼梯、打家劫舍、股票买卖
区间 DP 状态由区间 [i, j] 定义 最长回文子串、石子合并
背包 DP 有限资源下的最优选择 0-1 背包、完全背包、分割等和子集
树形 DP 在树结构上进行状态转移 树的最大独立集、二叉树打家劫舍
状态机 DP 用多个状态表示不同行为 股票买卖（含冷冻期、手续费）
数位 DP 按数字位进行状态设计 不含连续 1 的数字个数

⚠️ 六、DP vs 贪心 vs 分治

方法 是否保证最优 是否有重叠子问题 典型应用
分治 是（如果子问题独立） ❌ 无重叠 归并排序、快速排序
贪心 ❌ 仅局部最优 ❌ 活动选择、霍夫曼编码
动态规划 ✅ 全局最优 ✅ 有重叠 背包、LCS、最短路径

贪心是 DP 的特例——当局部最优能推出全局最优时，可用贪心（如股票 I）。

💡 七、如何判断一道题是否用 DP？

问自己三个问题：

  1. 问题是否可以分解为子问题？
  2. 子问题的最优解是否能组合成原问题的最优解？（最优子结构）
  3. 子问题是否被重复计算？（重叠子问题）

如果都回答“是”，那就考虑 DP！

✅ 总结

  • 动态规划 = 状态定义 + 状态转移 + 初始条件 + 遍历顺序
  • 核心思想：记住已经算过的，避免重复劳动
  • 常见场景：最优化问题（最大/最小/最多/最少）、计数问题、存在性问题
  • 刷题建议：从简单线性 DP 开始（如爬楼梯、打家劫舍），再进阶到背包、区间、状态机等
