extern关键字

发表于 2025-12-30 分类于 C_C++
本文字数： 1.6k 阅读时长 ≈ 2 分钟
本文详细介绍extern关键字。

extern 是 C 和 C++ 语言中的一个关键字，用于声明变量或函数具有**外部链接（external linkage）**属性，即该变量或函数的定义在其他文件（或当前文件的其他位置）中。它的主要作用是告诉编译器：“这个符号（变量或函数）是在别处定义的，请不要在此分配存储空间，只需引用它即可。”

一、基本用途

1. 声明外部变量

当你在一个源文件中定义了一个全局变量，并希望在另一个源文件中使用它时，就需要用 extern 来声明。

示例：

file1.c

file2.c

注意：extern 只是声明，不是定义。不能用 extern 初始化变量（除非在 C++ 中有特殊规则，见下文）。

2. 声明外部函数（通常可省略）

函数默认具有外部链接，所以一般不需要显式加 extern，但可以加上以增强可读性。

二、在 C++ 中的扩展用法

1. extern "C" —— 兼容 C 语言链接

C++ 支持函数重载，因此编译器会对函数名进行 名称修饰（name mangling）。而 C 语言不会。为了在 C++ 中调用 C 编写的函数（如标准库函数），需要用 extern "C" 告诉编译器按 C 的方式链接。

示例：

这样，c_function 就不会被 C++ 编译器进行名称修饰，能正确链接到 C 编译的目标文件。

2. extern 与 const 变量（C++ 特有）

在 C++ 中，const 全局变量默认具有内部链接（即只在本文件可见）。如果想让它在多个文件间共享，必须显式加上 extern。

示例：

constants.cpp

main.cpp

如果没有 extern，每个文件包含 const int MAX_SIZE = 100; 都会创建自己的副本，无法共享。

三、常见误区

误区 正确理解
extern 会分配内存 ❌ extern 只是声明，不分配存储空间
所有全局变量都要加 extern ❌ 只有在其他文件中使用时才需要在使用文件中加 extern 声明
extern 可以初始化变量 ❌ 一般不行（C 中绝对不行；C++ 中作为定义时可以，但此时它就不再是“纯声明”了）

在 C++ 中，extern int x = 5; 实际上是定义，不是声明。这种写法等价于 int x = 5;，但强调其具有外部链接。

四、总结

  • extern 用于声明变量或函数在别处定义。
  • 主要用于跨文件共享全局变量。
  • 函数通常不需要 extern（默认外部链接）。
  • C++ 中用 extern "C" 实现 C 兼容链接。
  • C++ 中 const 全局变量需 extern 才能跨文件共享。
