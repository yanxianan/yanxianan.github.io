CPSR和SPSR

发表于 2025-12-12 分类于 汇编
本文介绍CPSR、SPSR。

在 ARM 架构的汇编语言中，MSR、MRS、CPSR 和 SPSR 是与处理器状态寄存器密切相关的指令和寄存器。它们主要用于控制和读取处理器的工作模式、中断使能状态、条件标志等关键信息。

下面分别详细讲解：

1. CPSR（Current Program Status Register）

当前程序状态寄存器（CPSR） 是 ARM 处理器中的一个 32 位寄存器，它保存了当前处理器的状态信息，包括：

  • 条件标志位（N, Z, C, V）：
    • N（Negative）：结果为负
    • Z（Zero）：结果为零
    • C（Carry）：进位/借位
    • V（Overflow）：溢出
  • 中断禁止位：
    • I：禁止 IRQ 中断（1 = 禁止）
    • F：禁止 FIQ 中断（1 = 禁止）
  • 处理器工作模式（Mode bits [4:0]）：
    • 如用户模式（User）、系统模式（System）、IRQ 模式、FIQ 模式、管理模式（Supervisor）等
  • 其他控制位：如 Thumb 状态位（T）、Jazelle 位（J）等（视具体架构而定）

注意：并非所有模式下都可以修改 CPSR 的全部字段。例如，在用户模式下不能修改模式位或中断禁止位。

2. SPSR（Saved Program Status Register）

保存的程序状态寄存器（SPSR） 存在于除用户模式和系统模式之外的所有特权模式中（如 IRQ、FIQ、Supervisor、Abort、Undefined）。

  • 当处理器从一个模式切换到另一个异常模式时（如发生中断），当前的 CPSR 会被自动保存到新进入模式的 SPSR 中。
  • 异常处理完成后，可以通过将 SPSR 恢复到 CPSR 来返回原来的执行状态。

举例：当发生 IRQ 中断时，处理器进入 IRQ 模式，并将原来的 CPSR 值保存到 SPSR_irq 中。

典型应用场景

1. 关闭/开启中断

2. 切换处理器模式（需在特权模式下）

3. 异常返回（恢复 CPSR）

总结对比

名称 类型 作用
CPSR 寄存器 当前程序状态寄存器，反映处理器当前状态
SPSR 寄存器 异常发生时保存 CPSR 的副本，每个异常模式有自己的 SPSR
MRS 指令 从 CPSR/SPSR 读取到通用寄存器
MSR 指令 从通用寄存器/立即数写入 CPSR/SPSR

⚠️ 注意：在 ARMv8（AArch64）架构中，CPSR 被拆分为多个系统寄存器（如 PSTATE、DAIF、NZCV 等），MRS/MSR 仍然存在但语义不同。上述内容主要适用于 ARMv7 及更早的 AArch32 模式。

如果你使用的是 Cortex-M 系列（如 Cortex-M3/M4），则没有 CPSR/SPSR 的完整概念，而是使用 xPSR（Program Status Register）并通过类似方式访问。
