内部中断和外部中断

发表于 2025-11-03 分类于 嵌入式
本文字数： 1.5k 阅读时长 ≈ 2 分钟
本文详细介绍内部中断和外部中断。

在嵌入式系统中，中断（Interrupt） 是一种重要的机制，用于处理异步事件。它允许处理器暂停当前任务，转而去处理更紧急或更高优先级的事件，处理完毕后再返回原来的任务继续执行。根据中断源的不同，可以将中断分为 内部中断（Internal Interrupt） 和 外部中断（External Interrupt）。

一、内部中断（Internal Interrupt）

定义：

内部中断是由 CPU 内部产生的中断请求，通常由以下原因触发：

  • 异常（Exception）：如除零错误、非法指令、地址越界等。
  • 软件中断（Software Interrupt）：通过特定指令（如 ARM 的 SWI 指令，x86 的 INT 指令）主动触发，常用于系统调用（System Call）。
  • 定时器中断（部分归为内部）：如果定时器集成在 CPU 内部（如 Cortex-M 系列的 SysTick），其产生的中断有时也被视为内部中断。

特点：

  • 中断源来自 CPU 内部逻辑。
  • 通常与程序执行状态直接相关（如出错、调试、特权切换）。
  • 响应速度快，因为不需要经过外部引脚或总线。
  • 不可屏蔽（某些关键异常）或可通过软件控制是否响应。

示例：

  • 在 ARM Cortex-M 中，HardFault、MemManage、BusFault 都是内部异常。
  • 使用 svc（Supervisor Call）指令发起系统调用，触发 SVC 异常。

二、外部中断（External Interrupt）

定义：

外部中断是由 CPU 外部硬件设备 通过专用中断引脚（如 IRQ、NMI）或中断控制器（如 NVIC、GIC）向 CPU 发出的中断请求。

常见来源包括：

  • 按键按下（GPIO 触发）
  • 串口接收数据完成
  • ADC 转换结束
  • 外部传感器信号变化
  • 网络控制器收到数据包

特点：

  • 中断源来自芯片外部或片内外设（如 UART、SPI、GPIO 等）。
  • 通常通过 边沿触发（上升/下降沿） 或 电平触发（高/低电平） 方式激活。
  • 可配置优先级（尤其在支持 NVIC 的 MCU 中，如 STM32）。
  • 可被屏蔽（通过中断使能/禁止寄存器）。

示例：

  • STM32 的 EXTI（External Interrupt/Event Controller）模块可将 GPIO 引脚配置为外部中断源。
  • 当按键按下时，GPIO 电平变化触发 EXTI0 中断，CPU 跳转到对应的 ISR（中断服务例程）处理。

三、对比总结

特性 内部中断 外部中断
中断源 CPU 内部（异常、指令、定时器等） CPU 外部（外设、传感器、按键等）
触发方式 程序执行异常或软件指令 硬件信号（电平/边沿）
是否可屏蔽 部分不可屏蔽（如 NMI、HardFault） 通常可屏蔽
典型用途 错误处理、系统调用、调试 实时响应外部事件
响应延迟 极低 略高（取决于引脚和电路）
是否需要配置引脚 否 是（如配置 GPIO 为中断输入）

四、实际应用中的协同

在嵌入式开发中，内部中断和外部中断常常协同工作。例如：

  • 用户按下按键（外部中断） → 触发 ISR 设置标志位 → 主循环或 RTOS 任务读取标志并调用系统服务（可能涉及软件中断/SVC）。
  • 外设 DMA 传输完成（外部中断） → 触发内部调度器进行任务切换（可能涉及 PendSV 异常，属于内部中断）。

五、典型架构示例（以 ARM Cortex-M 为例）

Cortex-M 系列使用 NVIC（Nested Vectored Interrupt Controller） 统一管理中断，包括：

  • 系统异常（内部）：如 Reset、NMI、HardFault、SVC、PendSV、SysTick。
  • 外部中断（IRQ0~IRQ240）：对应各种外设中断，如 EXTI、UART、TIMER 等。

所有中断都具有可编程优先级，并支持嵌套中断（高优先级可打断低优先级）。
