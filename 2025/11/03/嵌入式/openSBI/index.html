RISC-V的OpenSBI

发表于 2025-11-03 分类于 嵌入式
本文字数： 2.8k 阅读时长 ≈ 4 分钟
本文介绍openSBI。

OpenSBI（Open Source Supervisor Binary Interface）是 RISC-V 架构中一个开源的、标准化的固件（firmware）实现，用于在 Machine 模式（M-mode） 下为上层软件（如操作系统、Hypervisor、Bootloader）提供统一的 SBI（Supervisor Binary Interface） 服务。

它是 RISC-V 软件栈中承上启下的关键组件，相当于 RISC-V 的“BIOS + 中断/电源管理服务层”。

一、为什么需要 OpenSBI？

RISC-V 的特权模式分层

RISC-V 定义了多级特权模式（Privilege Levels）：

  • M-mode（Machine Mode）：最高权限，直接控制硬件（必须实现）
  • S-mode（Supervisor Mode）：操作系统内核运行在此（如 Linux）
  • U-mode（User Mode）：应用程序运行在此

📌 问题：S-mode（如 Linux）不能直接访问某些硬件资源（如中断控制器、时钟、电源管理寄存器），因为这些操作必须在 M-mode 下完成。

解决方案：SBI（Supervisor Binary Interface）

  • SBI 是 RISC-V 官方定义的一套 M-mode 与 S-mode 之间的标准调用接口
  • 类似于 ARM 的 SMC（Secure Monitor Call） 或 x86 的 UEFI Runtime Services

✅ OpenSBI 就是 SBI 规范的一个高质量、开源、可移植的参考实现。

二、OpenSBI 的核心功能

OpenSBI 在 M-mode 运行，为 S-mode 软件提供以下服务：

功能类别 具体服务 说明
基础服务 sbi_console_putchar / getchar 串口调试输出（早期打印）
系统控制 sbi_system_reset 重启或关机
定时器管理 sbi_set_timer 设置 mtimecmp，触发定时器中断
IPI（核间中断） sbi_send_ipi 触发其他核心的软件中断（用于多核启动/调度）
中断转发 处理外部中断（PLIC）并转发给 S-mode 有些平台由 OpenSBI 代理中断使能
HSM（Hart State Management） sbi_hart_start/stop 控制其他 CPU 核（hart）的启停（多核支持）
RFENCE（远程屏障） sbi_remote_fence_i 多核缓存一致性操作
PMU（性能监控） 访问性能计数器 可选扩展

💡 所有这些操作都通过 ECALL 指令 + SBI 调用号 从 S-mode 发起，由 OpenSBI 在 M-mode 安全执行。

三、OpenSBI 在启动流程中的位置

典型的 RISC-V 启动流程（以 Linux 为例）：

  • OpenSBI 是 第一个运行在标准 RISC-V 环境下的通用固件
  • 它初始化：
    • CLINT / PLIC
    • 串口（用于 early console）
    • 多核（通过 HSM 启动 secondary harts）
    • FDT（设备树）传递
  • 然后跳转到下一阶段（如 U-Boot 或直接 Linux）

✅ 没有 OpenSBI，Linux 无法在标准 RISC-V 平台上启动（除非自己实现 SBI）。

四、OpenSBI 的架构特点

1. 模块化设计

  • Firmware：核心 SBI 实现
  • Platform Support：针对不同 SoC（如 QEMU、SiFive Unleashed、Allwinner、StarFive）的适配层
  • Domains（高级）：支持安全世界/非安全世界隔离（类似 ARM TrustZone）

2. 轻量高效

  • 代码精简（C + 汇编），通常 < 64KB
  • 启动速度快，适合嵌入式和服务器

3. 开源与社区驱动

  • 项目地址：https://github.com/riscv-software-src/opensbi
  • 由 RISC-V International 和多家公司（Andes, SiFive, Alibaba, Western Digital 等）共同维护
  • 已被 Linux、U-Boot、Zephyr、FreeBSD 等主流系统采用

五、OpenSBI 与类似组件对比

组件 架构 作用 类比
OpenSBI RISC-V M-mode 固件，提供 SBI 服务 RISC-V 的 “UEFI + PSCI”
ARM Trusted Firmware (ATF) ARM EL3 固件，提供 SMC 服务（如 PSCI） 功能类似，但用于 ARM
U-Boot SPL 多架构 第一阶段 bootloader OpenSBI 常由 SPL 加载
Coreboot x86/RISC-V 开源固件 可集成 OpenSBI 作为 payload

🔸 在 ARM 中，PSCI（Power State Coordination Interface）的功能 ≈ OpenSBI 的 HSM + system_reset。

七、总结

项目 说明
是什么 RISC-V 的开源 M-mode 固件，实现 SBI 标准
为什么需要 让 S-mode 软件（如 Linux）安全访问底层硬件
核心功能 多核管理、定时器、IPI、系统控制、中断代理
地位 RISC-V 软件生态的基础设施，类似 ARM 的 ATF
开源地址 https://github.com/riscv-software-src/opensbi

✅ 简单说：OpenSBI = RISC-V 的“标准固件层”，没有它，操作系统无法在真实硬件上正常运行。
