前缀和算法详解：从入门到精通

发表于 2025-11-13 更新于 2026-03-10 分类于 算法总结
本文字数： 5.6k 阅读时长 ≈ 8 分钟
本文详细讲解了数据结构与算法中的核心技巧——前缀和。通过C++代码实例，深入剖析一维前缀和、二维前缀和及其与哈希表结合的进阶用法，帮助读者快速掌握区间求和优化方案。

前言

在算法竞赛和面试中，我们经常遇到这样一类问题：频繁地查询一个数组中某个连续区间的元素之和。

如果每次查询都使用循环遍历累加，时间复杂度是 O(N)。如果有 M 次查询，总时间复杂度就是 O(N * M)。当数据量达到 10^5 级别时，这种暴力方法必然会导致超时（Time Limit Exceeded）。

这时候，前缀和（Prefix Sum） 算法就登场了。它是一种典型的“空间换时间”的预处理技巧，能够将区间查询的时间复杂度降低到 O(1)，而预处理的时间复杂度仅为 O(N)。

今天，我们就来彻底搞懂前缀和算法，从一维到二维，再到它与哈希表的强强联合。

一、什么是一维前缀和？

1.1 核心思想

想象你有一个数组 nums，我们需要构造一个新的数组 prefixSum。
prefixSum[i] 存储的是原数组中从第 0 个元素到第 i-1 个元素的总和。

用数学语言描述就是：

prefixSum[i] = nums[0] + nums[1] + ... + nums[i-1]

为了方便处理边界情况（比如查询从第一个元素开始的区间），我们通常会让前缀和数组的长度比原数组多 1，并且规定 prefixSum[0] = 0。

1.2 状态转移方程

构建前缀和数组的过程非常直观，当前项的前缀和等于“前一项的前缀和”加上“当前项的值”。

1.3 如何快速查询区间和？

这是前缀和的灵魂所在。如果我们想求原数组下标从 left 到 right（包含两端）的区间和，公式如下：

区间和 = prefixSum[right + 1] - prefixSum[left]

为什么是这样？

  • prefixSum[right + 1] 代表了 nums[0] 到 nums[right] 的总和。
  • prefixSum[left] 代表了 nums[0] 到 nums[left-1] 的总和。
  • 两者相减，中间重复的部分抵消，剩下的正好是 nums[left] 到 nums[right]。

二、二维前缀和：矩阵区域求和

一维前缀和处理的是线段，二维前缀和处理的则是矩阵区域。这在图像处理或网格类题目中非常常见。

2.1 核心思想

给定一个二维矩阵 matrix，我们要构建一个二维前缀和数组 preSum。
preSum[i][j] 表示从矩阵左上角 (0, 0) 到右下角 (i-1, j-1) 这个矩形区域内所有元素的和。

同样地，为了处理边界，我们将 preSum 的行数和列数都设为原矩阵加 1，且第 0 行和第 0 列全部初始化为 0。

2.2 状态转移方程（容斥原理）

计算 preSum[i][j] 时，我们不能简单相加，需要利用容斥原理：

preSum[i][j] = preSum[i-1][j] + preSum[i][j-1] - preSum[i-1][j-1] + matrix[i-1][j-1]

图解逻辑：

  1. 加上上方的矩形和 preSum[i-1][j]。
  2. 加上左方的矩形和 preSum[i][j-1]。
  3. 此时，左上角的区域 preSum[i-1][j-1] 被加了两次，所以需要减去一次。
  4. 最后加上当前格子的值 matrix[i-1][j-1]。

2.3 查询任意子矩阵的和

如果我们要查询左上角为 (row1, col1)，右下角为 (row2, col2) 的子矩阵和：

结果 = preSum[row2+1][col2+1] - preSum[row1][col2+1] - preSum[row2+1][col1] + preSum[row1][col1]

这也是一个容斥原理的应用：大矩形减去上方多余部分，减去左方多余部分，因为左上角被减了两次，所以要加回来一次。

三、进阶应用：前缀和 + 哈希表

前缀和不仅仅是用来求和的，它经常与哈希表结合，解决一些更复杂的子数组问题，例如：

  • “和为 K 的子数组个数”
  • “和可被 K 整除的子数组个数”

这类问题的核心思想是：利用前缀和的差值性质，将问题转化为寻找特定的前缀和值。

3.1 经典例题：和为 K 的子数组

题目描述： 给你一个整数数组 nums 和一个整数 k，请你统计并返回该数组中和为 k 的连续子数组的个数。

暴力解法： 双重循环枚举所有子数组，计算和，时间复杂度 $O(N^2)$。

优化解法（前缀和 + 哈希表）：
假设 prefixSum[i] 是前 i 项的和。
如果存在一个子数组 nums[j...i] 的和为 k，那么必然满足：

prefixSum[i+1] - prefixSum[j] = k
即：prefixSum[j] = prefixSum[i+1] - k

这意味着，当我们遍历到位置 i 时，只需要知道在它之前，有多少个位置 j 的前缀和等于 current_sum - k 即可。我们可以用一个哈希表来记录每个前缀和出现的次数。

关键点解析

  1. 初始化 prefixCount[0] = 1：这步至关重要。如果整个前缀和本身就等于 k，那么我们需要找到一个前缀和为 0 的位置（即起点之前），所以必须预先存入一个 0。
  2. 时间复杂度：只需遍历一次数组，哈希表操作平均为 $O(1)$，总复杂度降为 $O(N)$。
  3. 空间复杂度：O(N)，用于存储哈希表。

四、总结与注意事项

前缀和算法虽然思想简单，但在实际应用中非常强大。以下是几个需要注意的细节：

  1. 下标偏移：
    为了处理边界方便，前缀和数组通常会比原数组多一位（下标从 1 开始有效，0 位置留空或作为基准）。在写代码时，务必理清 i 和 i+1 的关系，避免越界或逻辑错误。

  2. 数据溢出：
    如果原数组元素较大或数组很长，前缀和可能会超出 int 的范围。在 C++ 中，记得根据题目数据范围使用 long long 类型来存储前缀和。

  3. 负数处理：
    前缀和算法完全支持负数。但在结合哈希表使用时（如“和可被 K 整除”的问题），要注意 C++ 中负数取模的结果可能是负数，通常需要调整为正数余数：(sum % k + k) % k。

  4. 不可变性：
    前缀和适用于静态数组（即数组元素不会频繁修改）。如果数组元素会频繁更新，每次更新都需要重新计算前缀和（$O(N)$），效率极低。这种情况下，应该考虑使用树状数组（Fenwick Tree）或线段树（Segment Tree）。

结语

前缀和是算法工具箱中一把锋利的“瑞士军刀”。它通过简单的预处理，将繁琐的区间计算转化为常数时间的减法操作。无论是处理基础的区间求和，还是结合哈希表解决复杂的子数组计数问题，掌握前缀和都能让你的代码运行得更快、更优雅。
