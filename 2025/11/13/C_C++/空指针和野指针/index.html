空指针和野指针

发表于 2025-11-13 分类于 C_C++
本文字数： 2.7k 阅读时长 ≈ 4 分钟
本文介绍空指针和野指针。

空指针（Null Pointer） 和 野指针（Wild Pointer / Dangling Pointer） 是 C/C++ 中非常常见且容易引发程序崩溃或未定义行为的两类指针问题。下面我们将从定义、成因、危害、示例、检测和防范等方面进行详细讲解。

一、空指针（Null Pointer）

1. 定义

空指针是一个明确不指向任何有效内存地址的指针，其值为 NULL（C）或 nullptr（C++11 起）。

  • 在 C 中：NULL 通常被定义为 (void*)0 或 0
  • 在 C++ 中：推荐使用 nullptr（类型安全）

2. 特点

  • 空指针是合法的指针值。
  • 解引用空指针（如 *p）会导致未定义行为（Undefined Behavior），通常表现为程序崩溃（如段错误 Segmentation Fault）。
  • 可用于表示“无对象”、“初始化未完成”、“函数返回失败”等语义。

二、野指针（Wild Pointer / Dangling Pointer）

“野指针”其实包含两类常见情况：

类型 名称 说明
1 未初始化指针（Wild Pointer） 声明后未赋值，值是随机垃圾地址
2 悬空指针（Dangling Pointer） 曾指向有效内存，但该内存已被释放

我们分别讲解。

▶ 类型 1：未初始化的野指针（Wild Pointer）

成因：

指针变量声明后没有初始化，其值是栈上残留的随机值（垃圾地址）。

危害：

  • 写入非法内存 → 段错误（Segmentation Fault）
  • 写入合法但不属于你的内存 → 数据被悄悄破坏（更难调试！）

▶ 类型 2：悬空指针（Dangling Pointer）

成因：

指针曾指向一块有效内存（如堆内存），但该内存已被释放（free/delete），而指针未置空，仍保留旧地址。

  1. malloc/free 或 new/delete 后未置空

危害：

  • 表面可能“正常运行”（因为内存尚未被覆盖），但结果不可靠。
  • 后续该内存被分配给其他对象，导致数据被意外修改。
  • 多线程环境下极易引发难以复现的 bug。

三、空指针 vs 野指针 对比表

特性 空指针（Null Pointer） 野指针（Wild/Dangling Pointer）
是否有明确值 ✅ 有（NULL/nullptr） ❌ 无（随机值 或 已失效地址）
是否合法 ✅ 合法（可比较、可判断） ❌ 非法（使用即危险）
解引用后果 通常立即崩溃（易发现） 可能延迟崩溃或静默破坏（难调试）
初始化建议 声明时赋 NULL 声明时赋 NULL；释放后置 NULL
检测难度 容易（加 if 判断即可） 困难（需工具辅助）

四、如何检测和调试？

1. 静态分析工具

  • clang-static-analyzer
  • PC-lint, Coverity

2. 动态检测工具

  • Valgrind（Linux/macOS）：

    能检测：使用未初始化内存、访问已释放内存、内存泄漏等。

  • AddressSanitizer (ASan)（GCC/Clang）：

    能实时捕获野指针访问、缓冲区溢出等。

3. 编码习惯

  • 所有指针初始化为 NULL
  • free/delete 后立即置 NULL
  • 使用前检查是否为 NULL
  • 避免返回局部变量地址
  • 尽量使用智能指针（C++）：std::unique_ptr, std::shared_ptr

六、总结

问题 根本原因 防范措施
空指针 未检查就解引用 使用前 if (ptr != nullptr)
未初始化野指针 声明后未赋值 声明时初始化为 nullptr
悬空指针 释放后未置空 / 返回局部地址 free/delete 后置 nullptr；不返回栈变量地址

🛑 黄金法则：
“指针要么指向有效对象，要么是空（null）。”
除此之外的状态都是危险的！
