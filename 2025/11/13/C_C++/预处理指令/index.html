预处理器与预处理指令

发表于 2025-11-13 分类于 C_C++
本文详细介绍预处理器与预处理指令。

结合例子（#define a 1 + 5）来详细讲解 C/C++ 中的预处理指令（preprocessor directives），尤其是 #define 的工作原理、常见用法和陷阱。

一、什么是预处理器？

在 C/C++ 编译过程中，预处理器（Preprocessor） 是编译的第一步。它在真正的编译开始前对源代码进行文本级别的处理。所有以 # 开头的行都是预处理指令，例如：

  • #define
  • #include
  • #ifdef / #ifndef / #endif
  • #pragma
  • #error

这些指令不会被编译器直接执行，而是在编译前由预处理器处理。

四、预处理的几个关键特点

特性 说明
文本替换 不进行语法或类型检查，只是字符串替换
无作用域 宏从定义处到文件结束（或 #undef）都有效
可取消定义 用 #undef a 可取消宏定义
不能调试 宏在编译前就被替换了，调试器看不到原始宏名

五、现代替代方案（C++ 推荐）

虽然 #define 仍有用武之地（如条件编译），但在定义常量或简单函数时，更推荐使用语言特性：

  • 常量：const int a = 6; 或 constexpr int a = 6;
  • 函数：用内联函数 inline int square(int x) { return x * x; }

这些方式有类型安全、作用域控制、可调试等优势。

六、总结

  • #define a 1 + 5 是文本替换，不是赋值。
  • a * 2 被替换为 1 + 5 * 2，结果是 11（因优先级）。
  • 务必给宏加括号：#define a (1 + 5)
  • 宏强大但危险，使用时要格外小心优先级和副作用。
  • 在现代 C/C++ 中，尽量用 const、constexpr、inline 替代简单宏。
