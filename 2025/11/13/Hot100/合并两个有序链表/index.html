21. 合并两个有序链表

题目：
将两个升序链表 list1 和 list2 合并为一个新的 升序 链表，并返回新链表的头节点。新链表是通过拼接给定的两个链表的所有节点组成的。

示例 1：
输入：list1 = [1,2,4], list2 = [1,3,4]
输出：[1,1,2,3,4,4]

示例 2：
输入：list1 = [], list2 = []
输出：[]

示例 3：
输入：list1 = [], list2 = [0]
输出：[0]

提示：

  • 两个链表的节点数目范围是 [0, 50]
  • -100 <= Node.val <= 100
  • list1 和 list2 均按 非递减顺序 排列

这道题是链表操作的基本功，也是 归并排序 (Merge Sort) 的核心步骤之一。

如果把算法比作搭积木，那么“合并两个有序链表”就是教你怎么把两串已经穿好的珠子，按照顺序重新穿成一串。它考察的是你对指针断链与重连的熟练程度。

1. 明确问题描述

题目名称：合并两个有序链表 (Merge Two Sorted Lists)

题目大意：

将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的（即原地拼接，尽量不要创建新节点）。

示例：

输入：l1 = [1, 2, 4], l2 = [1, 3, 4]

输出：[1, 1, 2, 3, 4, 4]

2. 核心思路分析

2.1 为什么这像“拉拉链”？

因为两个链表本身已经是有序的（从小到大），我们不需要重新排序，只需要比较两个链表的头节点，谁小就把谁拉出来，接在结果链表的后面。

这就像拉拉链一样，左右两边的齿轮依次咬合。

2.2 核心神器：虚拟头节点 (Dummy Head)

这是做链表题最实用、最能够减少 Bug 的技巧！

痛点：

如果没有虚拟头节点，我们需要写一堆额外的 if-else 来确定新链表的第一个节点是谁（是 l1 的头还是 l2 的头？）。初始化 head 指针会很麻烦。

解决方案：

我们人为地创建一个假节点（Dummy Node，通常值为 -1 或 0），把它放在新链表的最前面。

  • 我们只需要维护一个指针 tail（尾指针），永远指向新链表的最后一个节点。
  • 开始时，tail 指向 dummy。
  • 比较 l1 和 l2 的值，小的那个接在 tail 后面，然后 tail 往后移一步。
  • 最后返回 dummy->next 即可。

2.3 算法流程

  1. 创建一个 dummy 节点。定义 tail 指针指向 dummy。
  2. 循环比较：当 list1 和 list2 都还没走完时：
    • 如果 list1->val < list2->val：把 list1 接在 tail 后面，list1 前移。
    • 否则：把 list2 接在 tail 后面，list2 前移。
    • tail 指针前移（始终指向结果链表的末尾）。
  3. 处理剩余：
    • 循环结束后，肯定有一个链表先空了，另一个链表可能还有剩。
    • 因为剩下的链表本身就是有序的，直接把剩下的整条链表接在 tail 后面即可（不需要循环遍历）。

4. 复杂度分析

时间复杂度：$O(N + M)$

  • $N$ 和 $M$ 分别是两个链表的长度。
  • 我们最多遍历两个链表中的每一个节点一次。
  • 操作都是常数时间的指针修改。

空间复杂度：$O(1)$

  • 我们只是调整了现有节点的 next 指针，除了 dummy 节点外，没有创建任何新的节点。
  • 注：如果是递归解法，空间复杂度会是 $O(N+M)$ 的栈空间，所以迭代法更优。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么最后不需要 while 循环去接剩下的节点？
    • A: 这就是链表的优势！
    • 比如 l1 剩下了 7 -> 8 -> 9，tail 现在停在 6。
    • 我们只需要执行 tail->next = l1。
    • 此时 tail (也就是 6) 直接指到了 7。而 7 本来就指着 8，8 指着 9。
    • 一句话就把后面一串全接上了，不需要一个一个去接。
  • Q: 必须用 new ListNode(-1) 吗？
    • A: 不一定。你也可以写 ListNode dummy(-1); 然后用 ListNode* tail = &dummy;。
    • 这样 dummy 是在栈上分配的，函数结束自动销毁，不需要手动 delete。这在 C++ 中是更推荐的写法，但在 LeetCode 指针题解中，new 更常见以便于理解“节点”的概念。
