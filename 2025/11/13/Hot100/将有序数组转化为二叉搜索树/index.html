108. 将有序数组转换为二叉搜索树

题目：
给你一个整数数组 nums，其中元素按升序排列。请你将其转换为一棵高度平衡的二叉搜索树。

高度平衡二叉树 定义为：任意节点的左右子树高度差不超过 1。

示例 1：
输入：nums = [-10,-3,0,5,9]
输出：[0,-3,9,-10,null,5]
解释：[0,-10,5,null,-3,null,9] 也是一个有效答案。

示例 2：
输入：nums = [1,3]
输出：[3,1]
解释：[1,null,3] 也是一个有效答案。

提示：

  • 1 <= nums.length <= 10⁴
  • -10⁴ <= nums[i] <= 10⁴
  • nums 是严格递增的

这道题是 二叉搜索树 (BST) 构建问题的经典入门题。

它是力扣（LeetCode）第 108 题。

这道题考察的是你对 BST 性质的逆向思维：通常我们说“BST 的中序遍历是有序数组”，现在反过来，给你一个有序数组，让你还原出一棵平衡的 BST。

1. 明确问题描述

题目名称：将有序数组转换为二叉搜索树 (Convert Sorted Array to Binary Search Tree)

题目大意：

给你一个整数数组 nums ，其中元素已经按 升序 排列。

请你将其转换为一棵 高度平衡 的二叉搜索树。

高度平衡 的定义：

一个二叉树每个节点的左右两个子树的高度差的绝对值不超过 1。

示例：

输入：nums = [-10, -3, 0, 5, 9]

输出：[0, -3, 9, -10, null, 5]

（或者其他平衡的变种，答案不唯一）

2. 核心思路分析

2.1 为什么要找中间节点？

如果我们要构建一棵 平衡 的树，意味着左右两边的节点数量要大致相等。

在一个有序数组中，中位数 (Middle Element) 天然地把数组分成了数量相等的左右两部分。

  • 如果我们选 数组中间的数 作为 根节点：
    • 左边的数 都比它小 $\rightarrow$ 刚好构成 左子树。
    • 右边的数 都比它大 $\rightarrow$ 刚好构成 右子树。

这正是 分治法 (Divide and Conquer) 的完美应用场景。

2.2 算法流程

  1. 取中点：找到当前数组范围 [left, right] 的中间位置 mid。
  2. 建根：用 nums[mid] 创建一个新节点。
  3. 递归左边：用 [left, mid - 1] 这个范围去构建左子树，并挂在根节点的左边。
  4. 递归右边：用 [mid + 1, right] 这个范围去构建右子树，并挂在根节点的右边。
  5. 终止条件：当 left > right 时，说明当前范围没有数字了，返回 nullptr。

4. 复杂度分析

时间复杂度：$O(N)$

  • 每个元素都被访问一次用来创建节点。
  • 递归过程没有重复计算。

空间复杂度：$O(\log N)$

  • 这里的空间主要消耗在 递归调用栈。
  • 因为我们每次都取中点，构建出来的是一棵 平衡二叉树。
  • 平衡二叉树的高度是 $\log N$，所以递归深度也是 $\log N$。
  • 注：如果构建成链状树（比如每次取第一个元素做根），递归深度就是 $O(N)$，但本题算法保证了平衡。

5. 初学者常见疑问 (Q&A)

  • Q: 数组长度是偶数时，中间有两个数，选哪个？
    • A: 选左边的或者右边的都可以。
    • 比如 [1, 2]。
    • 选左边 1 做根：2 是右孩子。
    • 选右边 2 做根：1 是左孩子。
    • 这两种都是平衡的 BST。题目通常接受任意一种解法。
  • Q: 为什么要用 left + (right - left) / 2 而不是 (left + right) / 2？
    • A: 这是一个编程习惯问题。在 left 和 right 都非常大（接近整数最大值 INT_MAX）时，直接相加可能会导致 整型溢出 (Overflow) 变成负数。虽然这道题的数据范围可能不会溢出，但写成前者是更安全的工程规范。
