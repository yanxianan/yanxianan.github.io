437. 路径总和 III

题目：
给定一个二叉树的根节点 root 和一个整数目标值 targetSum，找出所有从根节点到叶子节点路径上节点值之和等于 targetSum 的路径总数。

注意：路径不需要从根节点开始，也不需要在叶子节点结束。但路径必须是向下的（只能从父节点到子节点）。

示例 1：
输入：root = [10,5,-3,3,2,null,11,3,-2,null,1], targetSum = 8
输出：3
解释：有三条路径满足条件：
5 -> 3
5 -> 2 -> 1
-3 -> 11

提示：

  • 树中节点数目范围是 [0, 1000]
  • -1000 <= Node.val <= 1000
  • -1000 <= targetSum <= 1000

这道题是 二叉树 (Binary Tree) 题目中难度系数较高的 Medium 题。

它是力扣（LeetCode）第 437 题。

之所以说它难，是因为它融合了 “树的遍历” 和 “前缀和 (Prefix Sum)” 两个概念。如果你只用暴力的递归解法（双重递归），时间复杂度是 $O(N^2)$，面试官通常不会满意。

我们要讲的是 $O(N)$ 的最优解法，这需要你把“数组里的前缀和”思想迁移到“树”上。

1. 明确问题描述

题目名称：路径总和 III (Path Sum III)

题目大意：

给定一个二叉树的根节点 root ，和一个整数 targetSum ，求该二叉树里节点值之和等于 targetSum 的 路径的数目。

关键约束：

  1. 路径方向：必须是 向下 的（只能从父节点到子节点）。
  2. 起止点：不一定 要从根节点开始，也 不一定 要在叶子节点结束。

示例：

输出：3

解释：和为 8 的路径有：

  1. 5 -> 3
  2. 5 -> 2 -> 1
  3. -3 -> 11

2. 核心思路分析

2.1 为什么暴力法慢？

暴力法的思路是：

  1. 遍历每个节点（比如 A）。

  2. 以 A 为起点，向下 DFS，看有多少路径等于 targetSum。

    这意味着树越深，重复计算越多。就像你在数组里找子数组和，如果你对每个起点都重新累加一遍，那就是 $O(N^2)$。

2.2 核心策略：前缀和 (Prefix Sum) + 哈希表

这就好比是在做一个 “两数之和” 的变种。

定义：

  • 当前路径和 (currSum)：从 根节点 到 当前节点 的所有节点值之和。

  • 我们想要找一条中间的路径（比如 Node A -> ... -> Node B），它的和是 targetSum。

  • 根据数学关系：

    $$\text{中间路径和} = \text{B 的前缀和} - \text{A 的前缀和}$$

    即：

    $$\text{targetSum} = \text{currSum} - \text{oldSum}$$

  • 移项得：

    $$\text{oldSum} = \text{currSum} - \text{targetSum}$$

算法流程：

  1. 我们维护一个全局的 哈希表 (prefixMap)，记录：{ 前缀和 : 出现的次数 }。
  2. 当我们遍历到一个节点时，算出从根到它的 currSum。
  3. 查表：看看哈希表里有没有 currSum - targetSum 这个值？
    • 如果有，说明如果不算上面那一段，剩下的这段路径和刚好是 targetSum。
    • 答案加上 prefixMap[currSum - targetSum] 的次数。
  4. 更新表：把当前的 currSum 存入哈希表（次数 +1），继续递归子节点。
  5. 回溯 (Backtracking)：这是最关键的一步！
    • 当离开当前节点（递归返回）时，要把当前的 currSum 从哈希表里 减掉。
    • 为什么？ 因为二叉树有分叉。左子树里的前缀和，不能被右子树用到（它们不在同一条路径上）。

4. 复杂度分析

时间复杂度：$O(N)$

  • 我们只遍历了一次树（DFS）。
  • 哈希表的查找和插入操作平均是 $O(1)$。
  • 所以总时间是线性的。

空间复杂度：$O(N)$

  • 哈希表最坏情况存储 $N$ 个节点的前缀和。
  • 递归栈深度最坏也是 $N$。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么要回溯 prefixMap[currSum]--？
    • A: 想象一下，左子树里有一条路径前缀和是 10。现在我们退回根节点，准备去遍历右子树。如果不减掉，右子树在查表时，可能会误以为自己的路径上也有一个前缀和是 10 的节点。但实际上那个节点在左边，跟右边不在一条路上（不在同一条向下的路径中）。
  • Q: 为什么要用 long long？
    • A: 题目数据范围中，节点值可能是很大的整数，累加后可能会超过 int 的范围（溢出）。在 C++ 中，涉及累加和的题目，最好都用 long long 保平安。
  • Q: prefixMap[0] = 1 是什么意思？
    • A: 这是一个通用的技巧。
    • 假设 target = 8，当前路径和 currSum 刚好也是 8。
    • 公式：oldSum = 8 - 8 = 0。
    • 我们需要找前缀和为 0 的次数。如果没有初始化，哈希表里没有 0，我们就会漏掉这条“从根节点开始刚好等于 target”的路径。
