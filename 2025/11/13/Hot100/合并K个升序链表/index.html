23. 合并 K 个升序链表

题目：
给你一个链表数组，每个链表都已按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

示例 1：

输入：lists = [[1,4,5],[1,3,4],[2,6]]
输出：[1,1,2,3,4,4,5,6]
解释：链表被合并为一个升序链表。

输入：lists = []
输出：[]

输入：lists = [[]]
输出：[]

这道题是 “合并两个有序链表” 的终极威力加强版。

它是力扣（LeetCode）第 23 题，难度为 Hard。

这道题是面试中非常经典的“分水岭”题目：

  • 写出暴力解法（两两合并） $\rightarrow$ 面试通过率一般。
  • 写出 归并（分治） 或 优先队列（堆） 解法 $\rightarrow$ High Hire（高评价）。

今天我们重点讲解 优先队列 (Priority Queue / Min-Heap) 的解法，因为这种思路不仅能解这道题，还是处理 “多路归并” 问题的通用模板。

1. 明确问题描述

题目名称：合并 K 个升序链表 (Merge k Sorted Lists)

题目大意：

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

示例：

输入：lists = [[1,4,5], [1,3,4], [2,6]]

输出：[1, 1, 2, 3, 4, 4, 5, 6]

2. 核心思路分析

2.1 为什么两两合并不够快？

如果我们复用上一题的代码，先合并第1和第2个，结果再和第3个合并……

假设有 $k$ 个链表，每个链表长 $n$。

  • 第一次合并：处理 $2n$ 个点。
  • 第二次合并：处理 $3n$ 个点。
  • …
  • 越往后，累积的链表越长，重复遍历的次数越多。时间复杂度会达到 $O(k^2 n)$。当 $k$ 很大时，会超时。

2.2 核心策略：优先队列 (最小堆)

想象一下，有 $k$ 个班级排成纵队，每个班级都是按身高从矮到高排的。现在要从这 $k$ 个队里选人，组成一个总的矮到高队列。

  • 谁是目前最矮的？
  • 肯定是在这 $k$ 个队伍的 队头 里面选。
  • 我们只需要比较这 $k$ 个队头，找出最小的那个，把他拉出来。
  • 他走了之后，那个队伍的 下一个 人就变成了新的队头，加入比较。

我们需要一个“裁判”，能快速从 $k$ 个数里找出最小值。

这就是 最小堆 (Min-Heap) 的拿手好戏！

算法流程：

  1. 入堆：把 $k$ 个链表的 头节点 全部扔进最小堆。
  2. 出堆：
    • 弹出堆顶元素（它是当前所有候选人里最小的）。
    • 把它接在结果链表的后面。
  3. 补充：
    • 如果刚才弹出的那个节点还有 next（后面还有人），就把它的 next 扔进堆里。
  4. 循环：重复步骤 2-3，直到堆变空。

4. 复杂度分析

假设共有 $k$ 个链表，所有链表的节点总数为 $N$。

时间复杂度：$O(N \log k)$

  • 堆的大小：堆里最多同时只有 $k$ 个节点（每个链表贡献一个）。
  • 操作次数：每个节点都会被 push 进堆一次，也会被 pop 出堆一次。总共 $N$ 个节点。
  • 单次操作耗时：堆的插入和删除操作复杂度是 $O(\log (\text{堆大小}))$，即 $O(\log k)$。
  • 总耗时：$O(N \log k)$。这是处理这类问题的理论最优解。

空间复杂度：$O(k)$

  • 堆需要存储 $k$ 个指针。
  • （不算输出结果占用的空间）。

5. 另一种解法：分治法 (Divide and Conquer)

这种解法不需要额外的数据结构（堆），而是利用递归的思想。

它的逻辑和 归并排序 完全一样，或者说像 锦标赛。

  • 第一轮：第 0 个和第 1 个合并，第 2 个和第 3 个合并… 变成了 $k/2$ 个链表。
  • 第二轮：两两合并… 变成了 $k/4$ 个链表。
  • …
  • 直到只剩 1 个。

6. 初学者常见疑问 (Q&A)

  • Q: 为什么优先队列的比较函数 a->val > b->val 代表最小堆？
    • A: 这是一个反直觉的设计。
    • std::priority_queue 默认将“最大”的元素放在顶部。
    • 比较器 comp(a, b) 返回 true 意味着 “a < b”（逻辑上）。在默认行为中，”小的” 被认为是 “优先级低的”（沉底）。
    • 当我们写 a > b 返回 true 时，我们是在骗编译器说：“大的那个其实是比较弱的（小的）”。于是编译器就把数值小的那个当成了“老大”放在了堆顶。
  • Q: priority_queue 里存 ListNode\* 还是 int？
    • A: 必须存 ListNode*。
    • 因为我们不仅需要知道最小值是多少，还需要知道这个最小值来自哪个链表，以便把该链表的下一个节点 (node->next) 推入堆中。
