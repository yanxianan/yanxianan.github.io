25. K 个一组翻转链表

题目：
给你一个链表，每 k 个节点一组进行翻转，请你返回修改后的链表。

k 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 k 的整数倍，那么最后剩余的节点保持原有顺序。

示例 1：
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]

示例 2：
输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]

提示：

  • 链表中节点的数目范围是 [0, 500]
  • 0 <= Node.val <= 1000
  • 1 <= k <= 链表长度

这是力扣（LeetCode）第 25 题，标记为 Hard。

这道题被誉为 “链表题的面霸”。

如果你在面试中遇到这道题，说明面试官对你的期望很高。它不再是单一技巧的考察，而是要求你将 “遍历计数”、“断链重连” 和 “局部反转” 这三套组合拳打得行云流水。

只要你能把这道题逻辑理顺，链表这个板块你就彻底通关了。

1. 明确问题描述

题目名称：K 个一组翻转链表 (Reverse Nodes in k-Group)

题目大意：

给你一个链表的头节点 head，每 k 个节点一组进行翻转，请你返回修改后的链表。

  • 如果节点总数不是 k 的整数倍，那么请将最后剩余的节点保持原有顺序。

关键约束：

  • 空间复杂度：$O(1)$（意味着不能用递归，必须用迭代）。
  • 不能只改变节点的值，必须修改节点指针。

示例：

输入：head = [1, 2, 3, 4, 5], k = 2

输出：[2, 1, 4, 3, 5] (1,2反转; 3,4反转; 5不够一组，不动)

输入：head = [1, 2, 3, 4, 5], k = 3

输出：[3, 2, 1, 4, 5] (1,2,3反转; 4,5不够一组，不动)

2. 核心思路分析

2.1 宏观策略：切断 -> 反转 -> 拼接

这道题的核心难点在于 “维护连接点”。

当我们把中间某一段 [start, end] 翻转后，它需要和前面的链表接上，也需要和后面的链表接上。

我们需要定义四个关键指针来定位每一组：

  1. pre：这一组的前驱节点（上一组的尾巴）。
  2. start：这一组原本的头（翻转后变成尾）。
  3. end：这一组原本的尾（翻转后变成头）。
  4. next：这一组的后继节点（下一组的头）。

2.2 详细步骤 (The Algorithm)

  1. 初始化：创建一个 dummy 节点指向 head。pre 和 end 一开始都指向 dummy。
  2. 寻找分组：
    • 让 end 向后走 k 步。
    • 如果在走这 k 步的过程中遇到了 NULL，说明剩余节点不足 k 个，直接结束（不需要反转）。
  3. 记录断点：
    • 记录 start = pre->next。
    • 记录 next = end->next。
    • 关键一步：把 end->next = nullptr。为什么要这么做？为了让我们的反转函数知道哪里是终点（把它当成一个独立的短链表来反转）。
  4. 局部反转：
    • 调用我们熟悉的 reverse(start) 函数。
  5. 重新连接：
    • pre->next 指向翻转后的新头（也就是原来的 end）。
    • start->next 指向刚才保存的 next（这一组的新尾巴连接下一组的头）。
  6. 重置指针：
    • pre 移动到 start（因为 start 变成了这一组的尾巴，也就是下一组的前驱）。
    • end 重置为 pre，准备开始下一轮寻找。

4. 复杂度分析

时间复杂度：$O(N)$

  • 我们看起来有很多指针跑来跑去，但本质上：
    • 每个节点会被 end 指针访问一次（计数）。
    • 每个节点会被 reverse 函数访问一次（翻转）。
  • 总操作次数是 $2N$，也就是线性的 $O(N)$。

空间复杂度：$O(1)$

  • 我们只用到了常数个指针变量 (dummy, pre, end, start, nextGroup)。
  • 符合题目要求的原地算法。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么要断开链表 (end->next = nullptr)？
    • A: 这为了复用标准的 reverse 函数。
    • 标准的 reverse 函数是“直到遇到 nullptr 停止”。如果我们不断开，reverse 会一直反转到链表真正的末尾，把后面不需要翻转的组也给翻了。
    • 当然，你也可以重写一个 reverse(start, end) 函数，但这会增加代码的复杂度和出错率。“断开-反转-重连” 是最稳健的策略。
  • Q: 为什么循环里要重置 end = pre？
    • A: 因为翻转后，原来的 end 已经跑到这一组的前面去了，而 pre (也就是 start) 才是这一组的最后。
    • 下一轮我们要从这个新的尾巴开始往后数 k 个，所以 end 必须回到起跑线（pre 的位置）。
