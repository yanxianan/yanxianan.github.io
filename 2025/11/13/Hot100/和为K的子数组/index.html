560. 和为 K 的子数组

题目：
给你一个整数数组 nums 和一个整数 k，请你统计并返回 该数组中和为 k 的连续子数组的个数。

示例 1：

示例 2：

提示：

  • 1 <= nums.length <= 2 * 10⁴
  • -1000 <= nums[i] <= 1000
  • -10⁷ <= k <= 10⁷

这道题是 前缀和 (Prefix Sum) 结合 哈希表 (Hash Map) 的经典中的经典。

很多同学在做这道题时，第一反应是：“能不能用滑动窗口？”

答案是：不能。

这道题是很多初学者的“滑铁卢”，因为输入数组中可能包含 负数。如果有负数，窗口扩大时和不一定增加，缩小时和不一定减小，这就打破了滑动窗口的单调性条件。

所以，我们需要一把新武器：前缀和 + 哈希表。

1. 明确问题描述

题目名称：和为 K 的子数组 (Subarray Sum Equals K)

题目大意：

给你一个整数数组 nums 和一个整数 k。请你统计数组中 和为 k 的 连续子数组 的个数。

关键约束：

  • 连续：子数组必须是数组中连续的一段。
  • 含负数：nums 中的数字可能是负数（这意味着不能用双指针/滑动窗口）。
  • 复杂度：暴力解法 $O(n^2)$ 会超时，我们需要 $O(n)$ 的解法。

示例：

输入：nums = [1, 1, 1], k = 2

输出：2

解释：[1, 1] (前两个) 和 [1, 1] (后两个)。

2. 核心思路分析

2.1 什么是前缀和？

假设数组是 nums = [1, 2, 3]。

前缀和 pre[i] 表示从数组开头到下标 i 的所有元素之和。

  • pre[0] = 1
  • pre[1] = 1 + 2 = 3
  • pre[2] = 1 + 2 + 3 = 6

神奇的公式：

任意一段子数组 nums[j..i] 的和，都可以用前缀和算出来：

$$\text{Sum}(j, i) = \text{pre}[i] - \text{pre}[j-1]$$

(也就是：当前位置的前缀和 - 之前某个位置的前缀和)

2.2 问题的转化 (The “Math” Part)

我们要找和为 k 的子数组，也就是要找满足下面公式的 i 和 j：

$$\text{pre}[i] - \text{pre}[j-1] = k$$

我们可以把公式稍微移项变换一下：

$$\text{pre}[j-1] = \text{pre}[i] - k$$

这句话翻译成人话就是：

当我们遍历到第 i 个位置，手里的前缀和是 pre[i] 时，我们只需要回头看看：

“嘿，前面的历史记录里，有没有哪个位置的前缀和刚好等于 pre[i] - k ？”

  • 如果有 1 个，那就说明找到 1 个满足条件的子数组。
  • 如果有 N 个，那就说明找到了 N 个。

2.3 为什么要用哈希表？

我们需要快速查找“历史记录”里有没有某个值，以及它出现了几次。

  • Key (键): 前缀和的值。
  • Value (值): 这个前缀和出现了多少次。

这样，我们每遍历到一个新数字，只需要问哈希表：“有没有 当前前缀和 - k？” 这一步查询是 $O(1)$ 的。

4. 复杂度分析

时间复杂度：$O(n)$

  • 我们只需要遍历数组一次。
  • 在每一轮循环中，哈希表的查找（count）和插入（mp[pre]++）操作平均复杂度是 $O(1)$。
  • 总耗时：线性时间。

空间复杂度：$O(n)$

  • 最坏情况下（例如数组全是正数），每一个位置的前缀和都不一样，哈希表需要存储 $n$ 个键值对。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么一定要 mp[0] = 1？
    • A: 举个例子：nums = [3], k = 3。
    • 循环开始：pre 变成 3。
    • 我们需要找 pre - k = 3 - 3 = 0。
    • 如果哈希表里没有 {0: 1}，我们就会以为没找到，结果返回 0，但实际上应该是 1。
    • 这个初始化代表了“空数组的和为0”，是前缀和问题的标准起手式。
  • Q: 为什么是先查询 mp[pre-k]，然后再 mp[pre]++？
    • A: 顺序不能乱！
    • 我们要找的子数组必须是之前的某个位置作为起点。
    • 如果先把自己存进去，当 k=0 时，pre - 0 就是 pre 自己，你会把自己误认为是满足条件的历史前缀和（这就变成了长度为0的子数组，不符合题目要求）。
  • Q: 如果数组全是正数，可以用滑动窗口吗？
    • A: 可以！如果全是正数，可以用类似“无重复字符最长子串”的伸缩窗口方法，空间复杂度可以降到 $O(1)$。但因为这道题有负数，所以只能用前缀和+哈希表。
