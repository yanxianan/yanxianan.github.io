236. 二叉树的最近公共祖先

题目：
给定一个二叉树，找到该树中两个指定节点的最近公共祖先。

最近公共祖先（LCA）定义为：对于有根树 T 的两个节点 p 和 q，它们的 LCA 是位于从根节点到 p 和 q 的路径之间的最低节点。

示例 1：
输入：root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 1
输出：3
解释：节点 5 和节点 1 的最近公共祖先是节点 3。

示例 2：
输入：root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4
输出：5
解释：节点 5 和节点 4 的最近公共祖先是节点 5，因为节点 5 是节点 4 的祖先。

提示：

  • 树中节点的数目范围是 [2, 10⁵]
  • -10⁹ <= Node.val <= 10⁹
  • 所有 Node.val 互不相同
  • p != q
  • p 和 q 均存在于树中

这道题是二叉树递归逻辑的 “集大成者”。

它是力扣（LeetCode）第 236 题。

这道题的代码非常短（核心只有 5 行），但逻辑极其精妙。它是检验你是否真正理解 “后序遍历” (Post-order Traversal) 和 “递归返回值” 的试金石。

1. 明确问题描述

题目名称：二叉树的最近公共祖先 (Lowest Common Ancestor of a Binary Tree)

题目大意：

给定一个二叉树，找到该树中两个指定节点 p 和 q 的最近公共祖先 (LCA)。

最近公共祖先的定义：

对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。

示例：

Plaintext

  • 输入：root = 3, p = 5, q = 1
    • 输出：3 (3 是 5 和 1 的分叉点)
  • 输入：root = 3, p = 5, q = 4
    • 输出：5 (因为 5 是 4 的祖先，且 5 是它自己，所以 LCA 是 5)

2. 核心思路分析

2.1 为什么是后序遍历？

我们需要在树的底部找到 p 和 q，然后把这个信息 “向上汇报”。

父节点需要根据子节点汇报上来的信息做决策：

  • “我的左边有没有找到 p 或 q？”

  • “我的右边有没有找到 p 或 q？”

    这种 先子后父 的逻辑，就是典型的 后序遍历。

2.2 递归函数的定义

函数 lowestCommonAncestor(root, p, q) 的含义是：

“在以 root 为根的子树中，去寻找 p 和 q。”

返回值的含义（这是最关键的）：

  1. 如果找到了 LCA，返回 LCA 节点。
  2. 如果只找到了 p，返回 p。
  3. 如果只找到了 q，返回 q。
  4. 如果都没找到，返回 NULL。

2.3 核心决策逻辑 (The Decision Logic)

当我们在某个节点 root 汇总左右子树的搜索结果时，有三种情况：

  1. 左边找到了，右边也找到了：
    • 这意味着 p 在左子树，q 在右子树（或者反过来）。
    • 当前节点 root 就是分叉点！ 即 LCA。
    • Action: 返回 root。
  2. 左边没找到，右边找到了：
    • 说明 p 和 q 都在右子树里（或者其中一个是另一个的祖先）。
    • Action: 返回右边找到的那个结果。
  3. 左边找到了，右边没找到：
    • 说明 p 和 q 都在左子树里。
    • Action: 返回左边找到的那个结果。
  4. 两边都没找到：
    • Action: 返回 NULL。

4. 复杂度分析

时间复杂度：$O(N)$

  • 最坏情况下，我们需要遍历整棵树才能找到 p 和 q。
  • 每个节点只被访问一次。

空间复杂度：$O(N)$

  • 递归栈的深度。最坏情况（退化成链表）为 $N$，平均情况为 $\log N$。

5. 初学者常见疑问 (Q&A)

  • Q: 如果 p 是 q 的父节点，代码怎么处理？
    • A: 这是一个很好的问题。
    • 当递归遍历到 p 时，触发 if (root == p) return root;。
    • 我们 直接返回了 p，没有继续去遍历 p 的子树（也就是没去找 q）。
    • 这会导致最终结果返回 p。
    • 这符合 LCA 的定义：如果一个节点是另一个节点的祖先，那么这个节点就是 LCA。代码逻辑天然涵盖了这一点。
  • Q: 二叉搜索树 (BST) 的 LCA 会更简单吗？
    • A: 会简单很多！
    • BST 有顺序：
      • 如果 p 和 q 都比当前节点小 -> 往左走。
      • 如果 p 和 q 都比当前节点大 -> 往右走。
      • 如果一个大一个小 -> 当前节点就是 LCA。
    • 这是力扣 235 题，不需要递归回溯，一次遍历就能搞定。
