2. 两数相加

题目：
给你两个 非空 的链表，表示两个非负整数。数字以逆序存储在链表中，每一位数字对应一个节点。

请你将这两个数相加，并返回一个新的链表来表示它们的和。

示例 1：
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807。

示例 2：
输入：l1 = [0], l2 = [0]
输出：[0]

示例 3：
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]

提示：

  • 两个链表的节点数目范围是 [1, 100]
  • 0 <= Node.val <= 9
  • 它保证两个输入数字不包含前导零

这道题是力扣（LeetCode）第 2 题 “两数相加” (Add Two Numbers)。

它虽然是链表题，但本质上是在模拟我们小学数学课上学的 “竖式加法”。

这道题非常适合练习 链表遍历 和 进位逻辑 的处理。

1. 明确问题描述

题目名称：两数相加 (Add Two Numbers)

题目大意：

给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照 逆序 的方式存储的，并且每个节点只能存储 一位 数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

关键点：

  • 逆序存储：个位在链表头，十位在第二个… 这其实是天大的好事！这意味着我们遍历链表的方向，刚好和我们做加法的方向（从低位到高位）是一致的。
  • 进位：相加如果超过 10，需要向后一个节点进位。

示例：

输入：l1 = [2, 4, 3], l2 = [5, 6, 4]

含义：$342 + 465 = 807$

输出：[7, 0, 8] (代表 807)

解释：

  • 个位：$2 + 5 = 7$
  • 十位：$4 + 6 = 10$ (留 0，进 1)
  • 百位：$3 + 4 + 1(\text{进位}) = 8$

2. 核心思路分析

2.1 模拟竖式加法

我们需要同时遍历两个链表 l1 和 l2。

对于每一个位置，我们计算：

$$\text{sum} = \text{val1} + \text{val2} + \text{carry}$$

其中 carry 是上一轮运算留下来的进位（初始为 0）。

  • 当前位的值：sum % 10 （我们要创建一个新节点存这个值）。
  • 新的进位：sum / 10 （留给下一轮使用）。

2.2 痛点解决：长短不一怎么办？

两个链表长度可能不一样，比如 99 + 1。

  • 当一个链表走完（变 NULL）而另一个还没走完时，我们不能停。
  • 技巧：如果某个链表已经走到头了，我们就当它那个位置的数字是 0。

2.3 容易被遗忘的边界：最后的进位

如果算到最后，l1 完了，l2 也完了，但是 carry 还是 1 怎么办？

比如 5 + 5。

  • 第一轮：$5+5=10$，存 0，carry=1。
  • 链表结束了？不行！
  • 我们需要再补一个节点存最后的 1。结果是 [0, 1]。

通用循环条件：

只要 l1 没走完 或者 l2 没走完 或者 carry 不为 0，循环就得继续！

4. 复杂度分析

时间复杂度：$O(\max(M, N))$

  • $M$ 和 $N$ 分别是两个链表的长度。
  • 我们要遍历较长的那个链表一遍。

空间复杂度：$O(1)$

  • 如果不计算存储结果所用的空间（返回值所需的链表），我们只用了常数个变量。
  • 如果计算结果链表，空间是 $O(\max(M, N))$，因为结果链表的长度最多比长链表多 1 位。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么不用把链表转成 int 或者 long long 加完再转回去？
    • A: 这是一个陷阱！链表的长度可能非常长（比如 100 个节点），这代表一个 100 位的整数。任何 long long 都存不下这么大的数（溢出）。所以必须一位一位地加。
  • Q: 如果题目给的是“正序”存储（高位在头）怎么办？
    • A: 那难度就升级了（这是力扣 445 题）。
    • 方法一：先把两个链表反转，然后用这一题的代码，算完再反转回去。
    • 方法二：利用栈 (Stack)。把两个链表压入栈，利用栈“后进先出”的特性，从个位开始弹出来相加。
