105. 从前序与中序遍历序列构造二叉树

题目：
给你两棵二叉树的前序遍历和中序遍历的结果，请你重建这棵树。

假设树中没有重复的值。

示例 1：
输入：preorder = [3,9,20,15,7], inorder = [9,3,15,20,7]
输出：[3,9,20,null,null,15,7]

示例 2：
输入：preorder = [-1], inorder = [-1]
输出：[-1]

提示：

  • 1 <= preorder.length <= 3000
  • inorder.length == preorder.length
  • -3000 <= preorder[i], inorder[i] <= 3000
  • preorder 和 inorder 均无重复元素
  • preorder 是 inorder 的前序遍历

这道题是二叉树题目中 “还原大师” 级别的题目。

它是力扣（LeetCode）第 105 题。

这道题考察的是你对 “前序遍历” 和 “中序遍历” 特性的深刻理解。这就好比侦探破案：给你两个不同的侧面线索，让你还原出唯一的作案现场（二叉树结构）。

1. 明确问题描述

题目名称：从前序与中序遍历序列构造二叉树 (Construct Binary Tree from Preorder and Inorder Traversal)

题目大意：

给定两个整数数组 preorder 和 inorder，其中：

  • preorder 是二叉树的 先序遍历。
  • inorder 是同一棵树的 中序遍历。
  • 题目保证数组中 没有重复元素（这点至关重要）。

请你构造并返回这棵二叉树。

示例：

Input:

  • preorder = [3, 9, 20, 15, 7]
  • inorder = [9, 3, 15, 20, 7]

2. 核心思路分析

2.1 寻找突破口：根节点在哪里？

我们回顾一下两种遍历的特性：

  1. 前序遍历 (Preorder)：[ 根 | 左子树... | 右子树... ]
    • 铁律：前序遍历的 第一个元素 永远是当前的 根节点。
  2. 中序遍历 (Inorder)：[ 左子树... | 根 | 右子树... ]
    • 铁律：一旦确定了根节点，中序遍历中 根节点左边 的所有元素属于左子树，根节点右边 的所有元素属于右子树。

2.2 算法流程 (递归分治)

第一步：找根

从 preorder 中取出第一个数 3。这就是整棵树的根。

第二步：切分左右子树

去 inorder 中找到 3 的位置（下标 1）。

  • inorder 变成了：[9] (左) + 3 (根) + [15, 20, 7] (右)。
  • 我们可以得出结论：
    • 左子树长度：1 个节点。
    • 右子树长度：3 个节点。

第三步：切分前序数组

回到 preorder，利用刚才算出的长度切分：

  • preorder 变成了：3 (根) + [9] (左) + [20, 15, 7] (右)。

第四步：递归

现在问题分解成了两个子问题：

  1. 左儿子：用 pre = [9], in = [9] 构建。
  2. 右儿子：用 pre = [20, 15, 7], in = [15, 20, 7] 构建。

3. 优化技巧：哈希表 (Hash Map)

在“第二步”中，我们需要在 inorder 数组中查找根节点的位置。

如果每次都用 for 循环遍历查找，时间复杂度是 $O(N)$，总复杂度会退化到 $O(N^2)$。

优化：

我们可以提前把 inorder 数组中的 元素 -> 下标 存到一个 哈希表 中。

这样查找根节点位置的操作就变成了 $O(1)$。

5. 复杂度分析

时间复杂度：$O(N)$

  • 有了哈希表，查找根节点是 $O(1)$。
  • 每个节点都会被构建一次，递归函数调用 $N$ 次。
  • 总耗时 $O(N)$。

空间复杂度：$O(N)$

  • 哈希表存储了 $N$ 个映射。
  • 递归栈的深度在最坏情况下（链状树）是 $O(N)$，平均是 $O(\log N)$。
  • 总空间 $O(N)$。

6. 初学者常见疑问 (Q&A)

  • Q: 为什么题目强调“没有重复元素”？
    • A: 如果有重复元素（例如两个 3），当我们在 inorder 中查找 3 时，就不知道哪个 3 才是当前的根节点了，无法唯一确定左右子树的边界。
  • Q: 必须用前序 + 中序吗？后序 + 中序行不行？
    • A: 行！
      • 前序：根在开头。
      • 后序：根在末尾 ([ 左... | 右... | 根 ])。
      • 只要有 中序，再加上 前序或后序 的任意一个，就能还原树。
      • 但是：前序 + 后序 是无法唯一还原二叉树的（除非是满二叉树），因为无法区分左右子树。
  • Q: 下标计算容易晕怎么办？
    • A: 记住一个核心公式：
      • 左子树大小 size = root_idx - in_start。
      • 前序切分永远是：[根], [start+1, start+size], [start+size+1, end]。
      • 中序切分永远是：[start, root_idx-1], [root], [root_idx+1, end]。
