138. 随机链表的复制

题目：
给你一个长度为 n 的链表，每个节点除了 next 指针外，还有一个 random 指针指向链表中的任意节点或者 null。

请你返回这个链表的深拷贝。

示例 1：
输入：head = [[7,null],[13,0],[11,4],[10,2],[1,0]]
输出：[[7,null],[13,0],[11,4],[10,2],[1,0]]

示例 2：
输入：head = [[1,1],[2,1]]
输出：[[1,1],[2,1]]

示例 3：
输入：head = [[3,null],[3,0],[3,null]]
输出：[[3,null],[3,0],[3,null]]

提示：

  • 0 <= n <= 1000
  • -10⁴ <= Node.val <= 10⁴
  • Node.random 是指向链表中任意节点或者 null

这道题是链表问题中非常独特的一道 “深拷贝” (Deep Copy) 题目。

它考察的不是复杂的算法逻辑，而是你对 数据结构内存模型 的理解，以及如何巧妙地利用 指针映射 关系。

面试中，这道题通常有两种解法：

  1. 哈希表法：易于理解，空间复杂度 $O(N)$。
  2. 拼接（交织）法：技巧性极强，空间复杂度 $O(1)$（最优解）。

我们将重点讲解 拼接法，这是能让面试官眼前一亮的解法。

1. 明确问题描述

题目名称：随机链表的复制 (Copy List with Random Pointer)

题目大意：

给你一个长度为 n 的链表，每个节点包含一个额外指针 random，该指针可以指向链表中的任何节点或空节点。

请你 深拷贝 这个链表。

  • 深拷贝 意味着你需要创建全新的节点，新节点的 next 和 random 指针必须指向新的对应节点，而不能指向原链表的节点。

示例：

输入：head = [[7,null],[13,0],[11,4],[10,2],[1,0]]

（[val, random_index]，比如 13 的 random 指向下标 0 的节点 7）

输出：一个新的链表，结构和值完全一样，但内存地址完全不同。

2. 核心挑战

为什么不能直接遍历复制？

当我们复制节点 A 时，它的 random 指针可能指向节点 Z。

  • 此时节点 Z 可能还没有被创建出来（在链表后面）。

  • 或者 Z 已经被创建了，但我们手里没有新节点 Z' 的地址，只有旧节点 Z 的地址。

    我们需要建立“旧节点”到“新节点”的一一对应关系。

3. 解法一：哈希表法 (Hash Map) —— 直观思路

这是最符合人类直觉的做法。

  1. 第一遍遍历：
    • 只管创建新节点。
    • 建立一个哈希表 unordered_map<Node*, Node*> map。
    • 存入映射：map[旧节点 A] = 新节点 A'。
  2. 第二遍遍历：
    • 再次遍历链表。
    • 配置指针：map[curr]->next = map[curr->next]。
    • 配置随机：map[curr]->random = map[curr->random]。
    • 因为所有新节点在第一遍都创建好了，所以这里直接查表就能拿到地址。
  • 优点：代码简单，逻辑清晰。
  • 缺点：空间复杂度 $O(N)$。

4. 解法二：拼接（交织）法 —— 空间优化 $O(1)$

如果我们不想用哈希表，怎么找到 旧 A 对应的 新 A' 呢？

我们可以利用链表本身的结构，把新节点 “藏” 在旧节点的旁边！

核心三步走：

Step 1: 复制节点并拼接 (Clone & Splicing)

遍历链表，为每个节点 A 创建副本 A'，并将 A' 插入到 A 和 B 之间。

  • 原链表：A -> B -> C
  • 变形后：A -> A' -> B -> B' -> C -> C'

此时，A->next 就是 A'。我们可以通过 旧节点->next 直接找到它对应的 新节点，不再需要哈希表。

Step 2: 复制随机指针 (Copy Random)

遍历链表（注意步长是 2，只看旧节点）。

  • A 的 random 指向 C。

  • 那么 A' (A->next) 的 random 应该指向 C'。

  • C' 在哪里？它就在 C 的后面，也就是 C->next！

  • 公式：curr->next->random = curr->random->next。

    (注意判空：如果 curr->random 是 NULL，那么 curr->next->random 也是 NULL)

Step 3: 拆分链表 (Unweaving)

把交织在一起的长链表拆回两个独立的链表。

  • 恢复原链表：A -> B -> C
  • 提取新链表：A' -> B' -> C'

6. 复杂度分析

时间复杂度：$O(N)$

  • 我们遍历了链表三次（拼接、连 Random、拆分）。
  • $3N$ 依然是线性时间。

空间复杂度：$O(1)$

  • 除了用于结果的新节点（这是必须的，不计入额外空间），我们只用了几个指针变量。
  • 我们没有使用哈希表来存储映射关系，利用了链表自身的结构特性完成了“映射”。

7. 初学者常见疑问 (Q&A)

  • Q: 为什么 Step 3 拆分时 curr->next = nextOriginal 很重要？
    • A: 题目通常要求“深拷贝”，不仅要返回新链表，原链表也不能被破坏。如果你不把原链表恢复成 A->B->C 的样子，虽然答案对了，但在工程上这叫“有副作用 (Side Effect)”，通常是不合格的。
  • Q: Step 2 为什么要判断 curr->random != nullptr？
    • A: 因为题目说 random 可能指向 null。如果 curr->random 是空，去访问 curr->random->next 就会报空指针异常 (Segmentation Fault)。
