230. 二叉搜索树中第 K 小的元素

题目：
给定一个二叉搜索树的根节点 root 和一个整数 k，请你返回树中第 k 小的值。

示例 1：
输入：root = [3,1,4,null,2], k = 1
输出：1

示例 2：
输入：root = [5,3,6,2,4,null,null,1], k = 3
输出：3

提示：

  • 树中节点数目范围是 [1, 10⁴]
  • 0 <= k <= 树中节点总数
  • 1 <= Node.val <= 10⁴

这道题是 二叉搜索树 (BST) 性质的教科书级应用。

它是力扣（LeetCode）第 230 题。

这道题不需要你发明什么新算法，它只需要你记住一句话：

二叉搜索树的中序遍历 (In-order Traversal) 结果是一个有序数组。

只要掌握了这句话，这道题瞬间就变成了：“在一个有序数组中找第 K 个元素”。

1. 明确问题描述

题目名称：二叉搜索树中第 K 小的元素 (Kth Smallest Element in a BST)

题目大意：

给定一个二叉搜索树的根节点 root 和一个整数 k ，请你设计一个算法查找其中第 k 个最小的元素（从 1 开始计数）。

示例：

Plaintext

输入：root = [3, 1, 4, null, 2], k = 1

输出：1

输入：同上，k = 3

输出：3 (顺序是 1, 2, 3, 4，第 3 个是 3)

2. 核心思路分析

2.1 笨办法：存下来再找

如果我们对这棵树进行一次完整的 中序遍历，并将结果存入一个数组 vector<int>。

  • 树：[5, 3, 6, 2, 4]
  • 中序遍历后：[2, 3, 4, 5, 6]
  • 第 k 小就是 arr[k-1]。

缺点：你需要遍历整棵树，并且需要 $O(N)$ 的额外空间来存数组。如果树有 1 亿个节点，内存就爆了。

2.2 聪明办法：边走边数 (Early Stop)

我们不需要把所有数字都存下来。

既然中序遍历是按顺序访问的（第一个访问的一定是最小的，第二个访问的是第二小的…），我们只需要 搞一个计数器。

算法流程：

  1. 开始中序遍历（左 -> 根 -> 右）。
  2. 每访问一个节点，计数器 k 减 1（或者 count 加 1）。
  3. 当 k 减到 0 时，说明当前访问的这个节点就是我们要找的第 K 小元素。
  4. 立刻停下来，直接返回结果，后面的节点（右边的大数）都不用看了。

3. C++ 代码实现 (迭代法 - 推荐)

为了能够“立刻停下来”，使用 迭代法（栈模拟） 是控制力最强、最容易理解的方式。这就用到了我们之前学的 “二叉树中序遍历” 的模板。

4. 复杂度分析

时间复杂度：$O(H + k)$

  • $H$ 是树的高度。我们需要先钻到最左下的节点（耗时 $O(H)$）。
  • 然后我们只需要遍历 $k$ 个节点就可以停下来了。
  • 不需要遍历整棵树 $O(N)$。

空间复杂度：$O(H)$

  • 栈的大小取决于树的高度。

5. 进阶面试题 (Follow-up)

面试官常问：

“如果这个 BST 会被频繁地修改（插入/删除操作），并且我们需要频繁地查找第 K 小的值，你会怎么优化？”

现有解法的问题：

每次查找都要重新遍历一遍，时间复杂度是 $O(k)$。如果 $k$ 很大（接近 $N$），就很慢。

优化解法：记录子树大小 (Subtree Size)

我们可以在定义 TreeNode 时多加一个字段 size，记录 以该节点为根的子树中节点的总数。

查找逻辑（类似二分查找）：

假设当前站在 root 节点：

  1. 左子树的大小为 left_size。
  2. 如果 k == left_size + 1：恭喜，root 就是第 k 小。
  3. 如果 k <= left_size：说明目标在左边，去左子树找第 k 小。
  4. 如果 k > left_size + 1：说明目标在右边，去右子树找第 k - (left_size + 1) 小。

复杂度变化：

  • 查找：优化为 $O(H)$（也就是 $O(\log N)$），不用遍历了，直接走路径。
  • 插入/删除：虽然需要更新路径上节点的 size，但复杂度依然保持 $O(H)$。

6. 初学者常见疑问 (Q&A)

  • Q: 用递归能写吗？
    • A: 可以。需要用一个全局变量（或者引用传参）来记录 count。但递归不好“直接中断”（除非抛异常或者设置全局 flag），大部分人写递归都会不小心遍历完整个树。迭代法在控制“提前退出”上更直观。
  • Q: 这里的 k 是从 0 开始还是从 1 开始？
    • A: 题目通常规定是从 1 开始。如果是从 0 开始，逻辑里改成 k-- 后判断 k == -1 或者初始判断 k == 0 即可。
