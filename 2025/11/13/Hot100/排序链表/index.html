148. 排序链表

题目：
给你链表的头节点 head，请将其按升序排列并返回排序后的链表。

示例 1：
输入：head = [4,2,1,3]
输出：[1,2,3,4]

示例 2：
输入：head = [-1,5,3,4,0]
输出：[-1,0,3,4,5]

示例 3：
输入：head = []
输出：[]

提示：

  • 链表中节点的数目范围是 [0, 5 * 10⁴]
  • -5 * 10⁴ <= Node.val <= 5 * 10⁴

这道题是链表排序问题的“终极考核”。

题目要求对链表进行排序，且要求 时间复杂度 $O(n \log n)$。

这直接就把冒泡排序、插入排序（$O(n^2)$）给否定了。在 $O(n \log n)$ 级别的排序算法中（快排、堆排、归并），归并排序 (Merge Sort) 是最适合链表的。

为什么？

  1. 随机访问难：快排依赖下标随机访问（找 pivot），链表做不到。
  2. 内存不连续：堆排需要构建二叉堆，通常依赖数组结构。
  3. 指针优势：归并排序的核心是“合并”，链表合并只需要改变指针，不需要像数组那样搬移数据，非常高效。

这道题其实是你之前学过的 “找中点” + “合并两个有序链表” 的组合拳。

1. 明确问题描述

题目名称：排序链表 (Sort List)

题目大意：

给你链表的头结点 head，请将其按 升序 排列并返回排序后的链表。

示例：

输入：4 -> 2 -> 1 -> 3

输出：1 -> 2 -> 3 -> 4

2. 核心思路分析：归并排序 (Divide and Conquer)

归并排序的思想是 “分而治之”：

  1. 分 (Divide)：把链表从中间一分为二，断开。
  2. 治 (Conquer)：递归地对左半部分和右半部分分别进行排序。
  3. 合 (Merge)：将两个已经排好序的子链表，合并成一个有序链表。

具体步骤拆解：

  1. 找中点（砍一刀）：
    • 使用 快慢指针。快指针一次走两步，慢指针一次走一步。
    • 当快指针走到头时，慢指针刚好在中间。
    • 关键点：我们需要把链表断开 (slow->next = nullptr)，分成 head 开头的左链表和 mid 开头的右链表。
  2. 递归排序：
    • left = sortList(leftHead)
    • right = sortList(rightHead)
    • 递归出口：当链表为空或只有一个节点时，默认有序，直接返回。
  3. 合并（缝合）：
    • 这正是我们之前讲过的 “合并两个有序链表” (LeetCode 21)。
    • 使用一个虚拟头节点 (dummy)，谁小接谁。

4. 复杂度分析

时间复杂度：$O(N \log N)$

  • 分 (Divide)：每次找中点遍历 $N/2$, $N/4$… 总共 $\log N$ 层。
  • 合 (Merge)：每一层所有的节点都会被遍历一次进行合并，耗时 $O(N)$。
  • 总耗时：$O(N \log N)$。

空间复杂度：$O(\log N)$

  • 这里使用的是 自顶向下 (Top-Down) 的递归方法。
  • 递归调用栈的深度是 $\log N$（因为每次二分）。
  • 注：虽然题目进阶要求 $O(1)$ 空间，但那需要使用“自底向上”的迭代法，代码极其复杂且可读性差，面试中通常写出递归版本并能解释空间主要消耗在栈上即可过关。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么找中点时 fast 要从 head->next 开始？
    • A: 这是一个处理由偶数个节点组成的链表时的细节。
    • 如果链表是 [1, 2]。
      • 若 fast = head: 循环结束时 slow 会停在 2。此时左半部分是 [1, 2]，右半部分是 nullptr。递归左半部分会死循环（永远切不出更小的）。
      • 若 fast = head->next: 循环结束时 slow 停在 1。mid 是 2。切断后左边 [1]，右边 [2]。成功分割。
  • Q: 链表可以用快速排序吗？
    • A: 可以，但很麻烦。
    • 你需要选取 Pivot，然后通过交换 值 (Value) 或者调整指针把比 Pivot 小的放左边，大的放右边。
    • 但是因为无法像数组那样随机选取 Pivot，如果链表本身是有序的，快排会退化成 $O(N^2)$。归并排序则稳定在 $O(N \log N)$。
