**进阶：**如果你已经实现复杂度为 O(n) 的解法，尝试使用更为精妙的 分治法 求解。

这道题是 动态规划 (Dynamic Programming, DP) 的入门必修课。

它的官方名称叫 最大子数组和 (Maximum Subarray)，但在算法圈子里，它有一个更响亮的名字 —— Kadane 算法。

这道题之所以经典，是因为它教会了我们如何只遍历一遍数组，就做出每一步的“最优选择”，从而推导出全局的“最大结果”。

1. 明确问题描述

题目名称：最大子数组和 (Maximum Subarray)

题目大意：

给你一个整数数组 nums，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），并返回其最大和。

关键约束：

  • 连续：必须是连在一起的一段。
  • 含负数：数组里可能有负数，这增加了难度（如果是全正数，把整个数组加起来就是最大的了）。
  • 复杂度：要求 $O(n)$ 时间复杂度。

示例：

输入：nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

输出：6

解释：连续子数组 [4, -1, 2, 1] 的和是 6，是最大的。

2. 核心思路分析

2.1 暴力法的瓶颈

如果用暴力法，我们需要枚举所有的起点和终点，复杂度是 $O(n^2)$。这显然不够好。

2.2 核心思想：贪心 / 动态规划

想象你在玩一个累加游戏：

你走在数组上，每一步你都要决定：“我是要把当前的数字加到之前的累加和里？还是扔掉之前的累加和，从当前这个数字重新开始算？”

决策逻辑 (The “Why”)：

设 pre 为 “以当前数字结尾的最大子数组和”。

当我们遍历到数字 x 时：

  1. 如果之前的累加和 (pre) 是正数：
    • 太好了！正数对我有增益。
    • 虽然 x 可能是负数，但我加上之前的正数，总比 x 孤零零一个人要大。
    • 决策：加入之前的队伍 (pre + x)。
  2. 如果之前的累加和 (pre) 是负数（或者 0）：
    • 之前的和是负的，它是“拖油瓶”。
    • 不管 x 是几，x 加上一个负数，肯定比 x 自己还要小。
    • 决策：果断抛弃之前的队伍，另起炉灶，自己做开头 (x)。

状态转移方程：

$$\text{pre} = \max(\text{nums}[i], \text{pre} + \text{nums}[i])$$

图解推导 [-2, 1, -3, 4, ...]：

  1. 遇到 -2：之前没数。pre = -2。最大值 max = -2。
  2. 遇到 1：之前的 pre 是 -2 (拖油瓶)。抛弃！pre 变成 1。最大值 max = 1。
  3. 遇到 -3：之前的 pre 是 1 (有益)。加入！pre 变成 1 + (-3) = -2。最大值 max 保持 1。
  4. 遇到 4：之前的 pre 是 -2 (拖油瓶)。抛弃！pre 变成 4。最大值 max 更新为 4。
  5. …以此类推。

4. 复杂度分析

时间复杂度：$O(n)$

  • 我们只遍历了一次数组。
  • 内部的操作全是 $O(1)$ 的加法和比较。

空间复杂度：$O(1)$

  • 经典的动态规划通常需要一个 dp 数组（空间 $O(n)$），记录每一个位置的结果。
  • 但我们发现，计算当前位置 dp[i] 时，只需要知道前一个位置 dp[i-1] 就行了。之前的历史不需要保留。
  • 这种优化技巧叫做 “滚动数组” 思想（虽然这里只滚了一个变量），将空间降到了 $O(1)$。

5. 初学者常见疑问 (Q&A)

  • Q: 如果数组全是负数怎么办？比如 [-2, -5, -1]。
    • A: 算法依然有效。
      • 遇到 -2：pre = -2, max = -2。
      • 遇到 -5：pre = max(-5, -2 + -5) = -5 (自立门户), max 还是 -2。
      • 遇到 -1：pre = max(-1, -5 + -1) = -1 (自立门户), max 更新为 -1。
      • 结果返回 -1。逻辑完美。
  • Q: 为什么和滑动窗口不一样？
    • A: 这是一个极好的问题！
    • 滑动窗口通常用于处理“窗口大小可变但具有单调性”或者“窗口大小固定”的问题。
    • 因为这里有负数，窗口扩大时，和不一定增大；缩小时，和不一定减小。单调性被破坏了，所以标准的双指针滑动窗口失效。
    • Kadane 算法本质上是一种动态规划，它关注的是“状态转移”。
  • Q: 这道题能输出具体的子数组（起始和结束位置）吗？
    • A: 可以。你需要额外的变量来记录。
      • 当 pre 重置（x > pre + x）时，记录新的 temp_start = i。
      • 当更新 maxAns 时，记录 final_start = temp_start 和 final_end = i。
