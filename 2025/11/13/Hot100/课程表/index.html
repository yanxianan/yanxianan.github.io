207. 课程表

题目：
你总共需要修 numCourses 门课，课程编号从 0 到 numCourses - 1。

给你一个数组 prerequisites，其中 prerequisites[i] = [ai, bi] 表示你要学习课程 ai，必须先完成课程 bi。

判断是否可能完成所有课程的学习。

示例 1：
输入：numCourses = 2, prerequisites = [[1,0]]
输出：true
解释：总共有 2 门课程，学习课程 1 之前必须完成课程 0。所以可以完成所有课程。

示例 2：
输入：numCourses = 2, prerequisites = [[1,0],[0,1]]
输出：false
解释：总共有 2 门课程，学习课程 1 之前必须完成课程 0，而学习课程 0 之前又必须完成课程 1。因此无法完成所有课程。

提示：

  • 1 <= numCourses <= 2000
  • 0 <= prerequisites.length <= 5000
  • prerequisites[i].length == 2
  • 0 <= ai, bi < numCourses
  • ai != bi

这道题是 图论 (Graph Theory) 中 “拓扑排序 (Topological Sort)” 的入门必修课。

它是力扣（LeetCode）第 207 题。

这道题不再是像“岛屿数量”那样在网格里乱跑，而是解决现实生活中最常见的 “依赖关系” (Dependency) 问题。

比如：修高数 II 之前必须修高数 I；安装 React 之前必须安装 Node.js。

如果你能解决这道题，你就掌握了 “有向无环图 (DAG)” 和 “环检测 (Cycle Detection)” 的核心。

1. 明确问题描述

题目名称：课程表 (Course Schedule)

题目大意：

你必须选修 numCourses 门课程，记为 0 到 numCourses - 1。

在选修某些课程之前需要一些先修课程。

给你一个数组 prerequisites，其中 prerequisites[i] = [ai, bi] ，表示想要学习课程 ai ，你需要先完成课程 bi。

  • 也就是存在一条有向边：$b_i \rightarrow a_i$。

请你判断是否可能完成所有课程的学习？

示例：

输入：numCourses = 2, prerequisites = [[1,0]]

输出：true

解释：想修课程 1，必须先修 0。完全没问题，先 0 后 1。

输入：numCourses = 2, prerequisites = [[1,0], [0,1]]

输出：false

解释：想修 1 得先修 0，想修 0 得先修 1。

死锁了（形成了环），谁都修不了。

2. 核心思路：有向图找环

这个问题本质上就是：判断这个有向图中是否存在“环” (Cycle)？

  • 如果有环 $\rightarrow$ 没法修完（False）。
  • 如果没环 $\rightarrow$ 这就是一个 DAG (Directed Acyclic Graph)，可以修完（True）。

我们有两种经典的解法：BFS (Kahn 算法) 和 DFS (三色标记法)。

面试中 BFS (入度表法) 最容易理解，也是首选。

解法一：BFS (入度表法 / Kahn 算法)

生活常识：

如果你想修一门课，前提是什么？

前提是它的“前置课程”都已经修完了。

换句话说，如果一门课 没有任何前置课程（或者前置都搞定了），那这门课就是 “入度为 0” 的，你可以直接去上。

算法流程：

  1. 建图：用邻接表 (adj) 存储图结构，同时维护一个 入度数组 (indegree)。
    • indegree[i] 表示课程 i 还有几门课没修。
  2. 找起点：把所有 入度为 0 的课程（不需要先修的课）扔进 队列。
  3. BFS 拆课：
    • 从队列里拿出一门课 u（这门课我修了！）。
    • 计数器 count++。
    • 找到 u 的所有后修课程 v（也就是 u -> v）。
    • 告诉 v：“你的前置 u 我搞定了，你的入度减 1”。
    • 关键：如果 v 的入度减到了 0，说明 v 的所有前置都修完了，把 v 也扔进队列。
  4. 判断：如果最后修完的课程数 count 等于总课程数 numCourses，说明没环，成功！

4. 解法二：DFS (三色标记法)

DFS 找环的逻辑稍微绕一点。我们需要给每个节点打标记，防止死循环。

我们定义三种状态：

  1. 0 (未访问)：这门课还没检查过。
  2. 1 (正在访问)：正在检查这门课的后续依赖链。如果在 DFS 过程中又遇到了状态 1 的节点，说明遇到了自己（形成环了）！
  3. 2 (已完成)：这门课及其所有后续都检查过了，没问题，是安全的。

算法流程：

对每门课发起 DFS。

  • 如果遇到 1 $\rightarrow$ 有环，返回 False。
  • 如果遇到 2 $\rightarrow$ 安全，直接返回 True。
  • 标记为 1，继续 DFS 后续。
  • DFS 结束，标记为 2。

5. 复杂度分析

时间复杂度：$O(V + E)$

  • $V$ 是课程数，$E$ 是先修关系数。
  • BFS: 我们初始化图遍历了所有的边 ($E$)，然后每个节点入队出队一次 ($V$)。
  • DFS: 每个节点最多被访问一次（状态变了就不进去了）。

空间复杂度：$O(V + E)$

  • 邻接表需要存储所有的点和边。

6. 初学者常见疑问 (Q&A)

  • Q: 为什么 BFS 这种方法叫“拓扑排序”？
    • A: 因为它不仅能判断是否有环，还能顺便生成一个 线性序列。
    • 如果你把出队的 u 按顺序存进一个数组，那个数组就是修课的顺序（例如：先修 Math 101，再修 Math 102）。
    • 这就是力扣 210 题 “课程表 II” 的解法（只需改一行代码，把 count++ 换成 res.push_back(u)）。
  • Q: 为什么 DFS 需要 3 种状态？用布尔值 visited 不行吗？
    • A: 不行。
    • 普通的 visited 只能表示“去过”。
    • 但我们需要区分 “这趟递归还没结束（正在栈里）” 和 “以前那趟递归已经查完了”。
    • 如果遇到“以前查完了”的节点（状态 2），那是交叉路径（菱形结构），是合法的。
    • 只有遇到“正在栈里”的节点（状态 1），才是回边，才意味着有环。
