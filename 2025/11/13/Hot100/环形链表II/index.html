142. 环形链表 II

题目：
给定一个链表的头节点 head，返回链表开始入环的第一个节点。如果链表无环，则返回 null。

为了表示给定链表中的环，使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos == -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。

示例 1：
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，其尾部连接到第二个节点。

示例 2：
输入：head = [1,2], pos = 0
输出：返回索引为 0 的链表节点
解释：链表中有一个环，其尾部连接到第一个节点。

示例 3：
输入：head = [1], pos = -1
输出：null
解释：链表中没有环。

提示：

  • 链表中节点的数目范围是 [0, 10⁴]
  • -10⁵ <= Node.val <= 10⁵
  • pos 为 -1 或者链表中的某个有效索引

这道题是“环形链表 I”的完美升华。

如果说第一题是在问“有没有鬼”，那这道题就是在问“鬼门关在哪里”。

解决这道题不需要新的数据结构，需要的只是一点点小学奥数级别的推导。一旦理解了那个数学关系，代码写起来比第一题还简单。

1. 明确问题描述

题目名称：环形链表 II (Linked List Cycle II)

题目大意：

给定一个链表的头节点 head，如果链表中存在环，返回链表开始入环的第一个节点（即环的入口）；如果链表无环，返回 null。

关键约束：

  • 不允许修改链表。
  • 空间复杂度要求 $O(1)$。

示例：

输入：3 -> 2 -> 0 -> -4，-4 指回 2。

输出：节点 2。

解释：节点 2 是环的入口，-4 是环里的最后一个节点，它指向了入口。

2. 核心思路分析

2.1 为什么不能直接看出来？

当我们用快慢指针 (slow, fast) 相遇时，相遇点通常不是环的入口。

相遇点可能在环里的任何位置（取决于环的大小和入环前的直线长度）。我们需要一种方法，从这个“随机”的相遇点，反推出“入口点”。

2.2 核心策略：数学推导 (The Math Magic)

我们把链表分为三段距离（看下图辅助理解）：

  1. $a$：从头节点 head 到环入口 entry 的距离（直线部分）。
  2. $b$：从环入口 entry 到快慢指针相遇点 meet 的距离。
  3. $c$：从相遇点 meet 继续走回到环入口 entry 的距离。

推导过程：

  1. 相遇时走了多远？
    • Slow (乌龟) 走了：$a + b$
    • Fast (兔子) 走了：$a + b + n(b + c)$
      • (解释：兔子跑得快，在乌龟进环之前，兔子已经在圈里跑了 n 圈了，最后在 b 处追上乌龟)。
  2. 速度关系：
    • 兔子的速度是乌龟的 2 倍。所以路程也是 2 倍。
    • $2 \times (a + b) = a + b + n(b + c)$
  3. 化简方程（高光时刻）：
    • $2a + 2b = a + b + n(b + c)$
    • 移项得：$a + b = n(b + c)$
    • $a = n(b + c) - b$
    • 我们知道 $(b + c)$ 就是环的长度 $L$。
    • $a = (n-1)L + (L - b)$
    • 看图可知，$L - b$ 其实就是 $c$（剩下的距离）。
    • 结论：$a = (n-1)L + c$

人话翻译结论：

$$a = c$$

(忽略圈数 $n$，因为转圈不影响最终位置)

这意味着：从“头节点”走到“入口”的距离 ($a$)，竟然等于从“相遇点”继续往前走到“入口”的距离 ($c$)！

2.3 算法流程

基于上面的结论，算法变得异常简单：

  1. 第一阶段：也就是上一题的逻辑。用快慢指针判断是否有环。如果没环，直接返回 NULL。
  2. 第二阶段：如果相遇了（找到了 meet 点）：
    • 把其中一个指针（比如 fast）扔回 链表头部 (head)。
    • 另一个指针（slow）保持在 相遇点 (meet) 不动。
    • 现在，两个指针同时开始走，而且每次都只走 1 步。
  3. 相遇即入口：
    • 根据 $a = c$，它们一定会在走了 $a$ 步之后，在环入口相遇。

4. 复杂度分析

时间复杂度：$O(N)$

  • 判断环的过程是 $O(N)$。
  • 寻找入口的过程，指针最多走 $a$ 步（直线长度），也是 $O(N)$。
  • 总体依然是线性的。

空间复杂度：$O(1)$

  • 我们只复用了 slow 和 fast 指针，没有申请额外空间。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么相遇时，乌龟一定没跑完一圈？
    • A: 这是 Floyd 算法的一个特性。当乌龟刚进环时，兔子已经在环里了。兔子追乌龟，距离最多就是一圈。因为兔子速度是乌龟的2倍，所以在乌龟跑完一圈之前，兔子绝对能追上它。所以公式里乌龟的路程写 $a+b$ 是对的（不需要加 $n$ 圈）。
  • Q: 如果链表就是一个大环（头节点就在环里），算法还有效吗？
    • A: 有效。此时 $a=0$。
    • 相遇点就是起点（或者起点附近的某点）。
    • fast 回到 head，slow 在相遇点。
    • 循环一开始 while (slow != fast) 就会发现它俩已经重合了，直接返回 slow（也就是 head）。逻辑自洽。
