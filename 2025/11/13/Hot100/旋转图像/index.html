48. 旋转图像

题目：
给你一个 n x n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在原地旋转图像，这意味着你需要直接修改输入的二维矩阵，不要使用另一个矩阵。

示例 1：
输入：matrix = [[1,2,3],[4,5,6],[7,8,9]]
输出：[[7,4,1],[8,5,2],[9,6,3]]

示例 2：
输入：matrix = [[5,1,9,11],[2,4,8,10],[13,3,6,7],[15,14,12,16]]
输出：[[15,13,2,5],[14,3,4,1],[12,6,8,9],[16,7,10,11]]

提示：

  • n == matrix.length == matrix[i].length
  • 1 <= n <= 20
  • -1000 <= matrix[i][j] <= 1000

这道题是力扣（LeetCode）第 48 题 “旋转图像” (Rotate Image)。

这道题在面试中出现的频率极高，因为它考察的不是复杂的算法，而是你对 二维数组下标 的变换能力，以及数学思维的敏捷度。

很多初学者试图用“找规律”的方法直接推导旋转后的坐标公式，结果往往因为下标越界或逻辑复杂而写出一堆 Bug。今天我教你一个最优雅、最容易记住的方法——“折叠 + 翻转”法。

1. 明确问题描述

题目名称：旋转图像 (Rotate Image)

题目大意：

给定一个 $n \times n$ 的二维矩阵 matrix（代表图像）。请你将图像 顺时针 旋转 90 度。

关键约束：

  • 必须 原地 旋转。
  • 这意味着你不能新建一个 $n \times n$ 的矩阵来存储旋转后的结果，必须在原矩阵上修改，空间复杂度只能是 $O(1)$。

2. 核心思路分析

2.1 为什么直接找规律很难？

如果要直接把一个点 (i, j) 移动到旋转后的位置，公式是 (j, n-1-i)。

在原地修改时，如果你把 (0,0) 的值直接放到目标位置 (0,2)，那 (0,2) 原本的值就会被覆盖丢失。

为了解决这个问题，你必须同时交换 4 个点（四个角转圈圈），代码写起来会有很多 i, j, n-1-i, n-1-j 的嵌套，非常容易晕。

2.2 核心策略：数学变换（转置 + 镜像）

与其痛苦地推导旋转坐标，不如利用线性代数的知识。

顺时针旋转 90 度 可以拆解为两个简单的动作：

  1. 第一步：转置 (Transpose)
    • 沿着主对角线（从左上角到右下角）对折矩阵。
    • 也就是交换 matrix[i][j] 和 matrix[j][i]。
  2. 第二步：水平镜像翻转 (Reverse Rows)
    • 把每一行倒序排列（左右翻转）。

4. 复杂度分析

时间复杂度：$O(N^2)$

  • 转置过程：我们需要遍历矩阵的大约一半元素，操作次数是 $\frac{N^2}{2}$。
  • 翻转过程：我们需要遍历整个矩阵的每一个元素进行交换，操作次数是 $\frac{N^2}{2}$。
  • 总时间复杂度是 $O(N^2)$。
  • 对于二维矩阵问题，我们需要访问每个元素至少一次，所以 $O(N^2)$ 已经是理论最优解。

空间复杂度：$O(1)$

  • 我们在原矩阵上进行 swap 操作，只使用了常数个额外变量。
  • 没有申请新的 $N \times N$ 矩阵。

5. 初学者常见疑问 (Q&A)

  • Q: 如果题目要求“逆时针”旋转 90 度怎么办？
    • A: 依然可以用两次变换，只是顺序变一下：
      • 顺时针 = 先转置，再左右翻转（翻转行）。
      • 逆时针 = 先转置，再上下翻转（交换行），或者先左右翻转再转置。
      • 记忆技巧：拿张纸画一下就出来了，不用死记。
  • Q: 转置时 j 为什么要从 i (或 i+1) 开始？
    • A: 这是转置操作最容易写错的地方。
    • 我们的目标是交换主对角线两侧的元素。
    • 如果你让 j 从 0 开始，那么 (0, 1) 和 (1, 0) 会被交换两次（复原了）。
    • 让 j 从 i 开始，相当于我们只遍历了矩阵的上三角部分，把它和下三角部分交换，这样每个配对只会被处理一次。
