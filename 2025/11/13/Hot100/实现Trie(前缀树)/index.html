208. 实现 Trie (前缀树)

题目：
实现一个 Trie（前缀树），支持以下操作：

  • Trie()：初始化 Trie 对象。
  • void insert(String word)：将字符串插入 Trie。
  • boolean search(String word)：如果字符串完全匹配某个单词，则返回 true。
  • boolean startsWith(String prefix)：如果某个单词以给定前缀开头，则返回 true。

示例：
输入：
[“Trie”, “insert”, “search”, “search”, “startsWith”, “insert”, “search”]
[[], [“apple”], [“apple”], [“app”], [“app”], [“app”], [“app”]]
输出：
[null, null, true, false, true, null, true]

提示：

  • 1 <= word.length, prefix.length <= 2000
  • word 和 prefix 仅由小写英文字母组成
  • 最多调用 insert、search 和 startsWith 方法 3 * 10⁴ 次

这道题是 数据结构 (Data Structure) 中的明星题目。

它是力扣（LeetCode）第 208 题 “实现 Trie (前缀树)”。

Trie（读音类似 “try”）是一种专门用于处理字符串的树形数据结构。它在 自动补全 (Autocomplete)、拼写检查、IP 路由 等场景中有着极其广泛的应用。

面试官考这道题，不是考你算法逻辑（因为它逻辑很简单），而是考你 “如何设计一个数据结构” 以及 “指针操作”。

1. 什么是 Trie（前缀树）？

Trie 的核心思想是 “空间换时间”，利用字符串的 公共前缀 来减少查询时间，最大限度地减少无谓的字符串比较。

Trie 的结构特点：

  1. 根节点 (Root) 不包含字符，除根节点外每一个节点都只包含一个字符。
  2. 从根节点到某一节点，路径上经过的字符连接起来，为该节点对应的字符串。
  3. 每个节点的所有子节点包含的字符都不相同。

举个例子：

假设我们要存入单词 "apple", "app", "apply", "bat"。

  • root -> a -> p -> p (这里标记为 结束，因为 “app” 是一个词) -> l -> e (标记为 结束)。
  • 同时在 p 下面分叉 -> l -> y (标记为 结束)。
  • root -> b -> a -> t (标记为 结束)。

2. 核心设计 (Class Design)

我们需要两个类：

  1. TrieNode (节点类)：
    • children：指向子节点的指针数组。通常题目限定只有小写字母 a-z，所以我们开一个大小为 26 的数组。
    • isEnd：布尔值。标记当前节点是否是一个单词的结尾。
      • 为什么需要这个？ 因为 “apple” 存进去了，”app” 也是路径的一部分，但如果只有 “apple” 是单词，那么走到 “app” 的那个 p 时，isEnd 应该是 false。
  2. Trie (树类)：
    • root：根节点。
    • 提供 insert、search、startsWith 三个方法。

4. 复杂度分析

时间复杂度

  • Insert: $O(L)$，其中 $L$ 是单词的长度。
  • Search: $O(L)$。
  • StartsWith: $O(L)$。
  • 对比: 如果用哈希表 (HashSet)，虽然平均是 $O(1)$，但在处理大量字符串的前缀匹配时，哈希表需要遍历所有 key，效率极低。Trie 此时是无敌的。

空间复杂度

  • $O(N \times 26 \times L)$：其中 $N$ 是单词数量。
  • 虽然 Trie 复用了公共前缀（比如 “apple” 和 “apply” 共享 “appl”），但每个节点都有一个长度 26 的数组，这在稀疏数据下会比较浪费空间。
  • 优化：工程中可以用 HashMap 代替数组来存 children，或者用压缩 Trie (Radix Tree) 来优化。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么不用 unordered_map<char, TrieNode\*> children？
    • A: 可以用，省空间，但稍微慢一点。
    • 在算法题中，通常明确说了是 a-z，用 vector 或数组 children[26] 访问速度最快（因为是直接偏移量访问内存），且代码写起来最简单。
  • Q: search 和 startsWith 的区别是什么？
    • A:
      • insert("apple")
      • search("app") -> 返回 false。因为 “app” 只是路径，没有被标记为单词结尾。
      • startsWith("app") -> 返回 true。只要路径存在即可。
  • Q: 析构函数（内存释放）呢？
    • A: 在面试或 LeetCode 中，通常不要求写析构函数（因为程序跑完内存就回收了）。但在实际工程中，必须递归删除所有节点以防止 内存泄漏 (Memory Leak)。
