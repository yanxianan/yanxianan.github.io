49. 字母异位词分组

题目：
给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

字母异位词 是由重新排列源单词的所有字母得到的一个新单词，且不区分大小写。所有源单词中的字母通常恰好只用一次。

这道题非常经典，因为它展示了哈希表最强大的功能之一：通过设计“键 (Key)”，将看似杂乱的数据归类。

1. 明确问题描述

题目名称：字母异位词分组 (Group Anagrams)

题目大意：
给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

什么是“字母异位词”？
就是由相同的字母按照不同顺序组成的单词。
比如：”eat”, “tea”, “ate” 是彼此的异位词，因为它们都包含 1个’a’, 1个’e’, 1个’t’。

示例：
输入：strs = ["eat", "tea", "tan", "ate", "nat", "bat"]

2. 核心思路分析

2.1 寻找“身份证” (The Canonical Form)

我们需要把一堆单词扔进哈希表里，让互为异位词的单词自动跑到同一个“桶”里。
这就面临一个问题：用什么作为哈希表的 Key（键）？

对于 “eat”, “tea”, “ate” 来说，它们虽然长得不一样，但本质是相同的。我们需要找到一个共性，作为它们的“身份证”。

最直观的方法：排序
如果我们将每个单词内部的字母按照字母表顺序排序：

  • “eat” 排序后变成 “aet”
  • “tea” 排序后变成 “aet”
  • “ate” 排序后变成 “aet”

破局点：
一旦排了序，所有的异位词都会变成完全一样的字符串！
我们可以把这个排序后的字符串作为哈希表的 Key。

2.2 数据结构选择

  • Key (键): 排序后的字符串 (String)。这代表了这一组异位词的“标准形式”。
  • Value (值): 一个字符串列表 (vector<string>)。用来存放原始的单词。
  • 容器: unordered_map<string, vector<string>>。

2.3 算法流程

  1. 遍历输入的每一个字符串 s。
  2. 创建一个临时变量 key = s，并将 key 进行内部排序（例如 “tea” 变成 “aet”）。
  3. 以这个 key 去哈希表中查找：
  • 不管有没有，直接把原始字符串 s 扔进 map[key] 这个列表里。
  1. 最后，把哈希表里所有的 value (即那些列表) 拿出来拼成结果返回。

4. 复杂度分析

假设数组中有 N个字符串，每个字符串的最大长度为K 。

**时间复杂度：*O（NKlogK）

  • 我们需要遍历 N个字符串。
  • 对于每个字符串，我们需要进行排序。排序长度为K的字符串需要O(KlogK)的时间。
  • 哈希表的插入操作平均是O(1)，但在最坏情况下（字符串很长），哈希计算和键比较需要O(K)。
  • 综合来看，主要瓶颈在排序上，所以是O（N*KlogK）。
  • 注：由于题目中K通常很小（比如 100 以内），这个复杂度是非常高效的。

空间复杂度：

  • 我们需要存储所有的字符串。
  • 哈希表中存了所有字符，结果列表里也存了所有字符。
  • 总体空间消耗与输入数据的总字符数成正比。

5. 进阶思考：能不能更快？(面试加分项)

虽然排序法O（NKlogK）已经足够通过题目，但如果k 非常大，排序可能会慢。有没有O(NK)的方法？

计数法 (Counting)
我们可以不排序，而是统计每个字母出现的次数。
比如 “aab” 可以表示为：a:2, b:1, c:0...。
我们可以把这个计数结果（比如一个长度 26 的数组或特定格式的字符串 "a:2,b:1..."）作为 Key。
这样就不需要排序了，只需要遍历一遍字符串即可。

但在 C++ 中，直接用 vector<int> 做 map 的 Key 稍微麻烦一点（因为 vector 默认没有哈希函数，需要自定义），或者需要把计数数组拼接成一个 unique 的字符串。对于初学者和大多数面试场景，排序法是最清晰、最不容易写出 Bug 的首选方案。
