128. 最长连续序列

提示：

  • 0 <= nums.length <= 10⁵
  • -10⁹ <= nums[i] <= 10⁹

它的难点不在于想出一个解法，而在于题目硬性规定了时间复杂度必须是 $O(n)$。这一限制直接判了“先排序再找”这种直观解法的“死刑”（因为排序最快也要 $O(n \log n)$）。

这道题是学习如何利用 HashSet (unordered_set) 进行 $O(1)$ 查找，以及如何通过剪枝（跳过不必要的计算）来优化算法的绝佳案例。

1. 明确问题描述

题目名称：最长连续序列 (Longest Consecutive Sequence)

题目大意：

给定一个未排序的整数数组 nums，找出数字连续的最长序列（不要求序列在原数组中连续，只要求数值连续），并返回该序列的长度。

关键约束：

  • 算法的时间复杂度必须是 $O(n)$。

示例：

输入：nums = [100, 4, 200, 1, 3, 2]

解释：

  1. 数组里有的数字是：1, 2, 3, 4, 100, 200
  2. 其中最长的连续序列是 [1, 2, 3, 4]。
  3. 输出长度：4。

2. 核心思路分析

2.1 为什么暴力法和排序法行不通？

  • 暴力枚举：对每个数 x，去数组里找 x+1, x+2… 每次查找要遍历数组，复杂度 $O(n^2)$。太慢。
  • 排序法：先 sort(nums)，然后线性扫描。虽然好写，但排序瓶颈是 $O(n \log n)$。不符合题目要求。

2.2 破局思维：哈希集合 + “找龙头”

我们需要一个数据结构，能让我们用 $O(1)$ 的时间快速判断“某个数字存不存在”。这显然是 unordered_set 的强项。

  1. 第一步：去重并存入集合

    把所有数字扔进 unordered_set。这样我们查找 x+1 是否存在只需要瞬间。

  2. 第二步：如何避免重复计算？（关键！）

    假设数组是 [1, 2, 3, 4, 5]。

    • 如果我们从 3 开始数，会数 3, 4, 5（长度3）。
    • 如果我们从 2 开始数，会数 2, 3, 4, 5（长度4）。
    • 如果我们从 1 开始数，会数 1, 2, 3, 4, 5（长度5）。

    你会发现：从 2 和 3 开始数完全是浪费时间，因为它们一定是 1 开头的序列的一部分。

    结论：我们只应该从序列的**起点（龙头）**开始数！

    怎么判断一个数 x 是不是“龙头”？

    很简单：只要看 x-1 是否存在。

    • 如果 x-1 存在，说明 x 前面还有数，x 不是龙头，直接跳过。
    • 如果 x-1 不存在，说明 x 是一个新序列的起点，开始向后计数 (x+1, x+2…)。

4. 复杂度分析 (这也是面试中最容易被问倒的地方)

很多初学者看到代码里有一个 for 循环，里面还套了一个 while 循环，第一反应就是：“这不是 $O(n^2)$ 吗？”

其实它是 $O(n)$。为什么？

  • 外层循环：虽然遍历了所有元素。
  • 内层循环 (while)：请注意 if (!num_set.count(num - 1)) 这个判断条件。
    • 这个条件保证了只有是序列起点的数字，才会进入 while 循环。
    • 比如序列 [1, 2, 3, 4]，只有 1 会进入 while 循环跑 4 次。2, 3, 4 在 if 判断时就被拦住了。
  • 全局视角：
    • 数组中的每个数字，最多被访问两次：
      1. 在 for 循环中被检查一次。
      2. 在 while 循环中作为序列的一部分被检查一次（如果它不是起点）。
    • 所以总操作次数是 $2n$，即线性复杂度 $O(n)$。

空间复杂度：$O(n)$

  • 我们需要一个 unordered_set 来存储数组中所有的数，空间消耗取决于 nums 的长度。

5. 易错点与边界情况

  1. 空数组：如果输入 []，代码中的 longestStreak 初始化为 0，直接返回 0，逻辑正确。
  2. 重复元素：题目说未排序数组，可能包含重复数字（如 [1, 2, 0, 1]）。unordered_set 自动帮我们去重了，所以重复元素不会影响连续序列长度的计算（序列长度看的是数值，不是个数）。
  3. 整数溢出：虽然题目一般给 int，但在 while 循环中 currentNum + 1 理论上可能溢出。但在 LeetCode 的测试用例约束下通常不需要处理 long long，如果是在非常严格的比赛中，可以将 currentNum 设为 long long 或先检查 INT_MAX。
