114. 二叉树展开为链表

题目：
给你二叉树的根节点 root，展开成一个单链表：

  • 单链表中的节点顺序应与二叉树中使用 前序遍历 得到的顺序相同。
  • 每个节点的左子指针应指向 null。
  • 每个节点的右子指针应指向下一个节点。

示例 1：
输入：root = [1,2,5,3,4,null,6]
输出：[1,null,2,null,3,null,4,null,5,null,6]

示例 2：
输入：root = []
输出：[]

示例 3：
输入：root = [0]
输出：[0]

提示：

  • 树中节点数目范围是 [0, 2000]
  • -100 <= Node.val <= 100

这道题是 二叉树 (Binary Tree) 和 链表 (Linked List) 的结合体。

它是力扣（LeetCode）第 114 题 “二叉树展开为链表 (Flatten Binary Tree to Linked List)”。

这道题非常经典，因为它要求你 原地 (In-place) 修改树的结构。这不仅考察你对递归的理解，还考察你对指针操作（断链、重连）的“微操”能力。

2. 核心思路分析

2.1 为什么直接先序遍历不行？

如果我们写一个标准的先序遍历（根-左-右），把节点存在一个数组里，然后重新连接，是可以的。但这样需要 $O(N)$ 的额外空间。题目通常暗示我们要 原地修改。

2.2 核心策略：寻找前驱节点 (Predecessor)

让我们观察一下转换的规律：

对于任意节点 root：

  1. 如果有左子树，那么 左子树 应该插入到 root 和 root->right 之间。
  2. 具体怎么插？
    • root 的 左子树 变成新的 右子树。
    • root 原来的 右子树，应该接到哪里？
    • 应该接到 左子树最右边的那个节点（也就是左子树遍历完的最后一个节点）后面。

算法步骤（像拉链一样）：

  1. 拿到当前节点 curr。
  2. 如果 curr 有左孩子 (curr->left)：
    • 找到左子树中 最右边 的节点，记为 predecessor（因为在先序遍历中，它是 curr->right 的前一个节点）。
    • 把 curr 的 右子树 接在 predecessor->right 上。
    • 把 curr 的 左子树 移到 curr->right 上。
    • 把 curr->left 置空。
  3. curr 向右移动（处理下一个节点）。

4. 另一种思路：递归法 (后序遍历的变种)

如果你觉得指针指来指去太晕，可以尝试 递归。

但是，常规的先序遍历（根左右）很难操作，因为你刚处理完根，就把右指针覆盖了，导致右子树丢失。

我们可以用 变形的后序遍历 (右 -> 左 -> 根)。

如果我们按照 右 -> 左 -> 根 的顺序遍历，我们总是先处理完后面的节点。

我们维护一个全局变量 prev，指向 已经展平链表的头。

操作逻辑：

  1. 先递归处理 root->right。
  2. 再递归处理 root->left。
  3. 最后处理 root：
    • root->right = prev
    • root->left = nullptr
    • prev = root

5. 复杂度分析

时间复杂度：$O(N)$

  • 迭代法：每个节点被访问的次数是常数次（寻找前驱节点虽然是循环，但每条边最多走两次）。
  • 递归法：每个节点访问一次。

空间复杂度：

  • 迭代法：$O(1)$。这是真正的原地算法。
  • 递归法：$O(N)$。递归栈的开销（最坏情况树退化为链表）。

6. 初学者常见疑问 (Q&A)

  • Q: 为什么要找左子树的“最右”节点？
    • A: 因为我们要保持 先序遍历 的顺序。
    • 先序是：根 -> [左子树...] -> [右子树...]。
    • 展开后：根 -> [左子树展开...] -> [右子树展开...]。
    • 显然，右子树 应该接在 左子树 的 最后一个节点 后面。
    • 而左子树先序遍历的最后一个节点，正是其结构上 最右下 的那个节点。
  • Q: 为什么 root->left 必须置空？
    • A: 题目明确要求展开后的结构是 单链表，只能用 right 指针。如果不把 left 置空，它就不是链表，且可能会导致后续遍历出现环或错误判断。
