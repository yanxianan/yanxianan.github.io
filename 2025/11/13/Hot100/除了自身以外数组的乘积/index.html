238. 除了自身以外数组的乘积

题目：
给你一个长度为 n 的整数数组 nums，返回一个数组 output，其中 output[i] 等于 nums 中除 nums[i] 外其余所有元素的乘积。

示例 1：
输入：nums = [1,2,3,4]
输出：[24,12,8,6]
解释：对于索引 0，结果是 2×3×4=24；索引 1 是 1×3×4=12，依此类推。

示例 2：
输入：nums = [-1,1,0,-3,3]
输出：[0,0,9,0,0]

提示：

  • 2 <= nums.length <= 10⁵
  • -10⁴ <= nums[i] <= 10⁴
  • 保证答案在 32 位整数范围内。

这道题是 前缀和/后缀和 思想的变种，在面试中出现频率极高。

它的难点在于题目给出的两个“反人性”限制：

  1. 不能使用除法（否则算出所有数的乘积，再除以当前数就搞定了，虽然还要处理 0 的情况）。
  2. 时间复杂度 $O(n)$ 且 空间复杂度 $O(1)$（结果数组不计入空间）。

这实际上是在考察你对 “左右乘积列表” 技巧的掌握和空间复用的优化能力。

1. 明确问题描述

题目名称：除自身以外数组的乘积 (Product of Array Except Self)

题目大意：

给你一个整数数组 nums，返回一个数组 answer，其中 answer[i] 等于 nums 中除了 nums[i] 之外其余所有元素的乘积。

关键约束：

  • 题目数据保证数组中任意元素的全部前缀元素和后缀的乘积都在 32 位整数范围内（不用担心溢出）。
  • 请不要使用除法。
  • 进阶：在 $O(1)$ 的额外空间复杂度内完成。

示例：

输入：nums = [1, 2, 3, 4]

输出：[24, 12, 8, 6]

解释：

  • 索引 0: $2 \times 3 \times 4 = 24$
  • 索引 1: $1 \times 3 \times 4 = 12$
  • …以此类推

2. 核心思路分析

2.1 朴素思想：左右拆解

既然不能用除法，那我们只能硬乘。

对于数组中任意一个元素 nums[i]，它的“除自身以外的乘积”其实就是：

$$\text{answer}[i] = (\text{i 左边所有数的乘积}) \times (\text{i 右边所有数的乘积})$$

如果我们能快速知道每个位置的 左积 (Left Product) 和 右积 (Right Product)，相乘就是答案。

2.2 空间 $O(n)$ 的解法（预处理）

我们可以开两个辅助数组：

  1. L[i]: 表示下标 i 左边所有元素的乘积。
  2. R[i]: 表示下标 i 右边所有元素的乘积。

举例 [1, 2, 3, 4]：

  • L 数组：[1, 1, 2, 6] (注意：索引0左边没有数，乘积初始化为1)
  • R 数组：[24, 12, 4, 1] (注意：索引3右边没有数，乘积初始化为1)
  • answer：L[i] * R[i] $\rightarrow$ [24, 12, 8, 6]

虽然这样满足时间 $O(n)$，但用了两个辅助数组，空间是 $O(n)$。

2.3 空间 $O(1)$ 的极致优化（空间复用）

题目允许 返回数组 answer 不计入空间复杂度。我们可以利用 answer 数组本身来充当中间容器。

算法流程：

  1. 第一轮遍历（从左往右）：

    先用 answer[i] 存储 i 左边所有元素的乘积。

    此时 answer 数组就是刚才提到的 L 数组。

  2. 第二轮遍历（从右往左）：

    我们需要把 i 右边所有元素的乘积 乘进去。

    由于我们不能再开一个数组存右积，我们用一个临时变量 R 来动态维护当前位置右边的乘积。

    • 初始 R = 1。
    • 对于下标 i (从最后往前遍历)：
      • answer[i] 目前存的是左积。
      • answer[i] = answer[i] * R （这就变成了 左积 * 右积）。
      • 更新 R = R * nums[i] （把当前数乘进 R，为处理前一个数做准备）。

4. 复杂度分析

时间复杂度：$O(n)$

  • 我们进行了两次遍历：
    1. 从左到右计算左积：$O(n)$。
    2. 从右到左计算右积并更新结果：$O(n)$。
  • 总复杂度 $2n \approx O(n)$。

空间复杂度：$O(1)$

  • 除了用于返回的 answer 数组外，我们只使用了常数个变量 (n, i, R)。
  • 完全符合题目进阶要求。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么可以不用除法？这背后的数学原理是什么？
    • A: 这利用了乘法的结合律。$A \times B \times C \times D$ 除以 $B$，等价于 $(A) \times (C \times D)$。也就是把 $B$ 挖掉，算出左边的积，再算出右边的积，拼起来就行。
  • Q: 如果数组里有 0 怎么办？
    • A: 这个算法完美兼容 0。
    • 如果数组里有一个 0（比如 [1, 0, 3, 4]），那么算出结果是 [0, 12, 0, 0]，只有 0 那个位置有值。
    • 如果数组里有两个及以上的 0，那么结果全是 0。
    • 我们的算法只是单纯的乘法，0 乘任何数都是 0，逻辑上不需要特殊处理。而如果用“总积除以当前数”的方法，遇到 0 做除数程序就崩了。
  • Q: 初始化 answer[0] = 1 和 R = 1 容易忘怎么办？
    • A: 记住“乘法单位元”的概念。乘法的“空状态”就是 1（就像加法的空状态是 0）。对于最左边的数，它左边“空空如也”，所以左积是 1；对于最右边的数同理。
