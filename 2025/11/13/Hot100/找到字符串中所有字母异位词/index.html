438. 找到字符串中所有字母异位词

题目：
给定两个字符串 s 和 p，找到 s 中所有 p 的 字母异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

字母异位词 指由相同字母重排列形成的字符串（包括相同的字符串）。

示例 2：

提示：

  • 1 <= s.length, p.length <= 3 * 10⁴
  • s 和 p 仅包含小写字母

这道题是 定长滑动窗口 (Fixed-size Sliding Window) 的经典代表作。

如果说“无重复字符的最长子串”是窗口大小忽大忽小（不定长）的变形金刚，那么这道题的窗口就是固定框大小的扫描仪。它会教你如何在窗口平移的过程中，高效地更新数据，而不是每次都重算。

1. 明确问题描述

题目名称：找到字符串中所有字母异位词 (Find All Anagrams in a String)

题目大意：

给定两个字符串 s 和 p。找到 s 中所有是 p 的 字母异位词 的子串，返回这些子串的起始索引。

关键点：

  • 字母异位词：指由相同字母重排列形成的字符串（比如 “cba” 是 “abc” 的异位词）。
  • 子串：必须是连续的。
  • 隐含信息：要找的子串长度固定，必须等于 p 的长度。

示例：

输入：s = "cbaebabacd", p = "abc"

输出：[0, 6]

解释：

  • 起始索引 0 的子串是 “cba”，它是 “abc” 的异位词。
  • 起始索引 6 的子串是 “bac”，它是 “abc” 的异位词。

2. 核心思路分析

2.1 暴力法为什么慢？

假设 s 长度为 $N$，p 长度为 $M$。

如果我们遍历 s 中的每一个位置，截取长度为 $M$ 的子串，然后排序比较（或者统计字符），复杂度大概是 $O(N \cdot M \log M)$ 或 $O(N \cdot M)$。

当字符串很长时，重复统计每个窗口内的字符是非常浪费的。

2.2 核心策略：定长滑动窗口 + 差量更新

我们维护一个长度永远为 p.size() 的窗口。

关键在于窗口移动时的变化：

当窗口向右移动一格：

  • 进来了一个新字符（右边加入）。
  • 出去了一个旧字符（左边移除）。
  • 中间的字符完全没变！

利用这个特性，我们不需要每次都重新数窗口里有几个 ‘a’, 几个 ‘b’。我们只需要更新“进来的”和“出去的”那两个字符的计数即可。

2.3 比较两个计数数组

怎么判断窗口内的字符和 p 是不是异位词？

  • 给 p 做一个统计表（比如 count_p）。
  • 给当前窗口做一个统计表（比如 count_window）。
  • 如果这两个表一模一样（vector 可以直接 == 比较），那就是异位词。

4. 复杂度分析

时间复杂度：$O(n)$ （更精确是 $O(n \times |\Sigma|)$）

  • 我们遍历了一遍字符串 s，循环次数是 $n$。
  • 在每次循环中，我们做了两次数组更新（$O(1)$）。
  • 关键点：window_count == p_count 这一步比较，虽然是一行代码，但底层是比较两个长度为 26 的数组。这是一个常数操作（最多比较 26 次）。
  • 所以总复杂度是 $O(26 \cdot n) \approx O(n)$。

空间复杂度：$O(1)$

  • 我们只用了两个长度为 26 的固定数组。空间消耗不随输入字符串长度增长。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么要用 vector 比较，而不是直接用个变量记录“匹配上的字符数量”？
    • A: 这是一个很好的优化思路！
    • 其实可以用一个变量 diff 来记录当前窗口和 p 之间有多少个字符计数不一样。
    • 如果 diff == 0，就是异位词。
    • 这种做法虽然能把比较的 $O(26)$ 降到 $O(1)$，但代码逻辑会复杂不少（要处理计数从 0 变 1，从 1 变 0 等各种情况）。对于面试，直接比较 vector (长度26) 既清晰又足够快，是性价比最高的写法。
  • Q: 为什么 for 循环只到 n - m？
    • A: 因为我们的窗口大小是 m。
    • 最后一个合法的窗口起始位置是 n - m（比如 s 长度 10，p 长度 3，最后一个窗口下标是 7, 8, 9）。
    • 再往后，剩下的字符就不够凑满一个窗口了。
