  • 0 <= s.length <= 5 * 10⁴
  • s 由英文字母、数字、符号和空格组成

这道题是 滑动窗口 (Sliding Window) 算法的开山之作。

如果说“双指针”通常用于解决“成对匹配”的问题（如两数之和、盛水容器），那么“滑动窗口”则是专门用来解决 “连续子数组/子串” 问题的利器。

这道题在 LeetCode Hot 100 中名列前茅，掌握它，你就掌握了处理连续区间问题的通用模板。

1. 明确问题描述

题目名称：无重复字符的最长子串 (Longest Substring Without Repeating Characters)

题目大意：

给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

关键点：

  • 子串 (Substring)：必须是字符串中连续的一部分（比如 “abc” 是 “abcde” 的子串，但 “ace” 不是）。
  • 无重复：子串里不能有相同的字符。

示例：

输入：s = "abcabcbb"

输出：3

解释：因为无重复字符的最长子串是 “abc”，所以其长度为 3。

2. 核心思路分析

2.1 为什么暴力法不可行？

暴力法是列出所有的子串，然后一个个检查是否有重复字符。

  • 子串总数是 $O(n^2)$。
  • 检查一个子串是否有重复字符需要 $O(n)$。
  • 总复杂度 $O(n^3)$。如果字符串长一点，电脑就跑不动了。

2.2 核心策略：伸缩的窗口 (Sliding Window)

想象一下，我们在字符串上有一个“窗口”：

  • 窗口的左边界是 left，右边界是 right。
  • 窗口里的字符就是我们当前正在考察的子串。

我们的目标：让这个窗口尽可能地变大，同时保证窗口里没有重复的字符。

操作流程 (The “How”)：

  1. 扩张 (Expand)：right 指针主动向右走，把新字符加入窗口。
  2. 检查 (Check)：每次加入新字符 s[right] 前，问问窗口里：“你们中间有没有人跟 s[right] 一样？”
    • 没有冲突：太棒了！安全加入，计算一下当前窗口长度，更新最大值。
    • 有冲突：麻烦了。s[right] 已经在窗口里出现过了（假设出现在位置 X）。
  3. 收缩 (Shrink)：既然有冲突，我们就必须移动 left 指针，把窗口左边的字符一个个扔出去，直到把那个导致冲突的老字符扔掉为止。

数据结构选择：

我们需要快速判断“窗口里有没有这个字符”。

显然，哈希集合 (unordered_set) 是最佳选择，查找和删除都是 $O(1)$。

举例演示：s = "pwwkew"

  1. [p]: 无重复，max=1。
  2. [pw]: 无重复，max=2。
  3. [pww]: right 指向第二个 w。冲突！
    • left 开始动：扔掉 p。窗口变成 ww。还有冲突！
    • left 继续动：扔掉第一个 w。窗口变成 w。冲突解决！
    • 加入新 w，窗口现在是 [w]。
  4. [wk]: 无重复…

4. 复杂度分析

时间复杂度：$O(n)$

很多初学者看到 for 循环里套了个 while 循环，会担心是 $O(n^2)$。

其实是 $O(n)$，原因如下：

  • right 指针从 0 走到 $n-1$，走了 $n$ 步。
  • left 指针也只能从 0 走到 $n-1$，最多走 $n$ 步（它不回退）。
  • 每个字符最多被 insert 一次，被 erase 一次。
  • 总操作次数是 $2n$，所以是线性的。

空间复杂度：$O(|\Sigma|)$

  • 我们需要一个 unordered_set 存储窗口内的字符。
  • $|\Sigma|$ 代表字符集的大小。如果全是 ASCII 码，set 最多存 128 个字符；如果是全是小写字母，最多 26 个。
  • 所以空间复杂度是常数级的 $O(1)$ （或者说 $O(128)$）。

5. 进阶优化 (面试加分项)

虽然上面的代码已经是 $O(n)$，但在 C++ 中，unordered_set 还是有一定的开销。

数组模拟哈希表：

由于字符（char）本质上就是整数（ASCII 码 0-127）。

我们可以用一个固定大小的数组 int dict[128] 来代替 unordered_set。

  • dict[c] = true/false (或者记录下标)。
  • 数组访问比哈希表快得多，这在 ACM 竞赛中是标准写法。

跳跃优化：

上面的 while 循环是一步步移动 left。其实如果我们可以记录字符的下标（使用 map 或数组记录 last_index），我们可以直接让 left 跳到重复字符的下一位。

但这不会改变最坏时间复杂度（依然是 $O(n)$），只是减少了操作次数。对于初学者，掌握“一步步收缩”的模板更通用，因为很多题目（如“和大于K的最短子数组”）只能一步步缩。

6. 初学者常见疑问 (Q&A)

  • Q: 为什么计算长度是 right - left + 1？

    • A: 这是一个经典的“植树问题”。比如下标 2 到 下标 4，包含 2, 3, 4 三个数。长度 = $4 - 2 + 1 = 3$。

  • Q: 如果字符串为空 "" 怎么办？

    • A: 循环不执行，直接返回 max_len (初始化为0)，逻辑正确。

  • Q: 这和“最大连续 1 的个数”那类题一样吗？

    • A: 非常像！它们都属于 不定长滑动窗口。核心逻辑都是：

      1. right 进场。

      2. while (条件不满足) left 出场。

      3. 更新结果。

        记住这个模板，能秒杀 80% 的滑动窗口题。
