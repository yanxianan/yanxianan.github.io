543. 二叉树的直径

题目：
给定一个二叉树，你需要计算它的 直径长度。二叉树的直径是指树中任意两个节点之间最长路径的长度。这条路径可能穿过根节点，也可能不穿过。

注意：路径的长度是以边的数量来计算的。

示例 1：
输入：root = [1,2,3,4,5]
输出：3
解释：路径 [4,2,1,3] 或 [5,2,1,3] 的长度为 3。

提示：

  • 树中节点的数目范围是 [1, 10⁴]
  • -100 <= Node.val <= 100

这道题是二叉树题目中 “最容易被误解” 的题目之一。

它是力扣（LeetCode）第 543 题。
很多初学者看到“直径”，第一反应是：“哦，就是左子树的最大深度 + 右子树的最大深度嘛！”
错！大错特错！

因为 最长路径不一定经过根节点。它可以是左子树内部的一个“倒 V”字形，或者右子树内部的一个弯钩。

这道题考察的是你对 后序遍历 (Post-order Traversal) 的理解，以及如何在递归过程中使用 全局变量 来记录“历史最大值”。

1. 明确问题描述

题目名称：二叉树的直径 (Diameter of Binary Tree)

题目大意：
给定一棵二叉树，你需要计算它的直径长度。
一棵二叉树的直径长度是 任意两个节点 路径长度中的 最大值。
这条路径 可能穿过也可能不穿过根节点。
注意：两个节点之间的路径长度是以它们之间边的数目表示。

示例：

返回 3。路径是 [4, 2, 1, 3] 或者 [5, 2, 1, 3]。

反例（不经过根节点的情况）：

在这个例子中，经过根节点 1 的最长路径是 5-3-2-1 (长度 3)。
但是，真正的直径 是在左子树内部的 5-3-2-4-6 (长度 4)！

2. 核心思路分析

2.1 转换思维：每个节点都是“转折点”

虽然我们不知道最长路径在哪里，但每一条路径（倒 V 字形）一定有一个 最高点（转折点/LCA）。

我们可以遍历树中的 每一个节点，假设它是这个“转折点”，计算经过它的最长路径长度。

然后，我们在所有节点计算出的结果中，取一个 最大值 (Global Max)。

2.2 递归函数的职责分离

这里有一个难点：递归函数到底该返回什么？

  • 如果函数返回“直径”，那父节点没法用子节点的“直径”来计算自己的深度。
  • 所以，递归函数必须返回“深度 (Height)”，这样父节点才能计算它自己的深度。
  • 而“直径”的计算，作为递归过程中的一个 副作用 (Side Effect)，存储在一个 全局变量 里。

算法流程：

  1. 定义一个全局变量 maxDiameter = 0。
  2. 编写一个函数 height(node)：

  • Base Case：如果 node 是空，返回 0。

  • 递归：

  • L = height(node->left) （左边有多深？）

  • R = height(node->right) （右边有多深？）

  • 更新全局最大值：maxDiameter = max(maxDiameter, L + R)。

  • （这一步就是计算以当前节点为转折点的直径）

  • 返回给父节点：max(L, R) + 1。

  • （告诉爸爸：我这条胳膊最长有多少）

4. 复杂度分析

时间复杂度：

  • 我们需要访问每一个节点一次来计算它的高度。

空间复杂度：

  • 是树的高度。递归栈的深度取决于树高。

5. 初学者常见疑问 (Q&A)

  • Q: 为什么 maxDiameter 要用 L + R 而不是 L + R + 1？

  • A: 题目问的是 “路径长度”（边的数量），而不是节点的数量。

  • 如果左深度是 1，右深度是 1，说明左边有一条边，右边有一条边。总共 2 条边。所以是 L + R。

  • 如果题目问的是“路径上的节点数”，那就是 L + R + 1。

  • Q: 能不能不用全局变量？

  • A: 可以。你可以把 maxDiameter 作为一个引用参数 int& maxD 传进递归函数里。这在工程上更干净，避免全局污染。
