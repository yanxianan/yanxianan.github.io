24. 两两交换链表中的节点

题目：
给定一个链表，两两交换其中相邻的节点，并返回交换后的链表。

你不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

示例 1：
输入：head = [1,2,3,4]
输出：[2,1,4,3]

示例 2：
输入：head = []
输出：[]

示例 3：
输入：head = [1]
输出：[1]

提示：

  • 链表中节点的数目范围是 [0, 100]
  • 0 <= Node.val <= 100

这道题是链表操作中的**“微操大师”**训练。

它不再像“反转链表”那样大刀阔斧地改变整条链的方向，而是要求你精细地、两两一组地进行局部反转。这道题极其考验你对指针顺序的把控能力，稍有不慎就会断链（Memory Leak）或成环。

1. 明确问题描述

题目名称：两两交换链表中的节点 (Swap Nodes in Pairs)

题目大意：

给你一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。

  • 要求：你必须在不修改节点内部的值的情况下完成本题（即，只能进行节点交换）。
  • 空间复杂度：$O(1)$（这意味着尽量不要用递归，而是用迭代）。

示例：

输入：1 -> 2 -> 3 -> 4

输出：2 -> 1 -> 4 -> 3

输入：1 -> 2 -> 3

输出：2 -> 1 -> 3 （最后剩一个 3，不用换）

2. 核心思路分析

2.1 为什么容易晕？

假设我们要交换 1 和 2。

  • 仅仅把 1->next 改成 3，把 2->next 改成 1 是不够的。
  • 关键在于：谁指向 1？（前面的节点必须指向新的头 2）。
  • 如果 1 是头节点，头指针要变；如果 1 在中间，前驱节点的 next 要变。

2.2 核心神器：虚拟头节点 (Dummy Head)

为了避免处理“头节点变了”这种特殊情况，我们依然祭出 Dummy Node。

让 dummy 指向 1。这样 1 和 2 就变成了“普通的中间节点”，逻辑完全统一。

2.3 操作步骤 (三步走)

设我们要交换 node1 和 node2，它们的前驱是 prev。

初始状态：prev -> node1 -> node2 -> nextNode

我们需要把线重新接成：prev -> node2 -> node1 -> nextNode

步骤详解：

  1. 控制前驱：prev->next = node2 （让前驱指向 2）
  2. 回马枪：node2->next = node1 （让 2 指回 1）
  3. 接续后续：node1->next = nextNode （让 1 指向后面的 3）

做完这三步后，原来的 node1 变成了这组的尾巴。

下一步准备：将 prev 指针移动到 node1，准备处理下一组。

4. 复杂度分析

时间复杂度：$O(N)$

  • 我们遍历了链表一次，每次处理两个节点。
  • 总操作次数是 $N/2$ 次交换，线性复杂度。

空间复杂度：$O(1)$

  • 我们只用了 dummy, prev, node1, node2 等几个指针变量，没有使用递归栈或额外数组。

5. 进阶对比：递归写法 (Recursive)

虽然题目推荐 $O(1)$ 空间，但递归写法非常简洁，逻辑也很优美，适合在面试中展示思维。

递归逻辑：

“我只负责交换前两个节点，剩下的链表交给递归函数去处理，处理完把新的头节点还给我，我接在屁股后面。”

6. 初学者常见疑问 (Q&A)

  • Q: 交换顺序有讲究吗？
    • A: 有！必须先保存 nextNode（或者 node2）。如果你上来就写 node1->next = node2->next，那么 node2 本身的地址可能就找不到了（如果没提前存 node2），或者链表断开后无法访问后续节点。保存现场是链表操作的第一准则。
  • Q: 循环条件 prev->next && prev->next->next 顺序能反吗？
    • A: 不能。必须先判断 prev->next (第一个节点) 存在，才能去访问 prev->next->next (第二个节点)。如果第一个都不存在，访问第二个会直接报空指针异常 (Segmentation Fault)。
