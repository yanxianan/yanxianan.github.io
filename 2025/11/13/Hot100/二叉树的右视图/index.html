199. 二叉树的右视图

题目：
给你一个二叉树的根节点 root，想象自己站在它的右侧，返回你看到的节点值的顺序。

示例 1：
输入：root = [1,2,3,null,5,null,4]
输出：[1,3,4]

示例 2：
输入：root = [1,null,3]
输出：[1,3]

示例 3：
输入：root = []
输出：[]

提示：

  • 树中节点数目范围是 [0, 100]
  • -100 <= Node.val <= 100

这道题是 BFS (广度优先搜索) 的一个极其巧妙的变种。

它是力扣（LeetCode）第 199 题。

很多初学者看到“右视图”，第一反应是：“哦，一直往右走 (root->right->right...) 不就行了吗？”

错！

因为 右视图 $\neq$ 右边界。

如果树的左边很长（像一条长长的左臂），而右边很短，那么当你站在右侧看时，你是能看到左边露出来的部分的。

2. 核心思路分析

2.1 为什么是 BFS？

既然我们要找的是 “每一层” 最右边的那个节点，那么最自然的思路就是 层序遍历 (Level Order Traversal)。

只要我们能把树一层一层地剥开，对于每一层，我们只需要 拿走最后一个被遍历到的节点，它一定就是最右边的那个。

2.2 算法流程 (BFS)

这完全复用了我们之前讲过的 层序遍历模板。

  1. 初始化：队列 q，放入 root。
  2. 大循环：当队列不空时。
  3. 批处理：
    • 获取当前层的节点数 size = q.size()。
    • 循环 size 次（遍历当前层的所有节点）。
    • 关键判断：如果当前节点是这一层的 最后一个（即下标 i == size - 1），把它加入结果集。
    • 将左右孩子加入队列。

4. 进阶思路：DFS (递归法)

虽然 BFS 很直观，但 DFS 其实写起来更帅，而且能体现你对遍历顺序的掌控。

核心思想：

我们要找每一层的最右边的节点。如果我们改变 DFS 的遍历顺序，从 “根 -> 左 -> 右” 改成 “根 -> 右 -> 左” 会发生什么？

  • 我们每一次来到一个新的深度（Depth），第一个 访问到的节点，一定是这一层 最右边 的节点！

算法流程：

  1. 维护一个 result 数组。
  2. 递归函数 dfs(node, depth)。
  3. 先访问右孩子，再访问左孩子。
  4. 判断：如果 depth == result.size()，说明我们 第一次 来到这个深度。
    • 既然是先往右走的，第一次到的肯定是最右边的。
    • 记录该节点。

5. 复杂度分析

时间复杂度：$O(N)$

  • 无论是 BFS 还是 DFS，都需要访问所有节点一次。

空间复杂度：

  • BFS: $O(W)$，取决于树的最大宽度（队列大小）。
  • DFS: $O(H)$，取决于树的高度（递归栈深度）。

6. 初学者常见疑问 (Q&A)

  • Q: 如果题目改成“左视图”怎么做？
    • A:
      • BFS: 判断 i == 0（这一层的第一个）时加入结果。
      • DFS: 遍历顺序改成 根 -> 左 -> 右，依然判断 depth == res.size()。
  • Q: 为什么 DFS 方法里的 if 判断是 depth == res.size()？
    • A:
      • 刚开始 depth=0, res 是空 (size=0)。相等 $\rightarrow$ 放根节点。
      • 去下一层 depth=1, res 有 1 个元素 (size=1)。相等 $\rightarrow$ 放右孩子。
      • 回溯后再去左边，depth=1，但此时 res 已经有 2 个元素了 (size=2)。1 != 2 $\rightarrow$ 跳过（因为这一层已经有人占座了）。
