19. 删除链表的倒数第 N 个节点

题目：
给你一个链表，删除链表的倒数第 n 个节点，并返回链表的头节点。

示例 1：
输入：head = [1,2,3,4,5], n = 2
输出：[1,2,3,5]

示例 2：
输入：head = [1], n = 1
输出：[]

示例 3：
输入：head = [1,2], n = 1
输出：[1]

提示：

  • 链表中节点的数目范围是 [1, 20]
  • 1 <= n <= 链表长度

这道题是链表 “双指针” 技巧中非常优雅的一道。

它不需要像“环形链表”那样推导数学公式，也不需要像“反转链表”那样复杂的指针操作。它考验的是你对 “间距” (Gap) 的理解。

通常我们遍历链表只能从头走到尾，那如何“从后往前”数呢？答案就是：让一把尺子先行一步。

1. 明确问题描述

题目名称：删除链表的倒数第 N 个结点 (Remove Nth Node From End of List)

题目大意：

给你一个链表，删除链表的 倒数 第 n 个结点，并且返回链表的头结点。

关键约束：

  • 尝试使用 一趟扫描 实现。
  • 链表长度未知。

示例：

输入：head = [1, 2, 3, 4, 5], n = 2

输出：[1, 2, 3, 5]

解释：倒数第 2 个节点是 4。删除它，剩下的连起来。

2. 核心思路分析

2.1 为什么暴力法有点笨？

如果我们不知道链表有多长，最直观的方法是：

  1. 先从头走到尾，数一下一共有 $L$ 个节点。

  2. 倒数第 $n$ 个，就是正数第 $L - n + 1$ 个。

  3. 重置指针，再从头走 $L - n$ 步，找到目标的前驱节点，删除目标。

    这需要走 两趟。虽然后果不严重，但面试官通常会问：能只走一趟吗？

2.2 核心策略：前后双指针 (快慢指针的变种)

想象两个人（指针）在跑步：

  1. 让快指针 (fast) 先跑 n 步。
    • 此时，fast 和 slow（还在起点）之间的间距刚好是 n。
  2. 然后两人同时跑，速度一样（每次都走一步）。
    • 这个间距 n 会一直保持不变。
  3. 当 fast 跑到终点 (NULL) 时：
    • 因为间距是 n，所以 slow 刚好就停在了 倒数第 n 个节点 的位置吗？
    • 等等！如果要删除节点，我们需要找到被删除节点的 前一个节点 (prev)。
    • 所以，我们需要让 slow 停在 倒数第 n+1 个节点 的位置。

优化后的流程：

  1. 创建一个 虚拟头节点 (dummy)，让 slow 和 fast 都指向 dummy。
  2. 让 fast 先走 n + 1 步。
  3. 两个指针同时移动，直到 fast 指向 NULL。
  4. 此时 slow 指向的就是 倒数第 n 个节点的前驱节点。
  5. 执行删除：slow->next = slow->next->next。

4. 复杂度分析

时间复杂度：$O(L)$

  • $L$ 是链表长度。
  • fast 指针遍历了 $L+1$ 步（算上 dummy），slow 指针遍历了 $L-n$ 步。
  • 总体是一次遍历。

空间复杂度：$O(1)$

  • 我们只使用了常数个指针变量。

5. 初学者常见疑问 (Q&A)

  • Q: 如果没有虚拟头节点 dummy 怎么做？
    • A: 如果不用 dummy，让 fast 先走 n 步。
      • 如果 fast 走了 n 步后直接变成 NULL 了，说明倒数第 n 个就是头节点（即 n == 链表长度）。这时候直接返回 head->next。
      • 否则，fast 和 slow 一起走，直到 fast 到最后一个节点。此时 slow 在被删节点的前面。
    • 结论：不用 dummy 需要多写一个 if 特判删除头节点的情况。用 dummy 更统一。
  • Q: 为什么 fast 先走 n+1 步？只走 n 步可以吗？
    • A: 可以。
    • 如果 fast 先走 n 步，那么当 fast 走到链表最后一个节点（不是 NULL）时，slow 刚好停在被删节点的前驱。
    • 只要你把终止条件和步数对齐，两种写法都可以。我的写法（走 n+1，走到 NULL）在逻辑上更符合“间距”的直觉。
