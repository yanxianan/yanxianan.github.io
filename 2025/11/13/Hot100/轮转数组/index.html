189. 轮转数组

题目：
给定一个数组，将数组中的元素向右轮转 k 个位置，其中 k 是非负数。

示例 1：
输入：nums = [1,2,3,4,5,6,7], k = 3
输出：[5,6,7,1,2,3,4]
解释：向右轮转 3 步。[1,2,3,4,5,6,7] -> [7,1,2,3,4,5,6] -> [6,7,1,2,3,4,5] -> [5,6,7,1,2,3,4]

示例 2：
输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]

提示：

  • 1 <= nums.length <= 10⁵
  • -2³¹ <= nums[i] <= 2³¹ - 1
  • 0 <= k <= 10⁵

这道题是数组操作中的“魔术题”。

为什么叫它“魔术题”？因为它有一个经典的解法，初次见到时会觉得“极其反直觉”，但一旦理解了，你会感叹数学的对称美。而且，这道题考察的是空间复杂度 $O(1)$ 的原地操作能力。

1. 明确问题描述

题目名称：轮转数组 (Rotate Array)

题目大意：

给定一个整数数组 nums，将数组中的元素向右移动 k 个位置。

关键约束：

  • 原地算法：尽可能在原始数组上操作，使用 $O(1)$ 的额外空间。
  • k 是非负数，但 k 可能大于数组长度 n。

示例：

输入：nums = [1, 2, 3, 4, 5, 6, 7], k = 3

输出：[5, 6, 7, 1, 2, 3, 4]

解释：

  • 向右轮转 1 步: [7, 1, 2, 3, 4, 5, 6]
  • 向右轮转 2 步: [6, 7, 1, 2, 3, 4, 5]
  • 向右轮转 3 步: [5, 6, 7, 1, 2, 3, 4]

2. 核心思路分析

2.1 坑点预警：k 可能很大

如果数组长度 n = 5，让你轮转 k = 7 次。

实际上，轮转 5 次等于没动。轮转 7 次的效果 等于 轮转 2 次。

第一步预处理：k = k % n。这是所有解法的前提。

2.2 为什么暴力法和辅助数组法不够好？

  1. 暴力法：每次把最后一个数挪到最前面，其余后移。移动一次 $O(n)$，移动 $k$ 次就是 $O(n \times k)$。超时。
  2. 辅助数组法：新建一个数组，直接把算好位置的数放进去。时间 $O(n)$，但空间也是 $O(n)$。不符合“原地”的高级要求。

2.3 核心策略：三次翻转法 (The “Three Reversals” Trick)

我们需要把数组的后 k 个元素（尾巴），搬到数组的最前面（头部）。

观察示例：[1, 2, 3, 4, 5, 6, 7], k = 3

目标是：[5, 6, 7] (原尾部) + [1, 2, 3, 4] (原头部)

神奇的操作步骤：

  1. 翻转整个数组：

    [7, 6, 5, 4, 3, 2, 1]

    • 观察：现在的顺序是反的，但是 5, 6, 7 已经成功跑到了最左边（头部），1, 2, 3, 4 跑到了最右边。只是它们内部的顺序也是反的。

  2. 翻转前 k 个元素（也就是前 3 个）：

    [5, 6, 7] [4, 3, 2, 1]

    • 观察：头部的顺序恢复正常了！

  3. 翻转剩余的元素（也就是后 n-k 个）：

    [5, 6, 7] [1, 2, 3, 4]

    • 观察：大功告成！

为什么有效？

本质上，翻转操作类似于矩阵转置 $(AB)^T = B^T A^T$。

我们想把数组看作两部分 $A$ (头部)和 $B$ (尾部)，原数组是 $AB$。我们想要 $BA$。

$(A^R B^R)^R = (B^R)^R (A^R)^R = BA$。

（这里 $R$ 代表 Reverse 翻转）。

4. 复杂度分析

时间复杂度：$O(n)$

  • 第一次翻转整个数组：遍历 $n$ 个元素。
  • 第二次翻转前 $k$ 个：遍历 $k$ 个。
  • 第三次翻转后 $n-k$ 个：遍历 $n-k$ 个。
  • 总遍历次数：$n + k + (n-k) = 2n$。
  • 忽略常数，结果为线性时间 $O(n)$。

空间复杂度：$O(1)$

  • 我们没有申请新的数组 (new vector)。
  • myReverse 函数只使用了常数个变量 (start, end, temp 交换用)。
  • 符合题目要求的原地算法。

5. 进阶思考：环状替换 (Cyclic Replacements)

除了翻转法，还有一种方法叫 环状替换。

它的思路是：直接把 nums[i] 放到它该去的位置 (i+k)%n，把那个位置原本的数拿出来，再找下一个位置…

为什么不推荐首选这种方法？

虽然空间也是 $O(1)$，但代码很难写对。

  • 如果 $n$ 和 $k$ 的最大公约数不是 1（比如 n=6, k=2），你会发现你在这个环里转圈圈，回到了原点，但还有很多元素没动过！
  • 你需要处理这种“多轮循环”的情况，代码逻辑复杂，边界条件多，面试时容易翻车。
  • 翻转法逻辑清晰，不易出错，是最佳选择。
