memset和memcpy函数

发表于 2025-11-04 分类于 C_C++
本文介绍memset和memcpy函数。

memset 和 memcpy 是 C 语言标准库 <string.h> 中两个非常常用的内存操作函数，它们都用于直接操作内存，但用途和行为有明显区别。下面分别详细讲解：

一、memset 函数

1. 函数原型

2. 功能

将内存区域 s 的前 n 个字节全部设置为值 c（注意：c 被转换为 unsigned char）。

3. 参数说明

  • s：指向要操作的内存块的指针（通常是一个数组或结构体的地址）。
  • c：要填充的值（以 int 形式传入，但只取低 8 位，即一个字节）。
  • n：要填充的字节数。

4. 返回值

返回 s 的原始指针（即传入的第一个参数）。

5. 典型用途

  • 初始化数组（尤其是清零）。
  • 初始化结构体。
  • 快速将一段内存置为特定值（如 0 或 -1）。

6. 示例

⚠️ 注意：memset 是按字节填充的，因此不能用于将 int 数组初始化为任意非 0 值（例如 memset(arr, 1, sizeof(arr)) 不会把每个 int 设为 1，而是每个字节设为 1，结果是 0x01010101）。

二、memcpy 函数

1. 函数原型

2. 功能

从源内存区域 src 复制 n 个字节到目标内存区域 dest。

3. 参数说明

  • dest：目标内存地址（必须足够大以容纳 n 字节）。
  • src：源内存地址。
  • n：要复制的字节数。

4. 返回值

返回 dest 指针。

5. 典型用途

  • 复制数组。
  • 复制结构体。
  • 内存块整体拷贝。

6. 示例

⚠️ 注意：

  • memcpy 不检查内存重叠。如果 src 和 dest 有重叠区域，结果是未定义的。
  • 如果存在内存重叠，应使用 memmove（它能正确处理重叠）。

三、对比总结

特性 memset memcpy
作用 填充内存（设为同一字节值） 复制内存（从 src 到 dest）
参数数量 3 3
是否处理重叠 不适用（单内存块操作） 不处理重叠（用 memmove）
常见用途 初始化（如清零） 数据拷贝
安全性 安全（只要指针有效） 若内存重叠则不安全

四、使用建议

  • 初始化数组或结构体为 0：用 memset(ptr, 0, size)。
  • 拷贝数据：用 memcpy（确保无重叠）或 memmove（可能有重叠）。
  • 不要用 memset 初始化非字节对齐的非零值（如 int 数组设为 5）。
