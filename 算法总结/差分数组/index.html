差分数组：区间更新的“降维打击”技巧

本文字数： 3.3k 阅读时长 ≈ 5 分钟
本文详细讲解了算法竞赛与面试中常用的“差分数组”技巧，通过C++代码演示如何将区间更新的时间复杂度从O(n)优化至O(1)。

差分数组：区间更新的“降维打击”技巧

大家好，我是你们的算法博主。在之前的文章中，我们聊过了“前缀和”这个处理静态区间查询的神器。今天，我们要介绍一位与前缀和形影不离、甚至可以说是“互逆运算”的好兄弟——差分数组。

如果你在刷题时遇到过这样的场景：

“给定一个长度为 N 的数组，需要进行 M 次操作，每次操作将区间 [L, R] 内的所有数字加上 K。最后输出修改后的数组。”

如果 N 和 M 都达到了 10^5 级别，普通的暴力循环（每次操作遍历 L 到 R）会导致时间复杂度高达 O(N \times M)，这在 C++ 中绝对会超时（TLE）。

这时候，差分数组就是那个能将时间复杂度瞬间降维打击到 O(N + M) 的救命稻草。

一、核心思想：什么是差分数组？

简单来说，差分数组记录的是原数组相邻两项的差值。

假设我们有一个原数组 arr：

我们可以构造一个差分数组 diff，其中 diff[i] 表示 arr[i] 与 arr[i-1] 的差（对于 i=0，通常视为 arr[0] - 0 或者直接等于 arr[0]）：

神奇的性质：前缀和还原

差分数组最神奇的地方在于：对差分数组求前缀和，就能还原出原数组。

让我们试一下：

  • arr[0] = diff[0] = 2
  • arr[1] = diff[0] + diff[1] = 2 + 3 = 5
  • arr[2] = diff[0] + diff[1] + diff[2] = 2 + 3 + (-2) = 3
  • …以此类推。

这就建立了 原数组 <–> 差分数组 的双向通道。

二、为什么它能优化区间更新？

回到我们的核心问题：如何让区间 [L, R] 的所有元素加上 K？

如果在原数组上操作，我们需要遍历 L 到 R，耗时 O(R-L+1)。

但在差分数组上，我们只需要做两步操作：

  1. diff[L] += K
  2. diff[R + 1] -= K （如果 R+1 没有越界）

原理图解

想象一下，当我们对 diff[L] 加上 K 时，由于原数组是差分数组的前缀和：

  • 从索引 L 开始，后面所有的 arr 值在计算前缀和时都会多出一个 K。
  • 但是，我们只希望 [L, R] 区间增加 K，而不希望 R 之后的元素也增加。
  • 所以，我们在 R + 1 的位置减去 K。这样，当计算到 R + 1 及以后的前缀和时，刚才加上的 K 和这里减去的 K 抵消了，后面的值就保持不变了。

这就是所谓的“标记法”：只在区间的头和尾做标记，中间的过程交给最后的前缀和去“自动完成”。

三、C++ 代码实战

光说不练假把式。下面我们通过一个经典的模板题来演示如何实现。

题目描述

有一个长度为 n 的数组，初始全为 0。进行 m 次操作，每次给定 l, r, c，表示将区间 [l, r] 内的数全部加上 c。最后输出整个数组。

代码解析

  1. 数组大小：我们定义了 n + 2 大小的数组。这是因为当 R = N 时，R+1 会访问到 N+1 索引，如果不预留空间会发生越界错误。
  2. 输入处理：使用了 ios::sync_with_stdio(false)，这是 C++ 刷题的基本素养，防止输入输出成为瓶颈。
  3. 核心逻辑：
    • 读取 l, r, c。
    • diff[l] += c：表示从 l 开始，数值要增加 c。
    • diff[r + 1] -= c：表示从 r+1 开始，刚才增加的 c 要取消掉。
  4. 还原过程：最后的一个循环其实就是求前缀和。current_val 不断累加 diff[i]，得到的正是原数组在该位置经过所有操作后的最终值。

四、进阶：如果原数组不是全 0 怎么办？

上面的例子假设原数组初始全是 0。如果原数组 arr 初始是有值的，比如 {1, 2, 3, 4, 5}，该怎么构建差分数组？

方法很简单：我们可以认为初始状态是对一个全 0 数组进行了 N 次“单点更新”操作。
或者更直接地，根据定义手动构建：

  • diff[1] = arr[1]
  • diff[i] = arr[i] - arr[i-1] (对于 i > 1)

构建好初始的差分数组后，后续的区间更新操作完全一样，最后再求一次前缀和即可。

五、二维差分数组（扩展阅读）

差分的思想不仅可以用于一维数组，还可以推广到二维矩阵。
场景：对一个子矩阵 (x_1, y_1) 到 (x_2, y_2) 的所有元素加上 c。

在一维中我们操作了两个点，在二维中我们需要操作四个点，利用容斥原理来实现：

最后还原时，需要计算二维前缀和：
arr[i][j] = arr[i-1][j] + arr[i][j-1] - arr[i-1][j-1] + diff[i][j]

虽然二维差分稍微复杂一点，但核心思想依然是“端点标记，前缀和还原”。

六、总结与适用场景

什么时候该用差分数组？

  1. 频繁区间更新：题目涉及多次对数组的一段连续区间进行加减操作。
  2. 离线查询：所有的更新操作完成后，才需要查询结果或输出最终数组。（如果是边更新边查询任意点的值，差分依然适用；但如果是边更新边查询区间和，可能需要线段树或树状数组）。
  3. 数据量大：N, M 在 10^5 级别，暴力 O(N*M) 必超时。

时间复杂度对比：

  • 暴力法：O(N * M)
  • 差分数组：O(N + M) （构建/标记 O(M)，还原 O(N)）

差分数组是典型的“空间换时间”策略，用少量的额外空间（一个差分数组），换取了巨大的时间效率提升。掌握它，能让你在处理区间问题时游刃有余。
