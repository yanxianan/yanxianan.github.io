BootROM阶段

发表于 2026-05-06 更新于 2026-05-11 分类于 ZYNQ
本文字数： 1.7k 阅读时长 ≈ 2 分钟
本文介绍ZYNQ裸机上电的第一个阶段BootROM。

ZYNQ芯片上电后运行的第一个阶段就是BootROM。它是一段固化在芯片内部、物理上不可修改的只读存储器（掩膜ROM）代码。你可以把它理解为ZYNQ芯片的“出厂本能”或最底层的BIOS，它的核心使命非常单一且明确：完成最基本的硬件唤醒，并找到、加载下一级引导程序（FSBL）。

以下是BootROM阶段的详细拆解：

1. BootROM的启动触发与硬件准备

当开发板上电或按下复位键时，芯片会经历一个上电复位（POR）过程。此时硬件会自动完成以下操作：

  • 采样启动模式引脚：硬件会读取PS端特定的MIO引脚（通常是MIO[8:2]）的高低电平状态。这些引脚的接线（通常由板载拨码开关控制）决定了芯片接下来要从哪里读取程序，例如SD卡、QSPI Flash、NAND Flash还是JTAG接口。
  • CPU0开始执行：复位结束后，ARM处理器核CPU0会自动从芯片内部的BootROM起始地址开始取指令执行，而CPU1则会保持等待事件（WFE）的休眠状态。

2. BootROM的核心执行流程

BootROM代码执行时，DDR内存还没有被初始化，因此它只能在芯片内部极小的**256KB片上内存（OCM）**中运行。其内部流程主要包含以下几个关键步骤：

  • 基础硬件初始化：根据之前采样到的启动模式，BootROM会配置对应的存储控制器（如SD卡控制器或QSPI控制器），并初始化最基础的时钟树。

  • 搜索启动镜像（BOOT.BIN）：

    • BootROM会尝试从选定的启动设备中寻找启动镜像。
    • 它会在存储设备上寻找一个特定的BootROM头部信息（Header）。这个Header通常包含一个“XLNX”魔术字作为标识。
    • 智能搜索机制：如果在默认地址（如0x0）没有找到有效的Header，BootROM会触发“多启动（MultiBoot）”搜索机制。它会以32KB为步进，不断向后跳跃搜索，直到找到有效的Header或达到搜索边界。这也是为什么可以在Flash中存放多个系统镜像（如主镜像和备份镜像）来实现系统冗余备份的原理。

  • 解析Header并加载FSBL：

    • 找到有效的Header后，BootROM会解析其中的目录信息，获取第一阶段引导加载程序（FSBL）在镜像文件中的偏移量、大小以及它应该被加载到OCM中的具体地址。
    • BootROM将FSBL代码从外部存储设备（如SD卡）精准地拷贝到OCM的指定位置（通常是OCM的低地址区域，0x00000000附近）。

  • 移交控制权：FSBL加载完毕后，BootROM会通过一条跳转指令，将CPU的执行权彻底交给FSBL。至此，BootROM的使命结束，后续复杂的DDR初始化、PL比特流加载等工作全部由FSBL接手。

3. 两种主要的启动模式

BootROM的执行行为会根据启动模式的不同而有所差异，主要分为两大类：

  • 主启动模式（从Flash/SD卡启动）：
    这是最常见的量产模式。BootROM主动配置控制器，从NAND、NOR (QSPI)、SD卡等闪存设备中读取BOOT.BIN并加载FSBL。
  • JTAG从启动模式（调试模式）：
    主要用于开发调试阶段。当MIO引脚被配置为JTAG模式时，BootROM执行最少的系统配置，随后使能JTAG接口并进入空闲等待状态。此时，开发者可以通过电脑上的Vivado/SDK工具，利用JTAG下载器直接将FSBL或应用程序下载到芯片的OCM或DDR中运行。

4. BootROM阶段的关键限制与冷知识

  • FSBL大小限制：由于BootROM阶段DDR尚未初始化，FSBL必须被加载到容量仅为256KB的OCM中。而BootROM自身在执行时会占用OCM的高64KB区域，因此FSBL的实际可用空间必须小于192KB，否则会导致启动异常。
  • 无法修改：BootROM是芯片出厂时通过光刻工艺固化在硅片里的，任何用户都无法通过软件手段擦除或修改它。
  • 安全性检测：在安全启动场景下，BootROM还会负责校验eFUSE中的安全配置位，并对加密的FSBL进行初步的解密和身份验证，确保加载的代码未被篡改。
