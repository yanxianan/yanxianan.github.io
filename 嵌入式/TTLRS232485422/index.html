TTL，RS232，RS485，RS422

发表于 2026-04-28 分类于 嵌入式
本文字数： 3.1k 阅读时长 ≈ 4 分钟
本文详细介绍TTL，RS232，RS485，RS422物理层电平标准。

这四种都是串行通信中常见的物理层标准，通常用来传输UART数据。它们最核心的区别在于电平定义、信号传输方式以及支持的拓扑结构。下面从定义到转换，为你逐一拆解。

一、TTL 电平串口

TTL 本身是“晶体管-晶体管逻辑”电平标准，用在串口通信时，泛指直接使用芯片的UART逻辑电平，最常见的有 3.3V TTL 和 5V TTL。

  • 电气特性（单端信号）
    • 逻辑“0”：0V ~ 0.8V（低电平）
    • 逻辑“1”：2.0V ~ VCC（高电平，VCC为3.3V或5V）
    • 空闲状态：高电平
    • 传输方式：以公共地(GND)为参考的单端信号。
  • 工作模式：通常为全双工，使用 TX（发送）、RX（接收）、GND 三根线。
  • 优缺点
    • 优点：直接从芯片引出，无需额外转换芯片，成本极低。
    • 缺点：抗干扰能力很差，仅适合极短距离（通常建议不超过20-30厘米），稍有共模电压差就容易烧毁芯片。
  • 典型场景：单片机与板载蓝牙、WiFi模块、GPS模块等的通信。比如 Arduino 的串口管脚。

二、RS-232

经典PC串口标准，为了解决早期TTL信号易受干扰、传输距离短的问题。

  • 电气特性（单端信号，负逻辑）
    • 逻辑“1”（MARK）： -3V 至 -15V
    • 逻辑“0”（SPACE）： +3V 至 +15V
    • 空闲状态：逻辑“1”，即负电压。
    • 传输方式：仍为单端信号，以GND为参考，但对电平范围和噪声容限做了巨大提升。
  • 接口与工作模式
    • 全双工，点对点通信，仅允许一主一从。
    • 除 TX、RX 外，标准还定义了 RTS、CTS 等完整的硬件流控信号线，常见接口为 DB9。
  • 优缺点
    • 优点：电平高、抗干扰比TTL强，互连简单。
    • 缺点：通信距离短（标准约15米），速率偏低（标准最大115.2kbps），且只支持点对点。
  • 典型场景：过去的PC机串口、早期鼠标、工控机调试口、对网速要求不高的仪器仪表连接。

三、RS-485

为多设备、长距离、高抗干扰的工业级通信而生，是一种差分标准。

  • 电气特性（差分平衡信号）
    • 逻辑“1”：A、B两线电压差 ( V_A - V_B \ge +200mV )
    • 逻辑“0”：A、B两线电压差 ( V_A - V_B \le -200mV )
    • 核心优势：不看绝对电压，只看两线间的差值。差分信号能有效抵消共模干扰，这是它传输千米级距离的关键。
  • 工作模式与拓扑
    • 半双工：一对差分线（A/B）既发送又接收，需通过使能引脚（DE/RE）切换。因此通常称为“两线制”。
    • 全双工：使用两对差分线，收发分开，即“四线制”，本质就是RS-422。
    • 多站点总线：支持一个总线上挂载最多32个标准节点（使用1/4单位负载芯片可达128甚至256个）。所有从机地址独立，主机轮询通信。
  • 优缺点
    • 优点：抗干扰极强，速率与距离平衡性好（100kbps下可达1200米），支持多点组网，成本低。
    • 缺点：是总线型网络，需要软件或硬件进行数据碰撞管理和收发状态切换。
  • 典型场景：工业Modbus总线、楼宇自控、安防摄像头云台控制、仪表数据采集等，是目前工业领域最流行的串行通信标准。

四、RS-422

它与RS-485的电气特性完全相同，区别在于物理拓扑和应用模式。

  • 电气特性：与RS-485完全一致，( V_A - V_B \ge +200mV ) 为逻辑1。
  • 工作模式与拓扑
    • 全双工：使用4根线（一对发送T+/T-，一对接收R+/R-），收发可同时进行。
    • 主从广播拓扑：一个主设备的发送器，可同时驱动最多10个从设备的接收器。这种“一发多收”的模式不能反向进行多设备回传，从设备只能通过其接收线收到主设备数据，回传需通过另一对线单独连回主设备的接收口，且通常需要主设备进行选通。
  • 优缺点
    • 优点：全双工，无需切换方向；差分信号，抗干扰强；收发独立，通信逻辑简单。
    • 缺点：线缆多，无法像RS-485那样构建廉价的多机共享总线。
  • 典型场景：需要长距离、全双工、抗干扰的点对点链路，或一主多从接收（如向多台显示器广播信息），现在常被RS-485或以太网替代。

六、使用场景速览

  • 选 TTL：当你直接用微控制器（MCU）连接一个紧挨着的模块时，比如在面包板上连接一个蓝牙透传模块。
  • 选 RS-232：当需要连接一台传统工控机或旧设备的调试口，且距离在几米以内，只连一台设备。
  • 选 RS-485：当需要将几十台传感器或控制器用两根线串接起来，在几百米外统一采集数据，如工厂能源监控系统。
  • 选 RS-422：当需要一条长距离、抗干扰的全双工链路，且是一主控器向几个终端单向广播数据，或简单的收发分离点对点，且不想处理方向控制时。

七、如何互相转换

这些接口无法直连，必须通过专用转换芯片或模块，本质是电平转换和信号模式转换。

1. TTL ↔ RS-232

  • 芯片：经典芯片是 MAX232（5V）、MAX3232（3.3V）。
  • 原理：内部有电荷泵电路，能将TTL电压升压并反相到±10V左右的RS-232电平，反之亦然。
  • 转换模块：市面上常见的“USB转TTL”或“USB转RS-232”模块已集成好。

2. TTL ↔ RS-485

  • 芯片：MAX485、SP3485（3.3V）等。
  • 原理：将TTL的单端信号转换为A/B差分信号。由于是半双工，需将芯片的发送使能(DE)和接收使能(RE)接在一起，通过一个GPIO控制数据方向。
  • 自动流向：部分模块用分立元件搭出自动方向控制电路，无需MCU参与，接线更简单。

3. TTL ↔ RS-422

  • 芯片：MAX490、SP3490等。
  • 原理：收发独立，将TTL的TX转为T+/T-，将R+/R-转为TTL的RX，无需方向控制。

4. RS-232 ↔ RS-485/422

  • 方法：这类转换器内部实际上经过了 RS-232 → TTL → RS-485 的两次转换。因此买一个集成的“RS-232转RS-485”无源/有源转换器即可。
  • 注意供电：无源转换器从RS-232的握手信号“偷电”，稳定性较差，建议优先选外部供电的有源转换器。

5. RS-485 ↔ RS-422

  • 被动接线：因电气一致，在点对点全双工场景下，可将一个RS-485设备（四线全双工模式）与一个RS-422设备直连，只要保证A接R+、B接R-，交叉对应即可。但用于总线时，两者拓扑不兼容。

总结

理解它们的核心是抓住两点：

  1. 电平与信号：TTL/RS-232是单端对地，易受干扰；RS-485/422是差分，能耐大。RS-232的±电压本质上还是单端，只是电平范围大，抗干扰有改善但远逊于差分。
  2. 拓扑与双工：RS-232 是远古的点对点全双工；RS-485 是现代的少线缆、多站点半双工总线；RS-422 则是用多出的一对线换来的全双工广播模式；TTL 是纯粹给芯片用的低电压语言。

所有转换，就是在这些物理标准之间，通过芯片搭起桥梁。
