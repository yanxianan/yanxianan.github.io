嵌入式中的软链接和硬链接

发表于 2026-04-28 分类于 嵌入式
本文字数： 2k 阅读时长 ≈ 3 分钟
本文详解linux中的软链接和硬链接。

在 Linux 里，“链接”可以理解成给文件或目录起的一个别名或快捷方式。主要分两种：硬链接 和 软链接（也叫符号链接）。我用最生活化的方式来拆解它们。

一、先弄懂文件是怎么存的

在 Linux 中，一个文件其实分为两部分：

  • 数据本身：硬盘上存着的真实内容（文字、图片等）
  • 文件名 + inode 编号：inode 就像这个文件的身份证号，记录着数据存放在哪、谁有权限等。而文件名只是一张写着 inode 编号的“标签”。

当我们 ls -l 看到一个文件时，第一列权限前面的数字就是链接数，表示有多少个“标签”指向同一个 inode。

二、硬链接 —— 同一个文件的前门和后门

比喻：同一间房子，开了好几个门。
你有一间房子（文件的数据块），这间房原本只有一个门牌号（文件名）。硬链接就是给同一间房多开一扇门，贴上不同的门牌。无论你从哪个门进去，看到的是完全同一间屋子。如果一个门拆了，只要还有别的门，房子依然在，里面的东西不会丢。

创建硬链接：

本质：
硬链接让多个文件名指向同一个 inode 编号。系统并不区分哪个是“原本的文件名”，它们是完全平等的。

硬链接的特点（很重要）：

  • 共享同一份数据：修改任何一个，另一个同步变化（因为它们就是同一个文件）。
  • 不能跨文件系统：硬链接得在同一个分区里，因为不同分区有各自的 inode 管理区域，无法共用。
  • 不能给目录创建硬链接（除非用超级用户强制，但系统一般不允许，防止目录循环引用导致问题）。
  • 删除原文件不影响：只要还有一个硬链接存在，文件数据就不会被删除。只有当最后指向该 inode 的文件名被删除时，数据才释放。
  • 链接数可以通过 ls -l 的第二列看出来，数字表示有几个“门”通向这个 inode。

三、软链接 —— 路标式的快捷方式

比喻：一块路牌，写着“请去那间屋子”。
软链接本身是一间很小的新“小屋”，里面只存放目标文件的路径。你敲软链接的名字，系统就会按照路径找到真正的文件。如果你把目标文件拆了（删除），路牌还在，但指向的地方已经空了，就会找不到。

创建软链接：

本质：
软链接是一个特殊的新文件，有自己独立的 inode，数据内容就是目标文件的“路径字符串”。

软链接的特点：

  • 可以跨文件系统：因为只存路径，和 inode 无关，所以可以链接到另一个硬盘分区的文件。
  • 可以链接目录：比如 ln -s /usr/local/bin mybin。
  • 删除原文件后，软链接会失效：变成一个“死链接”（broken link），访问时报错“没有那个文件或目录”。
  • 修改软链接文件的内容，实际上会写入原文件，因为它只是一个跳转。
  • ls -l 会显示 lrwxrwxrwx，并且用箭头 软链接 -> 原文件 清晰标示。
  • 软链接的权限始终是 lrwxrwxrwx，实际权限由目标文件决定。

四、核心区别一张表

特征 硬链接 软链接
本质 与源文件完全相同的 inode 独立文件，存的是源文件的路径
inode 号 相同 不同
能否跨分区 不能 能
能否链接目录 通常不能 能
删除源文件后 链接仍有效 链接失效（变成死链接）
文件权限 与源文件完全同步 始终 lrwxrwxrwx，受目标限制
空间占用 几乎不占额外空间（仅多一个目录项） 占用一点点空间（存路径）
相对/绝对路径 不涉及路径概念 可以用相对路径或绝对路径

五、什么时候用哪个？

  • 要备份重要文件，防止误删：用硬链接。比如你有重要的数据文件，创建一个硬链接，即使原文件名被 rm 了，数据依然在。
  • 要方便快速访问、且可能链接到其他硬盘或目录：用软链接。比如给经常用的长目录创建快捷方式，或者让某个脚本调用其它分区的库。
  • 需要多个入口点共享同一份不停变动的配置：硬链接很合适，改动同步无延时。
  • 避免循环链接或清晰的引用关系：软链接相对更安全，因为它能清楚地看出路径。

一个通俗的记忆：硬链接是“分身”，软链接是“指向牌”。分身和本体同等重要，一体同命；指向牌指向某个位置，位置没了牌子就成了废纸。
