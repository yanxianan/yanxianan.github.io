U-Boot环境变量bootcmd和bootargs

发表于 2026-03-30 分类于 嵌入式
本文字数： 2.2k 阅读时长 ≈ 3 分钟
本文详细介绍U-Boot中的环境变量bootcmd和bootargs。

在嵌入式Linux系统移植中，bootcmd 和 bootargs 是 U-Boot 引导加载程序里两个至关重要的环境变量。它们共同协作，确保 U-Boot 能够正确地找到并启动 Linux 内核。

简单来说，它们的关系可以这样理解：

  • bootcmd 是**“怎么做”**：它是一系列命令，告诉 U-Boot 如何加载内核。
  • bootargs 是**“用什么配置”**：它是一串参数，告诉内核启动时需要哪些配置。

下面我们来详细解析这两个变量。

🚀 环境变量 bootcmd

bootcmd 是 U-Boot 启动流程中的核心命令载体。当 U-Boot 完成自身初始化，并且启动倒计时结束后，就会自动执行 bootcmd 变量中定义的命令序列。

  • 功能与作用：它的核心任务是为启动 Linux 内核做准备。这通常包括：

    1. 初始化必要的硬件（如存储设备、网络）。
    2. 从指定的存储介质（如 eMMC、SD 卡、NAND Flash）或网络（TFTP）中，将 Linux 内核镜像（如 zImage、Image）和设备树文件（.dtb）加载到内存（DRAM）的指定地址。
    3. 执行启动命令（如 bootz 或 bootm），将控制权交给内核。

  • 内容形式：bootcmd 的内容是一串类似 Shell 脚本的命令，命令之间通常用分号 ; 分隔。它还可以引用其他环境变量，使配置更加灵活。

⚙️ 环境变量 bootargs

bootargs (Boot Arguments) 是 U-Boot 传递给 Linux 内核的命令行参数。这些参数直接影响内核的启动行为和初始配置。

  • 功能与作用：它的核心任务是告诉内核如何初始化系统。内核在启动时会解析这些参数，以获取关键信息，例如：

    1. 根文件系统（Root Filesystem）的位置和类型。
    2. 系统控制台（Console）的设备节点和波特率。
    3. 内存布局、启动级别等。

  • 内容形式：bootargs 的内容是一系列以空格分隔的键值对。

  • 常见的 bootargs 参数：

    • console=：指定内核启动信息的输出设备。例如 console=ttymxc0,115200 表示使用 ttymxc0 串口，波特率为 115200。
    • root=：指定根文件系统所在的设备。例如 root=/dev/mmcblk0p2 表示根文件系统在 mmcblk0 设备的第 2 个分区。
    • rootfstype=：指定根文件系统的类型。例如 rootfstype=ext4。
    • rootwait：指示内核等待，直到根文件系统设备准备就绪后再进行挂载。这对于 MMC/SD 设备等初始化较慢的存储介质非常重要。
    • rw / ro：指定根文件系统的挂载方式，rw 为可读可写，ro 为只读。

📊 bootcmd 与 bootargs 对比总结

特性 bootcmd bootargs
功能 定义加载和启动内核的命令序列。 定义传递给内核的启动参数。
执行时机 在 U-Boot 阶段执行，倒计时结束后自动运行。 在 bootcmd 执行启动命令时，由 U-Boot 传递给内核。
作用对象 主要作用于 U-Boot 自身和底层硬件。 直接作用于 Linux 内核。
内容形式 类似 Shell 的命令脚本，用分号 ; 分隔。 由空格分隔的键值对参数。

🔧 如何配置与保存

在 U-Boot 命令行中，你可以使用以下命令来查看和修改这两个环境变量：

  1. 查看变量：使用 printenv 命令。
  2. 修改变量：使用 setenv 命令。
  3. 保存配置：使用 saveenv 命令将修改永久保存到存储介质（如 eMMC）中，否则重启后会丢失。

在系统移植的源码层面，这些变量的默认值通常在 U-Boot 源码的板级配置头文件（如 include/configs/xxx.h）中通过 CONFIG_BOOTCOMMAND 和 CONFIG_BOOTARGS 等宏来定义。
