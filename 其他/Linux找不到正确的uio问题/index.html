Linux找不到正确的uio设备问题

发表于 2026-06-21 更新于 2026-06-24 分类于 其他
本文字数： 1.1k 阅读时长 ≈ 2 分钟
本文介绍如何解决Linux找不到正确uio设备的问题

通过反编译确认设备树中 BRAM 节点使用了 generic-uio 兼容属性，但系统启动后却找不到对应的 UIO 设备。经过排查，根本原因是 uio_pdrv_genirq 驱动未绑定 generic-uio 设备，模块虽已加载，但缺少 of_id 参数导致无法自动匹配。解决后需进一步实现开机自动注册。以下是问题与解决方案的整理总结。

问题回顾

  • 现象：设备树已配置 BRAM 为 generic-uio，但 /dev/uio* 及 /sys/class/uio/uio*/name 中只出现 axi-pmon，没有 BRAM 的 UIO 节点。
  • 直接原因：uio_pdrv_genirq 模块已加载（lsmod 可见），但未与 compatible = "generic-uio" 的设备绑定，因为模块默认未携带 of_id 参数。
  • 验证过程：
    1. 设备在内核模型中存在：/sys/bus/platform/devices/a0000000.axi_bram_ctrl
    2. 其 compatible 属性确实是 generic-uio
    3. 驱动目录 /sys/bus/platform/drivers/uio_pdrv_genirq/ 中原先没有该设备
    4. 重新加载模块并指定 of_id="generic-uio" 后，设备立刻绑定，axi_bram_ctrl 出现在 UIO 列表（uio4）
