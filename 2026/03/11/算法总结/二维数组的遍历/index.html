零基础详解二维数组遍历：从旋转图像到螺旋矩阵

大家好，我是你们的算法向导。今天我们要聊的话题是许多新手在刷算法题时的“噩梦”——二维数组的特殊遍历。

普通的二维数组遍历，大家都会写：两个 for 循环，外层控行，内层控列，从左到右、从上到下扫一遍。这很简单。但是，当题目要求你把图片旋转90度，或者像贪吃蛇一样螺旋着走完整个矩阵时，很多初学者就晕了：下标怎么变？边界在哪里？会不会越界？

别担心，今天我们就把这两个最经典的场景——旋转图像和螺旋矩阵，掰开了揉碎了讲清楚。我们不用复杂的数学公式，只用最直观的代码逻辑和边界控制来解决问题。

一、核心心法：边界与方向

在处理任何复杂的二维数组遍历时，请记住两个核心概念：

  1. 边界收缩（Boundary Shrinking）：想象你在剥洋葱，或者在一个 shrinking 的框里走路。定义上、下、左、右四个边界，每走完一条边，就把对应的边界向内缩一格。
  2. 方向模拟（Direction Simulation）：不要试图一次性算出所有坐标的变换公式（除非你是数学大神）。对于螺旋遍历，最好的方法是模拟“走路”的过程：向右走 -> 撞墙了 -> 向下走 -> 撞墙了 -> 向左走…

二、场景一：旋转图像（Rotate Image）

题目描述：给定一个 n x n 的二维矩阵表示一个图像，将图像顺时针旋转 90 度。
难点：要求原地旋转，即不能使用额外的矩阵空间。

1. 直觉误区

很多人的第一反应是：我要创建一个新矩阵 newMatrix，然后找出旧坐标 (i, j) 和新坐标 (j, n-1-i) 的关系，填进去。
但这违反了“原地”的要求，而且思维负担重。

2. 高手解法：两次翻转

计算机图形学中有一个非常巧妙的技巧：任何旋转都可以分解为多次翻转（Flip）。
顺时针旋转 90 度 = 先沿主对角线翻转（转置） + 再沿垂直中轴线翻转（左右镜像）。

让我们看看这个过程：
假设矩阵是：

第一步：沿主对角线翻转（Transpose）
也就是把 matrix[i][j] 和 matrix[j][i] 交换。这就相当于把行变成了列。

第二步：左右镜像翻转（Reverse each row）
把每一行的元素顺序颠倒过来。

恭喜！这就是原图顺时针旋转 90 度的结果。

3. C++ 代码实现

代码解析：

  • 转置循环：j 从 i 开始而不是 0，是因为对角线左下方的元素会在处理右上方时被交换过来。如果 j 从 0 开始，你会把刚换过去的又换回来，导致矩阵不变。
  • STL 利器：reverse 函数极大地简化了代码，避免了手写交换逻辑出错。

三、场景二：螺旋矩阵（Spiral Matrix）

题目描述：给你一个 m 行 n 列的矩阵，请按照顺时针螺旋顺序，返回矩阵中的所有元素。
难点：方向会不断变化，且容易在角落处重复访问或越界。

1. 解题思路：剥洋葱法

我们可以定义四个变量来标记当前的“活动范围”：

  • top: 当前未遍历的最上行索引
  • bottom: 当前未遍历的最下行索引
  • left: 当前未遍历的最左列索引
  • right: 当前未遍历的最右列索引

遍历流程：

  1. 从左到右遍历 top 行。遍历完后，top 向下移一行（top++），因为这行已经处理完了。
  2. 从上到下遍历 right 列。遍历完后，right 向左移一列（right--）。
  3. 从右到左遍历 bottom 行。注意：此时要检查 top 是否已经超过了 bottom，防止重复遍历。遍历完后，bottom 向上移一行（bottom--）。
  4. 从下到上遍历 left 列。注意：同样要检查 left 是否超过了 right。遍历完后，left 向右移一列（left++）。

重复上述步骤，直到 top > bottom 或 left > right。

3. 易错点深度解析

为什么要在第 3 步和第 4 步加 if 判断？
想象一个只有一行的矩阵 [1, 2, 3]：

  1. 第一步从左到右遍历完，result 是 [1, 2, 3]。
  2. 此时 top 变成 1，bottom 还是 0。
  3. 如果不加 if (top <= bottom) 判断，程序会继续执行第 3 步（从右到左遍历底行）。此时 bottom 是 0，但 top 已经是 1 了，这意味着这一行其实在第一步已经被遍历过了！如果不检查，就会把最后一行重复添加进结果。

所以，每次改变边界后，都要为接下来的反向遍历做合法性检查。

五、总结与建议

今天我们学习了二维数组处理的两个高级技巧：

  1. 旋转图像：不要硬算坐标，利用转置 + 镜像的组合拳，代码简洁且不易出错。
  2. 螺旋遍历：使用四边界收缩法。定义 top, bottom, left, right，每走完一条边就收缩对应的边界。切记在反向行走前要检查边界有效性。

给新手的建议：

  • 画图：在纸上画出矩阵，用笔模拟走一遍，标出 top, bottom 的变化，比干想代码有效得多。
  • 小数据测试：写完代码后，先用 1x1, 1xN, Nx1, 2x2, 3x3 这种极端和小规模的数据在脑子里跑一遍，往往能发现边界条件的漏洞。
  • 熟悉 STL：C++ 的 swap, reverse, vector 操作能帮你节省大量编写底层逻辑的时间，让你更专注于算法思想。
