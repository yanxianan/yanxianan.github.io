事务和并发控制

1. 事务基本概念

1.1 什么是事务

事务是一组 SQL 操作的集合，具有原子性，要么全部执行成功，要么全部回滚。

1.2 ACID 特性

A (Atomicity) - 原子性：事务中的所有操作要么全部完成，要么都不完成

C (Consistency) - 一致性：数据库从一个一致的状态转移到另一个一致的状态

I (Isolation) - 隔离性：并发事务之间相互隔离，不会相互干扰

D (Durability) - 持久性：事务提交后，数据永久保存

3. 隔离级别

3.1 四个隔离级别

隔离级别 脏读 不可重复读 幻读 性能
READ UNCOMMITTED 可能 可能 可能 最快
READ COMMITTED ✓ 可能 可能 快
REPEATABLE READ ✓ ✓ 可能 中
SERIALIZABLE ✓ ✓ ✓ 最慢

3.2 问题定义

脏读（Dirty Read）：读取未提交的数据

不可重复读（Non-repeatable Read）：两次读取同一数据得到不同结果

幻读（Phantom Read）：两次读取返回的行数不同

5. 并发控制策略

5.1 悲观锁

假设并发冲突很多，先加锁再操作。

优点：简单直接，避免冲突重试
缺点：性能差，容易产生死锁

5.2 乐观锁

假设并发冲突少，先操作再验证。

优点：高并发性能好，无死锁
缺点：可能需要重试，不适合冲突多的场景

5.3 MVCC（多版本并发控制）

InnoDB 的默认并发控制机制。
