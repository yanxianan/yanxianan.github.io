1. JOIN 的基本概念

JOIN 用于根据表间的关系联合两个或多个表的数据。根据不同的条件，可以获取满足需求的结果集。

2.3 INNER JOIN 的特点

  • 如果一个学生的班级不存在，则该学生不会出现在结果中

3.3 LEFT JOIN 的特点

  • 返回左表的所有记录，即使右表没有匹配记录
  • 右表中不匹配的列显示为 NULL
  • 常用于查找”没有”的数据

12. 小结

本章介绍了 MySQL 中各种 JOIN 类型：

  • INNER JOIN：只返回匹配的记录
  • LEFT JOIN：返回左表的所有记录
  • RIGHT JOIN：返回右表的所有记录
  • FULL OUTER JOIN：返回所有记录（通过 UNION 实现）
  • CROSS JOIN：笛卡尔积
  • 自连接：表与自身的关联
  • 多表 JOIN：同时关联多个表

JOIN 是 SQL 中最重要的操作之一，掌握 JOIN 能够构建复杂的多表查询。接下来，我们将学习子查询，进一步增强查询能力。
