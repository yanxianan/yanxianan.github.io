2. 数值类型

2.1 整数类型

类型 字节数 范围 用途
TINYINT 1 -128 ~ 127 或 0 ~ 255 存储小数值
SMALLINT 2 -32768 ~ 32767 存储中等数值
MEDIUMINT 3 -8388608 ~ 8388607 存储中等数值
INT/INTEGER 4 -2^31 ~ 2^31-1 最常用

  • AUTO_INCREMENT：自动递增，从 1 开始

3. 字符串类型

3.1 定长字符串（CHAR）

特点：

  • 存储空间固定
  • 查询速度快
  • 适合存储定长数据（邮编、身份证、电话区号等）

3.2 可变长字符串（VARCHAR）

VARCHAR 原理：

  • 实际占用空间 = 字符串长度 + 1 或 2 个字节（记录长度）
  • 字符数 <= 255 时，多占 1 个字节
  • 字符数 > 255 时，多占 2 个字节

CHAR vs VARCHAR：

特性 CHAR VARCHAR
存储空间 固定 变长
查询速度 快 稍慢
更新操作 快 可能慢
适用场景 固定长度 不同长度

3.3 大型文本类型

类型 最大长度 用途
TEXT 2^16 - 1 ≈ 65K 短文本
MEDIUMTEXT 2^24 - 1 ≈ 16M 中等文本
LONGTEXT 2^32 - 1 ≈ 4G 长文本

4. 日期时间类型

4.1 DATE 类型

存储日期，格式为 ‘YYYY-MM-DD’

4.2 TIME 类型

存储时间，格式为 ‘HH:MM:SS’

4.3 DATETIME 类型

存储日期时间，格式为 ‘YYYY-MM-DD HH:MM:SS’

4.4 TIMESTAMP 类型

存储时间戳，自动记录修改时间

DATETIME vs TIMESTAMP：

特性 DATETIME TIMESTAMP
存储范围 1000-01-01 ~ 9999-12-31 1970-01-01 ~ 2038-01-19
自动更新 不支持 支持
时区转换 不支持 支持
存储空间 8 字节 4 字节

5. 二进制类型

类型 最大长度 说明
BINARY 255 定长二进制
VARBINARY 65535 可变长二进制
BLOB 65535 二进制大对象
MEDIUMBLOB 16MB 中型二进制
LONGBLOB 4GB 长二进制

7.1 ENUM 类型

ENUM 特点：

  • 节省存储空间（内部存储为整数）

9. 小结

本章介绍了 MySQL 中的各类数据类型：

  • 数值类型：整数、浮点数、定点数
  • 字符串类型：CHAR、VARCHAR、TEXT
  • 日期时间类型：DATE、TIME、DATETIME、TIMESTAMP
  • 二进制类型：BLOB
  • JSON 类型：灵活存储结构化数据
  • 枚举和集合类型：ENUM、SET

关键原则：根据实际需求选择合适的数据类型，平衡存储空间、性能和灵活性。
